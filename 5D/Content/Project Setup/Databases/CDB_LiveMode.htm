About Live mode

When you download items from the central database, you'll enter Live mode. When you are in Live mode, you will be notified when there are updates to any of the downloaded items. These updates include changes saved to items, items locked by other users, and locks released by other users. Before you can edit items from the central database, you'll need to lock the item. When you lock an item, other users will not be able to make any changes to the item. See "Locking items".

The icons in the Data List pane indicate the status of an icon.

  • A green lock icon () indicates that the item is locked by you. See "Locking items".
  • A red lock icon () indicates that the item is locked by another user.
  • A green flag icon () indicates that there are updates available for the item.

When an update is available, the green flag icon () will also appear on top of the Live Mode icon () in the main toolbar. To download the updates, click the Refresh button () in the main toolbar.

Tip

An item doesn't have to be locked in order to view its properties or to run anti-collision analysis.

Locking items

Before you can edit an item or import objects from a local file, the item must be locked in the central database. Locking an item prevents other users from making changes to the item or any of its child items.

To lock an item in the central database:

  1. Go to the Data List pane and select the item you want to lock.
  2. Click the Lock button () in the main toolbar.

The item and all child items will be locked for other users, and a green lock icon () will appear next to the item in the Data List pane. If there are any target sets or lease shapes associated with the item, they will also be locked. When you right-click on an item that you've locked, the options for editing the item will be enabled. When you save changes to the item, other users who have the item open will receive a notification to download updates.

To unlock the item, click the Release All button () in the main toolbar. Locks are also automatically released when you close the field.

Saving items locally

To save a local copy of a field to your computer:

  1. Download the field from the central database. See "Downloading items".
  2. Click the Save As button () in the main toolbar.
  3. In the Save As window that opens, click the Browse button.
  4. Navigate to the location where you want to save the field, and click Save.
  5. To open the local file, select the Load local file after save as? option.
  6. Click OK.

A local copy of the field will be saved to your computer. If you selected the Load local file after save as? option, the field from the central database will be closed, and all locks will be released.
