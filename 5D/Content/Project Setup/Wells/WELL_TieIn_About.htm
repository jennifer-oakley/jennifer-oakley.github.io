About tie-in wells

A tie-in well is a specialized well type that lets you plan a wellpath from a specific location on a reference well when you don't have the complete trajectory data. Unlike standard wells that start from surface, tie-in wells begin from a defined point along an existing wellbore.

The tie-in point is defined by specific well survey parameters at the connection location:

  • Position: Measured depth (MD), true vertical depth (TVD), northing, and easting coordinates
  • Attitude: Inclination and azimuth at the tie-in point

Once you've set up the tie-in point, you can create, edit, and archive wellpath plans just like standard wells. Tie-in wells can also interact with other field objects such as lease lines and targets for planning purposes.

Tie-in wells are particularly useful in these drilling scenarios:

  • Sidetrack planning: When planning a sidetrack from a specific point in an existing well without complete survey data for the parent wellbore
  • Multi-lateral wells: Planning branch wells that connect to a main wellbore at predetermined locations
  • Re-entry operations: Planning new wellpath segments from known points in existing wells
  • Conceptual planning: Early-stage planning when complete reference well data is not yet available

Tie-in well limitations

Because tie-in wells don't start from surface like normal wells, several standard 5D features aren't available:

  • Anti-collision analysis: Full collision analysis requires complete wellpath trajectory from surface
  • Survey management: Can't add survey data or use field operations tools
  • Sidetrack wells: Can't create additional sidetracks from a tie-in well

However, 5D provides specialized tools for tie-in wells:

  • Proximity reports: Show distances between tie-in wells and other wellpaths in the field
  • Lease line analysis: Limited proximity analysis between tie-in wells and lease line boundaries
  • Attachment capability: Option to attach tie-in wells to reference wells when complete trajectory data becomes available

All reports generated for tie-in wells include a disclaimer stating that the data is for planning purposes only and should not be used for actual drilling decisions.

Tie-in well workflow

There are two main approaches for working with tie-in wells, depending on whether you're creating a new well or working with an existing one. For both approaches, you can also attach the tie-in well to a reference well when complete trajectory data becomes available. See "Copying Tie-In Plans".

Option 1: Create a new tie-in well

When you need to create a new tie-in well from scratch:

  1. Add the tie-in well. Right-click on the slot in the Data List and select Add Floating Tie-On Well. This creates the well and allows you to define the tie-in point parameters during the creation process.
  2. Plan the wellpath. Use standard wellpath planning tools to create the desired trajectory from the tie-in point. See "Wellpath planning".
  3. Analyze proximity. Use specialized proximity analysis tools to check distances to other wells and lease lines. See "Well Path Proximity Reports".

Option 2: Convert an existing well to a tie-in well

When you have an existing well that you want to convert to a tie-in well:

  1. Edit the tie-in point. For an existing well with plans only (no surveys), edit the first point to define it as a tie-in point. See "Edit tie-in points".
  2. Plan the wellpath. Use standard wellpath planning tools to create the desired trajectory from the tie-in point. See "Wellpath planning".
  3. Analyze proximity. Use specialized proximity analysis tools to check distances to other wells and lease lines. See "Well Path Proximity Reports".
