About outgoing WITSML depth data

On the Setup > DAQ > WITSML Out (Depth) page, you can send calculated depth-based values to the WITSML server. There are four types of depth-based curves that you can publish to WITSML:

  • T&D FF Sensitivity Depth Data – The curves in the T&D FF Sensisitivity Depth Data category are created when you add the T&D FF Sensitivity Depth Data indicator to a dashboard. The graphs in the T&D FF Sensitivity Depth Data indicator are generated from real-time data and show the effects of different friction factors on hookload and surface torque. Separate hookload and surface torque curves are created for each friction factor. See "Friction factors indicator".
  • T&D Plan Depth Data – The curves in the T&D Plan Depth Data category are created from the data in the T&D Plan Depth Data category on the Dashboards screen. Separate hookload and surface torque curves are created for each operation. To configure the settings used to calculate torque and drag planned depth data, go to the Setup > MVA Settings > T&D Planned Settings tab. See "Torque & drag settings".
  • ECD Management (Depth) – The curves in the ECD Management (Depth) category are created from the data in the ECD Management (Depth) category on the Dashboards screen.
  • HydraulicsDepthData – The curves in the HydraulicsDepthData category are created when you add the SPP Modeling Static indicator to a dashboard. The graph in the SPP Modeling Static indicator compares real-time standpipe pressure to the standpipe pressure values calculated by the hydraulics model. See "SPP indicator".

Configuring outgoing depth data

To send depth-based curve data from CENTRO® Engineer:

  1. Go to the Setup > DAQ > WITSML Out (Depth) page.
  2. Select the parameters that you want to send to the WITSML server.
  3. In the Unit field, select the unit you want to use in the WITSML log.
  4. Click the Save Changes button ().

Each time new values are calculated, the existing data in the log will be deleted and replaced with the new data. For curves in the T&D FF Sensitivity Depth Data category, CENTRO® Engineer will use the WITSML log name specified in the indicator settings. See "Friction factors indicator". For curves in the other categories, CENTRO® Engineer will create a new depth log for each category on the WITSML server.

Field definitions

The table below defines the WITSML channel settings on the Setup > DAQ > WITSML Out (Depth) page.

Properties of outgoing WITSML depth data
Field Name Definition
Category Category that contains the CENTRO® Engineer curve data
Parameter Name of the CENTRO® Engineer curve
WITSML Channel WITSML channel receiving the data from CENTRO® Engineer. Depth data can only be sent to the first channel listed on the Setup > DAQ > WITSML page. See "WITSML channels".
Unit Type Unit type for the selected CENTRO® Engineer curve. This is a read-only field, determined by the curve selected in the Parameter field.
Unit Unit assigned to the curve on the WITSML server

Using the WITSML Out page

The toolbar at the top of the DAQ > WITSML Out (Depth) page is used to save and discard changes. To select the columns shown in the Outgoing Parameters table, click the Show Columns button () in the first column of the table.
