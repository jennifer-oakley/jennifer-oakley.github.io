About OPC UA data

The OPC UA protocol is a data exchange standard used for industrial communication, which allows machine-to-machine and PC-to-machine communication. On the Setup > DAQ > OPC UA page, you can configure CENTRO® Engineer to send and receive data from OPC UA devices.

There are three basic steps for sending and receiving OPC UA data:

  • Add an OPC UA channel. To connect to an OPC UA device, you'll need to know the URL to access the OPC UA device. See "Adding OPC UA channels".
  • Add inbound data tags. You'll need to add a data tag for each inbound parameter received from the OPC UA device. See "Adding inbound data tags".
  • Add outbound data tags. If you want to send data to the OPC UA device, you'll also need to add data tags for each outbound parameter sent to the OPC UA device. See "Adding outbound data tags".

On the WITSML Out page, you can configure CENTRO® Engineer to send OPC UA data and calculated values from CENTRO® Engineer to the WITSML server. See "WITSML Out".

Warning

If you delete a OPC UA channel, any associated raw data curves used in a dashboard on the Dashboards screen will also be deleted.

Adding OPC UA channels

To add an OPC UA channel:

  1. Go to the Setup > DAQ > OPC UA page.
  2. Click the Add button on the right side of the page.
  3. In the Channel Name field, enter a name for the OPC UA channel.
  4. In the URL field, enter the full address of the OPC UA device.
  5. Enter the frequency with which CENTRO® Engineer should query the device for data.
  6. Add data tags for each inbound parameter received from the OPC UA device. See "Adding inbound data tags".
  7. Add data tags for each outbound parameter sent to the OPC UA device. See "Adding outbound data tags".
  8. Click the Save Changes button ().

CENTRO® Engineerwill begin receiving data from the OPC UA device. To view the incoming OPC UA data, click on the Guides utility (), then click on Indicators View and select the DAQ option. See "OPC UA data". To check the status of incoming and outgoing data for the channel, go to the Alarms > Status > OPC UA tab. See "OPC UA alarms".

Adding inbound data tags

To add an inbound data tag to an OPC UA channel:

  1. Go to the Setup > DAQ > OPC UA page.
  2. Select the OPC UA channel that you want to add a data tag to.
  3. Go to the Inbound tab for the channel.
  4. Click the Add Data Tag button (). A new row will appear in the lower portion of the screen.
  5. In the Name field, enter the name of the parameter on the OPC UA device.
  6. In the Node Id field, enter the ID of the node on the OPC UA device that is sending the data.
  7. In the Unit Type field, select the type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
  8. In the Unit field, select the measurement unit for the data.
  9. Click the Save Changes button ().

The data tags will be added to the CENTRO® Engineer database. The name entered in the Name field will appear in the mappings settings and the DAQ indicator for the OPC UA channel. To view the incoming OPC UA data, click on the Guides utility (), then click on Indicators View and select the DAQ option. See "OPC UA data".

Adding outbound data tags

To add an outbound data tag to an OPC UA channel:

  1. Go to the Setup > DAQ > OPC UA page.
  2. Select the OPC UA channel that you want to add a data tag to.
  3. Go to the Outbound tab for the channel.
  4. Click the Add Data Tag button (). A new row will appear in the lower portion of the screen.
  5. In the new row that appears, go to the Object field and select the category for the data you want to send.
  6. In the Parameter field, select the parameter that contains the data you want to send.
  7. In the Node Id field, enter the ID of the node on the OPC UA device that is receiving the data.
  8. In the Unit Type field, select the type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
  9. In the Unit field, select the measurement unit for the data.
  10. Click the Save Changes button ().

The new entry will be added to the CENTRO® Engineer database. The outgoing data will be sent on the selected OPC UA channel. The data acquisition service will also be restarted so that CENTRO® Engineer can connect to the OPC UA channel.

Field definitions

The table below defines the OPC UA channel settings on the Setup > DAQ > OPC UA page.

Properties of OPC UA channels
Field Name Definition
Channel Name of the OPC UA channel
URL Full address of the OPC UA device
Frequency Frequency with which CENTRO® Engineer should query the device for data

The table below defines the fields for the data tags in a OPC UA channel.

Properties of OPC UA data tags
Field Name Definition
Name Name of the parameter on the OPC UA device
Object Category for the data you want to send. Only available for outgoing data tags.
Parameter Parameter that contains the data you want to send. Only available for outgoing data tags.
Node Id ID of the node on the OPC UA device that is sending or receiving the data
Data Type Type of data that is being sent. OPC UA devices send double data, which is numeric data that allows decimal values. This field is read-only and cannot be edited.
Unit Type Type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
Unit Measurement unit for the incoming data, e.g. inches or feet
