About parameters

Well parameters such as bit depth, well depth, ROP, and hookload are used by CENTRO® Engineer to create persistent curves for graphs and gauges. If you lose connection with the WITS or WITSML server, gauges and graphs that are mapped to well parameter curves will remain intact. CENTRO® Engineer also uses well parameters to calculate values such as flow rates, bottomhole temperature and pressure, circulation times, and total pump strokes. The data for well parameters can be collected from calculations or external data sources.

On the Setup > Mappings > Parameters page, you can assign the data source that CENTRO® Engineer uses for each well parameter. There are three types of data sources:

  • Calculated values, using the public calculations on the Setup > OneClick™ Settings > Calculations tab. You can use calculations created for the Monitoring algorithm or the Planning algorithm. See "OneClick™ real-time calculations" and "OneClick™ analysis calculations".
  • WITS data, received from incoming WITS data defined on the Setup > DAQ > WITS > In tab. See "WITS data".
  • WITSML data, received from a WITSML channel defined on the Setup > DAQ > WITSML page. See "WITSML channels".
  • OPC-UA data, received from an OPC-UA channel defined on the Setup > DAQ > OPC UA page. See "OPC UA data".

Defining well parameters

To choose a data source for the well parameters:

  1. Go to the Setup > Mappings > Parameters page.
  2. Right-click on the header for the Source Category column, then select the type of data you want to use.
    • Calculations – Uses the result from a calculation.
    • Wits Data – Uses incoming WITS data.
    • WITSML channel name – Uses incoming data from the selected WITSML channel.
    • OPC UA channel name – Uses incoming data from the selected OPC UA channel.
  3. In the Source Parameter field for each parameter, select the name of the data source you want to use. If your data uses standard mnemonics, the parameter will be automatically mapped to the correct mnemonic. See "List of well parameters".
  4. In the Mapping Action field, select a mapping action for invalid and null values. See "Choosing a mapping action".
  5. When you have finished defining the well parameters, click the Save Changes button () to save your changes.

The well parameters will be updated with the values from the selected data sources. If your data uses standard mnemonics, the parameters will be automatically mapped to the correct mnemonics. See "List of well parameters". To view the incoming well parameter data, click on the Guides utility (), then click on Indicators View and select the Parameters option. See "Parameters indicator".

Choosing a mapping action

The option selected in the Mapping Action field tells CENTRO® Engineer how to handle invalid values for well parameters. Null values and values outside of the minimum and maximum range defined on the Alarms > Set Limits > Parameters page will be considered invalid, even if the alarm for that parameter is not enabled. See "Set Limits".When CENTRO® Engineer receives an invalid value, the selected mapping action will determine how CENTRO® Engineer handles the invalid value. You can also use a mapping action to override an incoming value, even if it a valid value.

There are four options available in the Mapping Action field:

  • As Is – Ignores the validation results and accepts the invalid value as-is.
  • Default – Overrides the invalid value with the value entered in the Value field. Only invalid values will be overwritten with the default value.
  • Previous – Overrides the invalid value with the last valid value received.
  • Override – Overrides any value with the value entered in the Value field. All values for the parameter will be overwritten with the override value, even if they are valid values.
Tip

When you select the Default option or the Override option, you can use a simple expression for the override value. Simple expressions allow you to multiply an incoming value by a constant value. To enter a simple expression in the Value field, use the following format: [ConstantValue]*$value, e.g. 0.9*$value.
