About flows

Flows are workflows that are used by the overall workflow for the run algorithm on the Run tab. They can also be used by other flows on the Flow tab. Each flow is defined by a combination of nodes. There are three types of nodes that you can add to a flow:

  • Flow nodes are workflow nodes that direct the run algorithm to a subordinate flow, which is also defined on the Flows tab. When the subordinate flow is completed, the run algorithm will return to the next step in the original flow. See "Adding flow nodes to a flow".
  • Trigger nodes are conditional nodes that use the condition checks defined on the Conditions tab. Each trigger node has a No branch and a Yes branch. If the condition check is false, the run algorithm will follow the No branch. If it is true, it will follow the Yes branch. See "Adding trigger nodes to a flow".
  • Action nodes are used to execute certain types of actions, such as triggering an event or adding a comment. When the run algorithm uses the flow, it will execute the actions listed in the action node. See "Adding action nodes to a flow".

Creating flows

To create a new flow:

  1. Go to the Flows tab on the Setup > OneClick™ Settings™ > Real Time page.
  2. Click the Add button () at the top of the screen.
  3. In the Flow Editor pane that appears, add nodes to the flow as needed.
  4. When you are finished adding nodes, click the Apply button () to close the Flow Editor pane.
  5. A new row will appear at the bottom of the table. Double-click inside the Flow column, and enter a name for the new flow.
  6. Click the Save Changes button () to save the new flow.

The new flow will be saved to the CENTRO® Engineer database. You can add it as a flow node in another flow, or you can add it to the run algorithm on the Run tab. See "OneClick™ real-time run algorithm".

Adding flow nodes to a flow

Before you can add a flow node to another flow, you'll need to create the appropriate subordinate flow. See "Creating flows".

To add a flow node to an existing flow:

  1. Go to the Flows tab on the Setup > OneClick™ Settings™ > Real Time page.
  2. Click the Edit button () in the row for the flow you want to edit.
  3. In the Flow Editor pane that appears, select the node that is closest to where you want to add the flow node.
    • To add a node immediately below the selected node, click the Flow Node button () in the gray box below the existing node.
    • To insert a node above the selected node, click the Insert Flow Node button () in the toolbar at the top of the screen.
  4. In the menu that appears, select the subordinate flow you want to use for the node.
  5. When you are finished editing the flow, click the Apply Subtree button () to close the Flow Editor pane.
  6. Click the Save Changes button () to save the flow.

The selected flow will be updated in the CENTRO® Engineer database. When the run algorithm uses the flow, the new flow node will be used to direct the run algorithm to the subordinate flow assigned to the flow node. When the subordinate flow is completed, the run algorithm will return to the next step in the original flow.

Adding trigger nodes to a flow

Before you can add a trigger node to a flow, you'll need to create the appropriate condition check on the Conditions tab. See "OneClick™ real-time condition checks".

To add a trigger node to an existing flow:

  1. Go to the Flows tab on the Setup > OneClick™ Settings™ > Real Time page.
  2. Click the Edit button () in the row for the flow you want to edit.
  3. In the Flow Editor pane that appears, select the node that is closest to where you want to add the trigger node.
    • To add a node immediately below the selected node, click the Trigger Node button () in the gray box below the existing node.
    • To insert a node above the selected node, click the Insert Left Trigger Node button () or the Insert Right Trigger Node button () in the toolbar at the top of the screen.
  4. In the menu that appears, select the condition you want to use for the node.
  5. When you are finished editing the flow, click the Apply Subtree button () to close the Flow Editor pane.
  6. Click the Save Changes button () to save the flow.

The selected flow will be updated in the CENTRO® Engineer database. When the run algorithm uses the flow, the new trigger node will be used to determine which branch of the workflow to follow.

Adding action nodes to a flow

Action nodes can be used to execute the following types of actions:

  • Calculate – Instructs CENTRO® Engineer to perform the selected calculation.
  • Calibrate FF – Instructs CENTRO® Engineer to calibrate the friction factor.
  • Comment – Adds a comment to the log.
  • Event – Triggers the selected event, defined on the Setup > OneClick™ Settings >Real Time > Events tab.
  • Message – Triggers a message that appears in the status banner at the top of the screen.
  • Operation – Changes the current rig operation.

To add an action node to an existing flow:

  1. Go to the Flows tab on the Setup > OneClick™ Settings™ > Real Time page.
  2. Click the Edit button () in the row for the flow you want to edit.
  3. In the Flow Editor pane that appears, select the node that is closest to where you want to add the action node.
    • To add a node immediately below the selected node, click the Action Node button () in the gray box below the existing node.
    • To insert a node above the selected node, click the Insert Action Node button () in the toolbar at the top of the screen.
  4. In the menu that appears, select the Actions option.
  5. In the table that appears, select the actions that should be associated with the new action node.
  6. When you are finished editing the flow, click the Apply Subtree button () to close the Flow Editor pane.
  7. Click the Save Changes button () to save the flow.

The selected flow will be updated in the CENTRO® Engineer database. When the run algorithm uses the flow, it will execute the actions listed in the new action node.
