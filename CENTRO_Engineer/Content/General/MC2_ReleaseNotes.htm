CENTRO® Engineer v2.4.15 Release Notes

In this article

What's new

Bugs fixed

Known issues

Welcome to CENTRO® Engineer v2.4.15. OneSync® has joined the CENTRO® family of products, and has been re-branded as CENTRO® Engineer. This version of CENTRO® Engineer contains several new features and enhancements, as well as some bug fixes. For a full list of the updates in v2.4.15, refer to the sections below.

In addition, installation and upgrade guides for CENTRO® Engineer v2.4.15 may be downloaded from the Software Solutions Site.

Note

If you currently have OneSync® v 2.1.4 or older installed, you must uninstall the existing version of CENTRO® Engineer before installing CENTRO® Engineer v2.4.15. All previously stored data will be lost. If you need to retain your existing data, contact the CENTRO® Engineer Support Team for assistance prior to installing CENTRO® Engineer v2.4.15. See "Reporting problems".

What's new

The features in this section are new for CENTRO® Engineer v2.4.15.

  • On the Setup > Fluids > Cement page, you can now enter dial readings for θ60 and θ30. See "Dial readings".
  • When you add a production tubing section on the Setup > Wells > Well Intervals page, you can now enter a setting depth that is less than the setting depth of the previous casing section. WISE# 1100736 See "Casing and liner sections".
  • On the Setup > Wells > Well Intervals page, you can now define separate axial and rotational friction factors for each well interval. Previously, dual friction factor modeling was only available when the Advanced FF option was enabled. See "Well intervals".
  • Removed torque and drag calibration analysis. To run a torque and drag analysis, go to the Planning > Mechanics > Torque and Drag (T&D) page. See "Torque and drag analysis".
  • Added a new measurement unit for torque, deca Newton-meters (daN-m). To switch to the new unit, go to the Global Settings > Units page and select the daN-m option in the Torque field. WISE #1094881 See "Working with unit systems".
  • On the Setup > Surface Equipment > Piping and Instrumentation page, removed the maximum input limit for the Max Pump Pressure and Pop Off Pressure fields. Previously, both fields had a maximum value limit of 2000 psi. WISE 1103642

Bugs fixed

The critical issues in this section have been fixed in CENTRO® Engineer v2.4.15.

  • When you add a casing section to a well interval on the Setup > MVA Settings > Well Activity Planner page, the default setting depth for the casing will now match the end depth of the well interval. Previously, the default setting depth was 20 ft less than the well interval's end depth. Zendesk #149199
  • When you add a bi-center bit to a work string, the OD of the pilot section will now be used to determine the bit's maximum OD. Previously, the OD of the reamer section was used, which sometimes prevented you from adding the bit to the active work string. Zendesk #926875
  • Fixed an issue that caused torque and drag analysis reports to sometimes give an error that an exception has been thrown. WISE #1100777
  • Corrected the validations for well intervals created on the Setup > MVA Settings > Well Activity Planner page. When you run an analysis, the max OD of each component will now be validated based on the ID of the well interval at the depth where the component is placed. WISE 1100618, 1100696
  • When you click the Range button () on the Dashboard page, the menu to set the range should now open correctly. Previously, the menu sometimes did not open when there was a large amount of data. WISE 110214
  • When you add a 6.75" LWD IDS tool to a work string, the maximum OD should now correctly reflect the stabilizer configuration. Previously, the maximum OD included the stabilizer, even for slick configurations.
  • Fixed several issues related to 6.75" LWD IDS tools with a slick configuration. WISE 1100550
    • The diagram shown in the Well View pane should no longer show a stabilizer for slick configurations.
    • The maximum OD should now be the same as the section OD.
    • When you select the Slick option in the Stabilizer Configuration field, the selection should no longer revert to the Sensor Stabilizer only option.
  • When you are gap filling data or recalculating data, the mapping action selected on the Setup > Mappings > Parameters page should now work correctly. Previously, the mapping action only triggered during realtime data acquisition. Zendesk #973637
  • When you run a hydraulics snapshot, the pressure drop calculations for HEL MWD tools will now include the pressure drop across the IDS component. WISE 1109207
  • When you change the flow rate for a vibration analysis experiment and run the analysis again, the critical speed maps in the generated report now display the updated flow rate. Previously, the maps in the report incorrectly showed the original flow rate. WISE 1109077

Known issues

The issues in this section are known issues for CENTRO® Engineer v2.4.15.

  • The makeup torque for some connection types is incorrect for Magnus™ RSS units.
  • The length of a Revolution© RSS unit cannot be edited in the table on the Setup > Work String page.
  • After you add a graph or T&D FF Sensitivity Depth Data indicator () to a dashboard, the index for the graph may disappear. To resolve this issue, restart the CENTRO® Engineer application.
  • When you import formation data from a CSV file on the Setup > Wells > Formations page, commas must not be used to separate values over 1000. CENTRO® Engineer will interpret these commas as an extra column, and the import will fail.
  • When you recalculate or reprocess data using a custom algorithm, the calculation time may be significantly slower.
  • When you drag a curve in a time-based graph from one track to another track, the curve data may get corrupted and show abnormal spikes in the data.
  • When you run a hydraulics snapshot analysis, cuttings transport ratio may not be calculated correctly. In some cases, the calculated value for cuttings transport ratio may exceed 100%.
  • When you run a hydraulics snapshot analysis, the values calculated for ECD, ECD with cuttings, cuttings concentration, and bed height may not correctly account for changes in the wellbore geometry.
  • When you run a hydraulics snapshot analysis, the fluid properties do not account for pressure and temperature changes downhole. In the analysis results, PV, YP, and LSYP will have the same values throughout the well.
