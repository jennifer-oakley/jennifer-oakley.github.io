About actual risks

Actual risks are risks that have actually occurred while drilling the current well. When you record a new risk or event, the record will be automatically published to the first WITSML channel listed on the Setup > DAQ > WITSML page, and a new risk object will be created on the WITSML server.

Recording actual risks

To record an actual risk or other type of event:

  1. Go to the Reports > Risks > Actual page.
  2. Click the Add button () at the top of the screen.
  3. In the panel that appears, select the type of event and the probability of the event occurring.
  4. Select a category and subcategory for the event.
  5. Enter the start time and end time. See "Field definitions".
  6. Enter a title for the event in the Subject field.
  7. Enter a brief description of the event in the Description field.
  8. Click the OK button () to save your changes.

The new risk will be automatically saved. If you have a WITSML channel configured, a new risk object will be automatically created on the WITSML server. See "WITSML channels".

Exporting actual risks

To export risks and events to a CSV file:

  1. Go to the Reports > Risks > Actual page.
  2. Click the Export Risks button () at the top of the screen.
  3. In the panel that appears, go to the Event Type field and select the type of event you want to include in the export.
  4. Click the Export button ().
  5. In the window that opens, navigate to the location where you want to save the export and click the Save button.

The selected risks and events will be saved to a CSV file in the selected file location.

Field definitions

The table below defines the fields for risks on the Reports > Risks > Actual page.

Field definitions on the Reports > Risks > Actual page
Field Name Definition
Event Type

Type of risk event

  • Best Practice – Comment recommends a best practice procedure that is generally accepted as superior to other alternatives.
  • Event – Comment describes a problem that occurs during rig operations, such as a kick, stuck pipe, or tool failure.
  • Lessons Learned – Comment describes lessons learned that should be taken into account in the future.
  • Near Miss – Comment describes a near miss event that had the potential to cause injury, environmental damage, equipment damage, or non-productive time.
  • Other – Comment describes a type of event that is not listed.
  • Risk – Comment describes a potential problem that may arise during rig operations, such as a potential loss circulation zone or a zone of high vibration.
  • Unknown – Comment describes an unknown type of event.
Probability

Probability of the event occurring
Category

Category for the risk event. The selected category determines which options are available in the Sub Category field.

  • Bit – Event is related to problems with the bit, such as excessive bit wear, high ROP, or broken cutters. Only available for Best Practice, Event, and Lesson Learned event types.
  • Casing – Event is related to problems with casing, such as casing collapse, surge losses, or shoe failures. Only available for Best Practice, Event, and Lesson Learned event types.
  • Completion – Event is related to problems encountered during completion, such as casing collapse, surge losses, or shoe failures. Only available for Best Practice, Event, and Lesson Learned event types.
  • Directional Drilling – Event is related to problems encountered during directional drilling, such as abnormal tendency changes, excessive doglegs, or well collisions.
  • HSE – Event is related to HSE concerns, such as political unrest, gas hydrates, or personnel.
  • Hydraulics – Event is related to hydraulics problems, such as gas kicks, lost circulation, or high mud weight.
  • Mechanical – Event is related to mechanical issues, such as failed inspections, stuck pipe, or uneven wear.
  • Other – Event is related to an area that is not listed.
  • Unknown – Event is related to an unknown area.
  • Wellbore Stability – Event is related to wellbore stability issues, such as fracture problems, leak-off tests, or unstable zones.
Sub Category Sub-category for the event. The options listed are determined by the category selected in the Category field.
Start Time

Time that the event began or is expected to begin
End Time

Time that the event ended or is expected to end
Start Hole MD Measured depth of the well at the beginning of the event. This value is read-only, and cannot be edited.
End Hole MD Measured depth of the well at the end of the event. This value is read-only, and cannot be edited.
Start Hole TVD TVD of the well at the beginning of the event. This value is read-only, and cannot be edited.
End Hole TVD

TVD of the well at the end of the event. This value is read-only, and cannot be edited.
Start Bit MD Measured depth of the bit at the beginning of the event. This value is read-only, and cannot be edited.
End Bit MD Expected measured depth at the end of the event. Only available for actual risks. This value is read-only, and cannot be edited.
Subject

Title of the event
Description

Brief description of the event
