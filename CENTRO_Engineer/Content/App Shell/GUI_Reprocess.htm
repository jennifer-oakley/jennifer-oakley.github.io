About reprocessing data

Reprocessing data allows you to process historical data as if CENTRO® Engineer had acquired it in real-time. It also allows you to re-calculate real-time data, in the event that incorrect parameters were used in the initial well configuration.

CENTRO® Engineer reprocesses data by populating a new well with data from a file or a WITSML log. Before you begin reprocessing the data, you can update the surveys, well intervals, work strings, fluids, WITSML channels, well parameters, and other parts of the well configuration. If you are reprocessing data from a WITSML log, the WITSML channel and well parameters must be configured before you can begin reprocessing. See "WITSML channels" and "Parameters".

Depending on the speed of your computer and the amount of data, reprocessing can take anywhere from a few minutes to several hours to complete. To reduce the time required for reprocessing, you can change the options selected in the Reprocessing Setup window. See "Field definitions".

Warning

You can only start reprocessing data once per well. If you stop reprocessing a well, you will not be able to restart reprocessing.

Reprocessing data from a file

Well data can be reprocessed from a CSV or LAS file. LAS files must be in the LAS 2.0 format. For CSV files, refer to the guidelines below.

  • The first column must be a date/time index.
  • The name of the first column must be Time.
  • The date/time index should use one of the formats listed below. Hours should always use a two-digit, 24-hour format.
    • YYYY-MM-DD HH:MM:SS
    • MM-DD-YYYY HH:MM:SS
  • Each time stamp in the date/time index must be unique.
  • The time stamps in the date/time index must be in ascending order, starting from the oldest time stamp and ending with the newest time stamp.
  • All curve data should appear after the date/time index.
  • The names of columns with curve data should be in the following format: CurveName (Unit), using the standard abbreviation for the unit. For best results, use the abbreviations that appear on the Setup > Units page. See "List of unit types".
Warning

You should always check the formatting of your data prior to reprocessing. If the data is not formatted according to the guidelines listed above, the reprocessing results may be unreliable.

To reprocess data from a CSV or LAS file:

  1. Go to the Projects > All Projects > Projects page.
  2. Create a new well or copy an existing well. See "Creating a new well".
  3. Select the Mark For Reprocessing option, then click the Save Changes button ().
  4. Update the well configuration as needed, including surveys, well intervals, drillstrings, and fluids. See "Setup screen".
  5. When you are ready to start reprocessing data, click on the clock and press the Start button ()
  6. In the Reprocessing Setup window that opens, go to the Populate Data field and select the File option.
  7. In the Populate Data File field, select the file that contains your data.
  8. In the Mappings table, go to the Source Category field and select the source of the data you want to use.
    • To use data from the CSV or LAS file, select the Data Source option.
    • To use a calculated value, select the Calculations option.
  9. In the Source Parameter field, select a data source and mapping for each parameter. See "Parameters".
  10. In the Data Runs table, select the start time, end time, well interval, drillstring, and fluid for each bit run.
  11. In the Options field, select the calculations you want to include in the reprocessing. See "Field definitions".
  12. Click the Start Reprocessing button () to begin reprocessing data.

CENTRO® Engineer will begin reprocessing the data. The completion percentage of the reprocessing will appear in the status banner at the top of the screen. When reprocessing is finished, the reprocessed data will appear on the Dashboards screen.

If you need to stop reprocessing, click on the click and press the Stop button (). CENTRO® Engineer will not process any additional data. However, you will not lose the data that has been processed so far. The reprocessed well will open on the Dashboards screen with the data that had been processed prior to canceling.

Reprocessing data from a WITSML log

To reprocess data from a WITSML server:

  1. Go to the Projects > All Projects > Projects page.
  2. Create a new well or copy an existing well. See "Creating a new well".
  3. Select the Mark For Reprocessing option, then click the Save Changes button ().
  4. Update the well configuration as needed, including surveys, well intervals, drillstrings, and fluids. See "Setup screen".
  5. Configure the WITSML channel and map the well parameters. See "WITSML channels" and "Parameters".
  6. When you are ready to start reprocessing data, click on the clock and press the Start button ()
  7. In the Reprocessing Setup window that opens, go to the Populate Data field and select the WITSML option.
  8. In the Data Runs table, select the start time, end time, well interval, drillstring, and fluid for each bit run.
  9. In the Options field, select the calculations you want to include in the reprocessing. See "Field definitions".
  10. Click the Start Reprocessing button () to begin reprocessing data.

CENTRO® Engineer will begin reprocessing the data. The completion percentage of the reprocessing will appear in the status banner at the top of the screen. When reprocessing is finished, the reprocessed data will appear on the Dashboards screen.

If you need to stop reprocessing, click on the click and press the Stop button (). CENTRO® Engineer will not process any additional data. However, you will not lose the data that has been processed so far. The reprocessed well will open on the Dashboards screen with the data that had been processed prior to canceling.

Field definitions

The table below defines the fields in the Reprocessing Setup window. When the WITSML option is selected in the Data Source field, the values in the Mappings table are read-only. To edit the mappings for well parameters, go to the Setup > Mappings > Parameters tab. See "Parameters".

Field definitions for the Reprocessing Setup window
Section Field Name Definition
-- Data Source

Type of data that will be reprocessed.

  • File – Reprocesses data from a CSV or LAS file.
  • WITSML – Reprocesses data from a WITSML log.
-- File Path Location of the file that will be processed. Only available when the File option is selected in the Data Source field.
Mappings Target Parameter Name of the well parameter
Source Category

Source of the data for the parameter.

  • Data Source – Uses the data source selected in the Data Source field.
  • Calculations — Uses the results from a calculation on the Setup > OneClick™ Settings > Calculations tab.
Source Parameter Mnemonic for the parameter in the data source
Unit Measurement unit used for the mnemonic in the data source
Mapping Action Method for handling null and invalid values. See "Choosing a mapping action".
Value Override or default value for the parameter. See "Choosing a mapping action".
Unit Measurement unit used for the parameter in the current unit system
Data Runs # Run number
Start Date and time that the run starts
End Data and time that the run ends
TZ Offset Time zone offset for the data
Well Interval Well interval for the run
Drill String Drillstring used during the run
Fluid Fluid used during the run
-- Options

Calculations that will be performed during reprocessing

  • Rig States – Includes calculations for rig activities and KPIs.
  • Hydraulics – Includes hydraulics model calculations and ECD management calculations.
  • T & D – Includes torque and drag model calculations, such as calibrated friction factors, derived loads on bit, and friction factor sensitivity.
  • OneClick Calculations – Includes OneClick™ calculations from the Setup > OneClick™ Settings page.

Using the Reprocessing Setup window

The toolbar at the bottom of the Reprocessing Setup window is used to update bit runs and start reprocessing

Toolbar buttons in the Reprocessing Setup window
Button Function
Discards all changes and closes the Reprocessing Setup window
Starts reprocessing the well, using the current settings
Adds a new bit run below the selected run
Adds a new bit run above the selected run
Deletes the selected bit run
