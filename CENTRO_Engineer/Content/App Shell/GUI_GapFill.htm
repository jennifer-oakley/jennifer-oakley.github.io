About filling data gaps

When CENTRO® Engineer loses connection with the WITSML server, it can download the missing data when it reconnects to the WITSML server.

There are two primary steps required to fill gaps due to missing data:

  • Choose a data source type. When you create the well, the data source must be set to the WITSML option. This option cannot be changed after the well is created. See "Creating a new well".
  • Select a primary WITSML log. When you create the initial WITSML channel, you'll need to select the WITSML log that will be used to fill gaps due to missing data. The primary log should be the log that contains the drilling parameter data for the well. See "WITSML channels".

If CENTRO® Engineer stops receiving data from the primary WITSML log, CENTRO® Engineer will automatically retry the connection every two minutes. When the connection to the WITSML server is restored, CENTRO® Engineer will automatically resume data acquisition and download the missing data. If you manually stop data acquisition, you can choose the settings for downloading the missing data when you resume data acquisition. See "Filling a data gap".

Missing data will only be downloaded from the primary log selected for the first WITSML channel listed on the Setup > DAQ > WITSML page. Data from depth-based logs and additional time-based logs will not be downloaded.

Note

If CENTRO® Engineer does not automatically reconnect to the WITSML server, you may need to restart the WITSML channel on the Alarms > DAQ Status > WITSML page. See "WITSML alarms". If restarting the WITSML channel doesn't restore the connection, go to the Setup > DAQ > WITSML page. Select the first WITSML channel and click the Reload Logs button ().

Filling a data gap

To resume data acquisition and download missing data:

  1. Click on the clock and press the Start button ().
  2. In the Gap Filling Options window, go to the Method field and select the method you want to use to fill the data gap. See "Field definitions".
  3. If you selected the By Hour option, enter the number of hours of data you want to download.
  4. If you selected the By Time option, select the starting point for the download.
  5. In the table, select the start time and fluid for each bit run that occurred during the data gap. To add a new row, go to the first blank row and double-click the Starting At field.
  6. In the Options field, select the calculations you want to include when filling the data gap. See "Field definitions".
  7. Click the Start Gap Filling button ().

CENTRO® Engineer will begin downloading the missing data from the primary log selected for the first WITSML channel listed on the Setup > DAQ > WITSML page. The progress of the download will be shown in the Notifications banner at the top of the screen. If you are downloading more than 24 hours of missing data, the data will be downloaded in 24-hour chunks. When the download completes, real-time data acquisition will automatically resume.

Field definitions

The table below defines the fields in the Gap Filling Options window.

Field definitions for the Gap Filling Options window
Field Name Definition
Method

Method that CENTRO® Engineer will use to fill the data gap.

  • By Hour – Downloads data for a specified number of hours. The starting point for the download will be the current time, minus the number entered in the Number of Hours field.
  • By Time – Downloads data for a specified time frame. The starting point for the download will be the data and time selected in the Starting From (Rig Time) field, and the ending point will be the current time.
  • Everything – Downloads all missing data. The starting point for the download will be the data and time shown in the Last Data Point field.
  • None – Missing data will not be downloaded.
Number of Hours Hours of missing data to download. The starting point for the download will be the current time, minus the number entered. This field is only visible when the By Hour option is selection in the Method field.
Starting From (Rig Time) Starting point for downloading missing data. This field is only visible when the By Time option is selection in the Method field.
Last Data Point Time stamp of the last data point received for the primary log. This field is read-only and cannot be edited.
Fluid Drilling fluid that was used during the data gap. The selected fluid will be used for hydraulics calculations and other real-time calculations.
Bit Runs

Table used to define the bit runs that occurred during the data gap.

  • Starting At – Date and time that the run starts. The start time of the first run is the last data point received, and cannot be edited. If there are multiple runs, the end time of each run will be the start time of the next run.
  • Fluid – Drilling fluid that was used during the data gap. The selected fluid will be used for hydraulics calculations and other real-time calculations.
Options

Calculations that will be performed when filling the data gap.

  • Rig States – Includes calculations for rig activities and KPIs.
  • Hydraulics – Includes hydraulics model calculations and ECD management calculations.
  • T & D – Includes torque and drag model calculations, such as calibrated friction factors, derived loads on bit, and friction factor sensitivity.
  • OneClick Calculations – Includes OneClick™ calculations from the Setup > OneClick™ Settings > Real Time page.
