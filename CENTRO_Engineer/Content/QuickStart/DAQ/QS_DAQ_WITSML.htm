About WITSML data

This video tutorial will teach you how to configure CENTRO® Engineer to receive real-time data from a WITSML log. (2:07)

Warning

You must be signed into Microsoft with your Weatherford log-in to view this video. If the video does not automatically load, click the Sign In link to sign into your account. If the sign-in window does not appear, make sure that pop-ups are not blocked.

WITSML wellsite information transfer standard markup language is the petroleum industry's standard for sharing drilling and completions data. Using WITSML allows you to easily send and receive real-time data, without worrying about incompatible data formats and software.

The Setup > DAQ > WITSML page is used to configure CENTRO® Engineer to send and receive WITSML data, such as time-based and depth-based log data, survey data, and wellbore geometry. There are two basic steps for receiving WITSML data:

  • Add a WITSML channel. To connect to a WITSML server, you'll need to know the address of the WITSML server and the log-in information. See "Adding WITSML channels".
  • Choose log mnemonics. CENTRO® Engineer will only receive data for the mnemonics selected for each time-based log. See "Receiving log data".

On the WITSML Out page, you can configure CENTRO® Engineer to send WITSML data and calculated values from CENTRO® Engineer to the WITSML server. See "WITSML Out".

Warning

If you delete a WITSML channel, any associated raw data curves used in a dashboard on the Dashboards screen will also be deleted. However, the curves will be retained if a new channel is created with matching mnemonics.

Adding WITSML channels

To add a WITSML channel:

  1. Go to the Setup > DAQ > WITSML page.
  2. Click the Add button on the right side of the page.
  3. In the Channel field, enter a name for the channel.
  4. In the URL field, enter the full address of the WITSML server.
  5. Enter your user name and password, then click the Connect button ().
  6. After CENTRO® Engineer connects to the channel, select the well name and wellbore.
  7. Select the log data you want to receive. See "Receiving log data".
  8. In the Primary TimeLog Name field, select the WITSML log that contains the drilling parameter data.
  9. Click the Save Changes button ().

To start acquiring real-time data, click on the clock and press the Start button (). See "Real-time and playback controls". CENTRO® Engineer will begin receiving data from the WITSML server. To view the incoming WITSML data, click on the Guides utility (), then click on Indicators View and select the DAQ option. See "Data acquisition indicators".

When you begin acquiring data, the clock will update to show the current rig time. The rig time is based on the time zone selected for the primary WITSML log. If data acquisition is paused or the connection to the WITSML channel is lost, the clock will show the time that data was last received from the primary log. When the connection is reestablished, you can download the missing data from the primary log. See "Gap fill".

Tip

If additional WITSML objects are added to the server after you configure the channel, click the Reload Logs button () to refresh the list of WITSML objects for the channel.

Receiving log data

To receive data from a WITSML log:

  1. Go to the Setup > DAQ > WITSML page.
  2. Select the WITSML channel that you want to update.
  3. In the bottom portion of the screen, select the tab for the WITSML log that contains the data you want to use.
  4. Go to the list of mnemonics and select the mnemonics that you want CENTRO® Engineer to receive.
  5. For each mnemonic selected, update the Title, Unit Type, and Unit fields as needed.
  6. Click the Save Changes button ().

The selected mnemonics will be added to the CENTRO® Engineer database. The name entered in the Title field will appear in the mappings settings and the WITSML window. To view the incoming WITSML data, click on the Guides utility (), then click on Indicators View and select the DAQ option.

Note

Hookload and WOB data with "lb" and "kip" units will be associated with force units, instead of mass units. This change only applies when the following mnemonics are used: HOOKLOAD, HOOKLOAD_W, HOOK, HKLA, HKLX, HOOKLOAD_MAX, HOOKLOAD_AVG, WOB, WOB_W, WOBA, WOBX, BIT_WEIGHT.

Tip

If additional WITSML objects are added to the server after you configure the channel, click the Reload Logs button () to refresh the list of WITSML objects for the channel.

After you configure the WITSML channel, the next step is to configure any incoming OPC UA data. See "Setting up OPC UA channels".
