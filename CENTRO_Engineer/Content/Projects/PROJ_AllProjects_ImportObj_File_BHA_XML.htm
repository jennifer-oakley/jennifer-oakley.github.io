About WITSML work string imports

On the Import Objects > File tab, you can import BHAs and work strings from a WITSML XML file, created from a tubular object on a WITSML server. The contents of the imported file must adhere to the WITSML specification for tubular objects. The WITSML XML file must contain the following information:

  • Type – The type of work string component. This value must match one of the WITSML tool types defined for the WITSML schema.
  • Description – A description of the component.
  • OD – Nominal OD of the component. This column can only contain numeric values.
  • Component Length – Length of the component. This column can only contain numeric values.

CENTRO® Engineer will import all tools into a separate catalog. Drill bits, drill collars, drill pipe, and HWDP will be imported as the appropriate tool type. Drilling motors will be imported as custom motors. All other types of components will be imported as custom tools. After you import a work string, you should verify that the work string components were imported correctly on the Setup > Work String page. See "Work strings".

CENTRO® Engineer will attempt to automatically detect the tool type and connection types. However, you should verify these values in the preview table before saving the BHA. If any of the values need to be modified, double-click on the component you want to edit to open the editing panel. See "Importing work strings from WITSML".

Note

The name defined for the custom data section in the XML file will be used to determine the name of the work string component. If the file doesn't have a custom data section, then the description defined for the tubular component section will be used.

Validation errors

If there are any validation errors in your data, the cells containing the errors will be outlined in red. Specific validation errors will also appear in the Validation Error column in the preview table.

Errors in the columns for the Type, Description, OD, and ID fields must be fixed in the XML file before you can import the data. If there are errors in the columns for any of the optional fields, you will still be able to import the data. However, the invalid values will be ignored and will not be imported. For more information about the validations used for each field, refer to the Field Definitions section. See "Field definitions".

Importing work strings from WITSML

When you import a work string, you can import it with or without an explicitly-defined BHA. To import a work string with an explicitly-defined BHA, enter a value in the # of BHA Components field. If no value is entered, the work string will be imported without an explicitly-defined BHA.

  • In a work string with an explicitly-defined BHA, the BHA and drillstring components will be grouped into separate tables.
  • In a work string without an explicitly-defined BHA, all BHA and drillstring components will appear in one table.

To import work strings from a WITSML XML file:

  1. Go to the File tab on the All Projects > Import Objects page.
  2. In the Utility field, select the Work String Import (WITSML) option.
  3. In the Decimal Separator field, verify the character used to separate decimal portion of a number from the integer.
  4. In the Work String Name field, enter a name for the work string.
  5. To create a work string with an explicitly-defined BHA, go to the # of BHA Components field and enter the number of components that make up that BHA portion of the work string.
  6. In the Path field, click the Browse button () and navigate to the file you want to import. A preview of the data will appear in the table at the bottom of the screen.
  7. In the preview table, double-click on the component description to open the editing panel and edit the imported values as needed.
  8. Review the remaining data shown in the preview table, and verify that there are no validation errors.
  9. Click the Save Changes button () to import the data.

A new work string will be created on the Setup > Work String page, using the name entered in the Work String Name field. All of the tools in the work string will be imported in a separate catalog.

Field definitions

The table below defines the fields on the All Projects > Import Objects > File tab, when the Work String Import (WITSML) option is selected.

Fields on the Import Objects > File tab for WITSML work string imports
Column Type Definition
Utility

Type of data in the import file

  • Work String Import (CSV) – File contains BHA and work string data, in a CSV file format. See "Work string imports from CSV".
  • Work String Import (XLSX) – File contains BHA and work string data, in an XSLX file exported from AccuView. See "Work string imports from XLSX".
  • Work String Import (WITSML) - File contains BHA and work string data, in a WITSML XML file format. See "Work string imports from WITSML".
  • Actual Risk Import – File contains actual risks and events data. See "Actual risk imports".
  • Planned Risk Import – File contains planned risks and events data. See "Planned risk imports".
Decimal Separator

Character used to separate the decimal portion of a number from the integer portion, usually either a period or a comma
Work String Name

Name for the work string that you want to appear on the Setup > Work String page
# of BHA Components

Number of components that make up that BHA portion of the work string. If there is no value entered in this field, the work string will be imported without an explicitly-defined BHA.
Path

File path for the import file

Data Type Definition
