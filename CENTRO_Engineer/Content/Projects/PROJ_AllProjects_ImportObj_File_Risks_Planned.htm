About planned risk imports

On the Import Objects > File tab, you can import planned risks, lessons learned, best practices, and other events from a CSV file. The CSV file must contain at least two columns:

  • Risk Type – The type of risk event. The values in this column must match the values listed for the Event Type field in a risk record on the Reports > Risks page.
  • Risk Category – A category to describe the risk event. The values in this column must match the values for the Category field in a risk record on the Reports > Risks page.

The CSV file may also contain columns for start time, well MD, subject of the events, and descriptions of the events. However, this information is optional and is not required. See "Field definitions".

The first table on the File tab lists the available fields, with the units and column positions assigned to each field. The Position column indicates which column in your data will be used for each field. When you select the file you want to import, a preview of the data will appear in the table at the bottom of the screen. The columns in your data will be numbered in order, from left to right.

CENTRO® Engineer will automatically attempt to match the columns in the CSV file to the equivalent field for planned risks. However, you can also manually edit the column positions. If the Position value is blank, no data will be imported for that field.

Validation errors

If there are any validation errors in your data, the cells containing the errors will be outlined in red. Specific validation errors will also appear in the Validation Error column in the preview table.

Errors in the columns for the Risk Type and Risk Category fields must be fixed in the CSV file before you can import the data. If there are errors in the columns for any of the optional fields, you will still be able to import the data. However, the invalid values will be ignored and will not be imported. For more information about the validations used for each field, refer to the Field Definitions section. See "Field definitions".

Importing planned risks

To import planned risks and events from a CSV file:

  1. Go to the File tab on the All Projects > Import Objects page.
  2. In the Utility field, select the Planned Risk Import option.
  3. In the table, verify the units and column positions assigned to each field and update the values as needed.
  4. In the Decimal Separator field, verify the character used to separate decimal portion of a number from the integer.
  5. In the Date Format field, enter the format used for dates, e.g. MM-DD-YYYY.
  6. In the Path field, click the Browse button () and navigate to the file you want to import. A preview of the data will appear in the table at the bottom of the screen.
  7. Review the data shown in the preview table, and verify that there are no validation errors.
  8. Click the Save Changes button () to import the data.

The risks and events will be added to the risks on the Reports > Risks > Planned tab. If you have a WITSML channel configured, a new entry will be added to the WITSML log for the selected event type. If a WITSML log for the selected event type doesn't exist, a new log will be created. See "WITSML channels".

Field definitions

The CSV file may contain columns for the following information:

Required and optional columns for planned risk imports
Column Type Definition
Risk Type The type of risk event. This column is required for risk imports. The values in this column must match the values listed for the Event Type field in a risk on the Reports > Risks > Planned tab. See "Risks".
Risk Category A category to describe the risk event. This column is required for risk imports. The values in this column must match the values for the Category field in a risk on the Reports > Risks > Planned tab. See "Risks".
Risk Subcategory A sub-category to describe the risk event. The values in this column must match the values for the Sub Category field in a risk on the Reports > Risks > Planned tab. See "Risks".
Subject Title of the event
Description

Brief description of the event
Start Hole MD Measured depth of the well at the beginning of the event. This column can only contain numeric values.
End Hole MD Measured depth of the well at the end of the event. This column can only contain numeric values.
Start Time Time that the event began or is expected to begin. The values in this column can use any standard date/time format.
End Time Time that the event ended or is expected to end. The values in this column can use any standard date/time format.
