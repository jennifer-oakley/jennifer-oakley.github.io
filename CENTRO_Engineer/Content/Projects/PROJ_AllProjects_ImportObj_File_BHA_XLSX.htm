About work string imports from XSLX

On the Import Objects > File tab, you can import BHAs and work strings from an XSLX file. The XSLX file must contain the following columns:

  • Description – A description of the component.
  • OD – Nominal OD of the component. This column can only contain numeric values.
  • Length – Length of the component. The value for length can represent either the length of each individual component or the total length of the consecutive components. This column can only contain numeric values.
Note

Currently, only BHAs exported from AccuView can be imported into CENTRO® Engineer.

The first table on the File tab lists the available fields, with the units and column positions assigned to each field. The Position column indicates which column in your data will be used for each field. When you select the file you want to import, a preview of the data will appear in the table at the bottom of the screen. The columns in your data will be numbered in order, from left to right.

CENTRO® Engineer will automatically attempt to match the columns in the XSLX file to the equivalent field for work string components. However, you can also manually edit the column positions. If the Position value is blank, no data will be imported for that field.

CENTRO® Engineer will import all tools into a separate catalog. Drill bits, drill collars, drill pipe, and HWDP will be imported as the appropriate tool type. All other types of components will be imported as custom tools. After you import a work string, you should verify that the work string components were imported correctly on the Setup > Work String page. See "Work strings".

Warning

It is important to select the correct mud motor on the Setup > Work String page. Because mud motors are initially imported as custom tools, the hydraulics model calculations will not be accurate until the correct mud motor is selected. See "Drilling motors".

Validation errors

If there are any validation errors in your data, the cells containing the errors will be outlined in red. Specific validation errors will also appear in the Validation Error column in the preview table.

Errors in the columns for the fields must be fixed in the file before you can import the data. If there are errors in the columns for any of the optional fields, you will still be able to import the data. However, the invalid values will be ignored and will not be imported. For more information about the validations used for each field, refer to the Field Definitions section. See "Field definitions".

Importing work strings from an XSLX file

When you import a work string, you can import it with or without an explicitly-defined BHA. To import a work string with an explicitly-defined BHA, enter a value in the # of BHA Components field. If no value is entered, the work string will be imported without an explicitly-defined BHA.

  • In a work string with an explicitly-defined BHA, the BHA and drillstring components will be grouped into separate tables.
  • In a work string without an explicitly-defined BHA, all BHA and drillstring components will appear in one table.

To import work strings from a XSLX file:

  1. Go to the File tab on the All Projects > Import Objects page.
  2. In the Utility field, select the Work String Import (XSLX) option.
  3. In the Decimal Separator field, verify the character used to separate decimal portion of a number from the integer.
  4. In the Work String Name field, enter a name for the work string.
  5. In the "Length" Represents field, select what the value for length represents in your data.
    • If the length value is for the length of each component, select the Component Length option.
    • If the length value is for the total length of the consecutive components, select the Total Length option.
  6. To create a work string with an explicitly-defined BHA, go to the # of BHA Components field and enter the number of components that make up that BHA portion of the work string.
  7. In the Path field, click the Browse button () and navigate to the file you want to import. A preview of the data will appear in the table at the bottom of the screen.
  8. In the first table, above the list of components, verify the units and column positions assigned to each field and update the values as needed.
  9. Review the data shown in the preview table, and verify that there are no validation errors.
  10. Click the Save Changes button () to import the data.

A new work string will be created on the Setup > Work String page, using the name entered in the Work String Name field. All of the tools in the work string will be imported in a separate catalog.

Field definitions

The table below defines the fields on the All Projects > Import Objects > File tab, when the Work String Import (XSLX) option is selected.

Fields on the Import Objects > File tab for XLSX work string imports
Column Type Definition
Utility

Type of data in the import file

  • Work String Import (CSV) – File contains BHA and work string data, in a CSV file format. See "Work string imports from CSV".
  • Work String Import (XLSX) – File contains BHA and work string data, in an XSLX file exported from AccuView. See "Work string imports from XLSX".
  • Work String Import (WITSML) - File contains BHA and work string data, in a WITSML XML file format. See "Work string imports from WITSML".
  • Actual Risk Import – File contains actual risks and events data. See "Actual risk imports".
  • Planned Risk Import – File contains planned risks and events data. See "Planned risk imports".
Source

File format of the import file. Currently, only XSLX files are supported.
Decimal Separator

Character used to separate the decimal portion of a number from the integer portion, usually either a period or a comma
Work String Name

Name for the work string that you want to appear on the Setup > Work String page
"Length" Represents

Indicates what the value for length represents in your data.

  • Component Length – The Length column represents the length of each individual component.
  • Total Length – The Length column represents the total length of consecutive components.
# of BHA Components

Number of components that make up that BHA portion of the work string. If there is no value entered in this field, the work string will be imported without an explicitly-defined BHA.
Path

File path for the import file
Has Header Row

Indicates that the first row in your data is a header row
