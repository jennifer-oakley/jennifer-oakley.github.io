About bit runs

The Bit Runs window allows you to create and manage bit runs. Defining the bit runs for a job gives you another way to filter historical data on the Home screen. To open the Bit Runs window, click the Indicators View button () in the Utilities toolbar and select the Bit Runs option.

Victus™ Simulation automatically detects when a bit run begins and ends. The current run will end when the bit moves above the first formation. A new run will begin when the bit descends below the first formation.

The start depth of the first run is the depth of the active well interval when you start acquiring data. If you edit the interval depth after you start receiving data, the start depth of the first run won't be automatically updated. However, you can manually update the run.

Managing bit runs

If the bit runs are not detected correctly, you can also manually manage bit runs. However, bit runs are not allowed to overlap. For this reason, you may need to edit bit runs in backwards order, starting with the most recent run.

There are several way to manage bit runs:

  • Change the start or end time. When you enter a new date and time, Victus™ Simulation will query the database to find the closest matching well depth.
  • Change the start or end depth. When you enter a new well depth, Victus™ Simulation will query the database to find the closest matching date and time.
  • Add bit runs. When you insert a run between two existing runs, the start and end of the new run will be automatically calculated based on the start and end of the two existing runs. When you add a run after the current run, the current run will be ended.
  • Delete bit runs. When you delete a run, the start and end of the remaining runs won't be affected. This means that you may need to edit the start or end of the remaining runs.

Using the Bit Runs window

The toolbar at the bottom of the Bit Runs window is used to add and remove runs.
