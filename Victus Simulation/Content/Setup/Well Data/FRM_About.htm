About formation data

The Setup > Well Data > Formations tab contains the geological and geophysical characteristics of the formation layers in the wellbore. These properties include porosity, permeability, pore pressure, and fracture pressure. Formation layers are indexed by measured depth along the length of the wellbore, from the surface to the bit. The TVD of each layer is automatically calculated, based on the wellpath data for the well. See "Wellpath".

Victus™ Simulation allows you to store multiple sets of formation data, based on the source of the data. The Source Type menu at the bottom of the screen is used to switch between the different sets of formation data. There are five source types:

  • None – No information is available about the source of the data.
  • Interpreted – The data was acquired by analyzing the available samples.
  • Core – The data was acquired through core testing.
  • Cuttings – The data was acquired through cuttings tests.
  • Unknown – The source of the data is unknown.

Interpreted formation data is used for model calculations in Victus™ Simulation. The other source types are only used for record keeping, and are not used in any calculations.

For a description of the formation types available in Victus™ Simulation, refer to the Catalogs > Lithologies tab. See "Lithologies catalog".

Importing formation data from a file

Formation data can be imported into Victus™ Simulation from a CSV or LAS file. Imported data must include depth data, indexed by MD or TVD.

To import formation data from a CSV or LAS file:

  1. Go to the Formations tab on the Setup > Well Data page.
  2. Go to the Source Type menu at the bottom of the screen, and select the source type you want to import data into.
  3. Click the Import Formations button () at the bottom of the screen.
  4. In the menu that appears, select the File option. Click Yes on the warning message that appears. This menu will only appear if you are connected to a WITSML channel with mud log data.
  5. Navigate to the file you want to import and click Open. A list of import options will appear in Victus™ Simulation.
  6. Select the elevation reference used in your data.
    • If the depths are relative to mean sea level, select the Depth is referenced to Mean Sea Level option.
    • If the depths are not relative mean sea level, clear the Depth is referenced to Mean Sea Level option. In the Elevation Offset field that appears, enter the distance between mean sea level and the reference point.
  7. In the first column, go to the first row and select the type of depth index.
  8. In the middle column of each row, select the name of the column that contains the data.
  9. In the last column of each row, select the measurement unit used in your data.
  10. Click the OK button () to begin the import.

The data will be imported into Victus™ Simulation, using default values for lithology type, formation name, porosity, and permeability.

Note

When you import formation data, the existing formation data is not overwritten with the new formations. The imported formations are added to the list of existing formations. If you do not want to keep the existing formation data, manually delete the existing formation data before the import.

Importing formation data from WITSML

Data for the formation tops can also be imported from WITSML. However, this data is limited to the lithology type and the measured depth of the formation top. The well on the WITSML server must have mud logs with the formation tops. Before you can import mud logs from WITSML, you'll need to create a WITSML channel for the WITSML server that is sending the mud log data. See "WITSML channels".

To import mud logs from WITSML:

  1. Go to the Formations tab on the Setup > Well Data page.
  2. Click the Import Formations button () at the bottom of the screen.
  3. In the menu that appears, select the WITSML channel and mud log that contains the data you want to import.
  4. A list of import options will appear. Select the Auto Import option, then select the frequency with which you want the mud log data to be updated.
  5. Click the OK button () to begin the import.

The mud log data from the WITSML server will be imported into Victus™ Simulation, under the source type specified in the mud log. Victus™ Simulation will periodically query the WITSML server and update the formation data.

Adding formation layers

To describe the wellbore formation, you'll need to update the properties of the first formation layer, then add formation layers to describe the rest of the wellbore formation.See "Adding formation layers". You can also import some formation data, such as pore pressure and fracture pressure gradients, from a CSV or LAS file. See "Importing formation data from a file".

The first layer of the formation is created for you, using default properties. The measured depth of this layer is calculated automatically, based on the rig elevations entered on the Surface Equipment page. See "Rig Elevations".

To add a formation layer:

  1. Go to the Formations tab on the Setup > Well Data page.
  2. Go to the Source Type menu at the bottom of the screen, and select the source of the formation data. See "Field definitions".
  3. Select a row in the table.
    • To add a layer above that row, click the Insert Above button ().
    • To add a layer below that row, click the Insert Below button ().
  4. In the Lithology column, select the type of formation. See "Lithologies catalog".
  5. Enter the name of the formation layer in the Name column.
  6. Enter the layer's porosity as a decimal, then enter the permeability of the layer.
  7. Enter the MD at the top of the formation layer. TVD will be automatically calculated from the survey data. See "Wellpath".
  8. Enter the pressure gradients for the formation layer.
  9. Enter the horizontal stresses for the layer.
  10. Add additional formation layers as needed.
  11. When you have finished entering formation data, click the Save Changes button () at the bottom of the screen.

The formation data will be stored in Victus™ Simulation's database. A preview of the formation layers in the well will appear in the Well View pane. If the rig is located offshore, a layer of water will appear at the top of the Well View pane. This layer represents the water between the kelly bushing and the sea floor.

Customizing the formation composition

If a formation layer is composed of multiple lithologies, such as sandstone and shale, you can customize the composition of the formation layer to reflect the concentration of each lithology.

To customize the formation composition:

  1. Go to the Formations tab on the Setup > Well Data page.
  2. Select the formation layer you want to customize, then click the Edit button () at the bottom of the screen.
  3. In the panel that appears, add rows for each different type of lithology in the formation layer.
  4. Select the lithology for each row, then go to the Percentage field and enter the lithology concentration as a percentage.
  5. When you are finished making changes, click the OK button () to close the customization panel.
  6. Click the Save Changes button () at the bottom of the screen.

The formation data for the well will be updated with the new lithology information. In the Well View pane, the preview of the formation will be updated to reflect the concentration of each lithology in the formation layer.

Field definitions

The table below defines the columns in the Formations table.

Field definitions for formation properties
Column Name Description
Lithology Composition of the formation layer. For a description of each lithology available, refer to the Lithologies tab on the Setup > Catalogs page. See "Lithologies catalog".
Name Name used to identify the formation layer
Porosity The porosity of the layer, expressed as a decimal.
Permeability The permeability of the layer
MD MD at the top of the formation layer
TVD TVD at the top of the formation layer. TVD is automatically calculated from MD, and can't be manually entered.
PPG Pore pressure gradient
WBSG Wellbore stability gradient
FIT Result from a formation integrity test
FPG Fracture pressure gradient
LOT Result from a leak-off test
Min HS Minimum horizontal stress
Max HS Maximum horizontal stress
D Max HS Direction of the maximum horizontal stress
UCS Uniaxial compressive strength. UCS is the maximum axial compressive stress that a right-cylindrical sample of material can withstand before failing, expressed in pounds per square inch.
MSE Mechanical specific energy. MSE is the energy required to remove a unit volume of rock, expressed in pounds per square inch.

Using the Formations tab

To export formation data to a CSV file, press the Ctrl + S keys. The Source Type menu at the bottom of the screen is used to switch between different sets of formation data. The toolbar at the bottom of the screen on the Formations tab is used to add and remove formation layers.
