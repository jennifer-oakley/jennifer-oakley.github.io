About modbus data

Modbus is a serial communication protocol that is commonly used for sending data from equipment such as downhole deployment valves, pressure release valves, and other PLC programmable logic controllers. Modbus data can be sent through a serial port on the UPI, as well as across the network through a TCP/IP connection.

The Setup > DAQ > Modbus tab is used to configure Victus™ Simulation to send and receive modbus data from the UPI. There are three basic steps for sending and receiving modbus data:

  • Add a modbus channel. To connect to a modbus device, you'll need to know the IP address or port name for the device. See "Adding modbus channels".
  • Add inbound data tags. You'll need to add a data tag for each inbound parameter received from the modbus device. See "Adding inbound data tags".
  • Add outbound data tags. If you want to send data to the modbus device, you'll also need to add data tags for each outbound parameter sent to the modbus device. See "Adding outbound data tags".
Warning

If you delete a modbus channel, any associated raw data curves used in a dashboard on the Home screen will also be deleted.

Adding modbus channels

To add a modbus channel:

  1. Go to the Setup > DAQ > Modbus tab.
  2. Click the Add button (). A new tab will appear in the right toolbar.
  3. In the Channel Name field, enter a name for the modbus channel.
  4. In the Interface Type field, select the connection type for the modbus device.
  5. Enter the remaining connection settings. See "Field definitions".
  6. Add data tags for each inbound parameter received from the modbus device. See "Adding inbound data tags".
  7. Add data tags for each outbound parameter sent to the modbus device. See "Adding outbound data tags".
  8. Click the Save Changes button ().

To start acquiring real-time data, click on the clock and select the x1 option. See "Real-time controls". Victus™ Simulation will begin receiving data from the modbus device. To view the incoming modbus data, click the Indicators View button () in the Utilities toolbar and select the DAQ option. See "Modbus data". To check the status of incoming and outgoing data for the channel, go to the Alarms > Status > MODBUS tab. See "Modbus alarms".

Adding inbound data tags

To add an inbound data tag to a modbus channel:

  1. Go to the Setup > DAQ > Modbus tab.
  2. Select the modbus channel that you want to add a data tag to.
  3. Go to the Inbound tab for the channel.
  4. Click the Add Data Tag button (). A new row will appear in the lower portion of the screen.
  5. In the Name field, enter a name for the sensor on the modbus device.
  6. Select the Byte Swap and Word Swap options as needed. These settings must match the settings used on the modbus device.
  7. In the Address field, enter the data address for the parameter.
  8. In the Register Type field, select the type of table where the data is located.
  9. In the Conversion Type field, select the type of data that is being sent. See "Field definitions".
  10. In the Unit Type field, select the type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
  11. In the Unit field, select the measurement unit for the data.
  12. Click the Save Changes button ().

The data tags will be added to the Victus™ Simulation database. The name entered in the Name field will appear in the mappings settings and the DAQ indicator for the modbus channel. To view the incoming data from the log, click the Indicators View button () in the Utilities toolbar and select the DAQ option.

Adding outbound data tags

To add an outbound data tag to a modbus channel:

  1. Go to the Setup > DAQ > Modbus tab.
  2. Select the modbus channel that you want to add a data tag to.
  3. Go to the Outbound tab for the channel.
  4. Click the Add Data Tag button (). A new row will appear in the lower portion of the screen.
  5. In the new row that appears, go to the Object field and select the category for the data you want to send.
  6. In the Parameter field, select the parameter that contains the data you want to send.
  7. Select the Byte Swap and Word Swap options as needed. These settings must match the settings used on the modbus device.
  8. In the Address field, enter the data address for the parameter.
  9. In the Unit Type field, select the type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
  10. In the Unit field, select the measurement unit for the data.
  11. Click the Save Changes button ().

The new entry will be added to the Victus™ Simulation database. The outgoing data will be sent on the selected modbus channel. The data acquisition service will also be restarted so that Victus™ Simulation can connect to the modbus channel.

Field definitions

The table below defines the modbus channel settings on the Setup > DAQ > Modbus tab.

Properties of modbus channels
Field Name Definition
Channel Name of the modbus channel
Address Type

Addressing method used for parameters sent through the modbus device.

  • One-Based – Parameter addresses start numbering from 1.
  • Zero-Based – Parameter addresses start numbering from 0.
Interface Type

Type of connection used to connect the modbus device to the UPI.

  • RTU Over TCP – Device is connected to a remote terminal unit through a network connection.
  • Serial – Device is connected through a serial port.
  • TCP – Device is connected through a network port.
IP Address IP address for the modbus connection. Only required for TCP and RTU Over TCP connections.
Port Port number for the modbus connection. Only required for TCP and RTU Over TCP connections.
Slave ID ID of the slave device that is sending the data.
Baud Rate Transmission rate of the data, typically 9600 baud. Only required for serial connections.
Port Name Name of the serial port on the UPI that the modbus device is connected to. Only required for serial connections.
Parity

Type of parity checking used by the modbus device. Parity checking is used to ensure that each data string contains the correct number of bits. Only required for serial connections.

  • Even – Adds a parity bit to the beginning of each data string to ensure that it contains an even number of bits. The value of the parity bit can be 0 or 1, depending on the number of bits in the rest of the data string.
  • Mark – Adds a parity bit with a value of 1 to the beginning of each data string.
  • None – Parity checking is not enabled.
  • Odd – Adds a parity bit to the beginning of each data string to ensure that it contains an odd number of bits. The value of the parity bit can be 0 or 1, depending on the number of bits in the rest of the data string.
  • Space – Adds a parity bit with a value of 0 to the beginning of each data string.
Stop Bits

Type of stop bit used by the modbus device. Stop bits are used to indicate the end of a data string. Only required for serial connections.

  • None – Stop bits are not enabled.
  • One – One stop bit is added to the end of each data string.
  • OnePointFive – 1.5 stop bits are added to the end of each data string.
  • Two – Two stop bits are added to the end of each data string.

The table below defines the fields for the data tags in a modbus channel.

Properties of modbus data tags
Field Name Definition
Name Name of the parameter on the modbus device
Object Category for the data you want to send. Only available for outgoing data tags.
Parameter Parameter that contains the data you want to send. Only available for outgoing data tags.
Byte Swap Indicates that byte swapping is enabled on the data tag. Select this option if byte swapping is enabled on the modbus device.
Word Swap Indicates that word swapping is enabled on the data tag. Select this option if word swapping is enabled on the modbus device.
Address Data address of the parameter on the modbus device
Register Type

Type of table where the data is located.

  • Coil – Read-write table, with single bit data. Select this option for register numbers ranging from 1-9999.
  • Discrete Input – Read-only table, with single bit data. Select this option for register numbers ranging from 10001-19999.
  • Holding – Read-write table. Select this option for register numbers ranging from 40001-49999.
  • Input – Read-write table, with 16-bit data. Select this option for register numbers ranging from 30001-39999.
Conversion Type

Type of data sent for the parameter.

  • For numeric data that contains decimals, select the IEEE-754 32-bit Float option.
  • For whole numbers that fall within the range of -32,768 to 32,767, select the Signed 16-bit Integer option.
  • For whole numbers that fall within in the range of -2,147,483,648 to 2,147,483,647, select the Signed 32-bit Integer option.
  • For whole numbers that fall within the range of 0 to 65535, select the Unsigned 16-bit Integer option.
  • For whole numbers that fall within the range of 0 to 4,294,967,295, select the Unsigned 32-bit Integer option.
Unit Type Type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
Unit Measurement unit for the incoming data, e.g. inches or feet

Using the Modbus tab

The toolbar at the bottom of the screen on the Modbus tab is used to add and remove data tags for the selected modbus channel. The toolbar to the right is used to create and delete modbus channels. The tabs above the toolbar buttons are used to switch between channels.

Toolbar buttons on the Modbus tab
Button Function

Discards all changes made since the last time you saved

Saves all changes
Deletes the selected data tag from the current modbus channel
Adds a data tag to the current modbus channel. See "Adding inbound data tags" and "Adding outbound data tags".
Adds new modbus channel. See "Adding modbus channels".
Deletes the selected modbus channel
