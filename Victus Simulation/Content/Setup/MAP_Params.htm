About parameters

Well parameters such as bit depth, well depth, ROP, and hookload are used by Victus™ Simulation to create persistent curves for graphs and gauges. If you lose connection with the WITS or WITSML server, gauges and graphs that are mapped to well parameter curves will remain intact. Victus™ Simulation also uses well parameters to calculate values such as flow rates, bottomhole temperature and pressure, circulation times, and total pump strokes. The data for well parameters can be collected from calculations or external data sources.

On the Setup > Mappings > Parameters tab, you can assign the data source that Victus™ Simulation uses for each well parameter. There are five types of data sources:

  • Calculated values, using the public calculations on the Setup > OneClick™ Settings > Calculations tab. See "OneClick™ calculations".
  • WITS data, received from incoming WITS data defined on the Setup > DAQ > WITS > In tab. See "WITS data".
  • WITSML data, received from a WITSML channel defined on the Setup > DAQ > WITSML tab. See "WITSML channels".
  • Modbus data, received from a modbus channel defined on the Setup > DAQ > Modbus tab. See "Modbus data".
  • OPC-UA data, received from an OPC-UA channel defined on the Setup > DAQ > OPC UA tab. See "OPC UA data".
Note

For pump rates, Victus™ Simulation uses the simulated sensor selected in the rig configuration. If you are using an external data source for pump rates, you'll need to update the information on the Setup > Surface Equipment page. See "Specifying the pump source".

Defining well parameters

To choose a data source for the well parameters:

  1. Go to the Parameters tab on the Setup > Mappings page.
  2. Right-click on the header for the Source Category column, then select the type of data you want to use.
    • Calculations – Uses the result from a calculation.
    • Wits Data – Uses incoming WITS data.
    • WITSML channel name – Uses incoming data from the selected WITSML channel.
    • Modbus channel name – Uses incoming data from the selected modbus channel.
    • OPC UA channel name – Uses incoming data from the selected OPC UA channel.
  3. In the Source Parameter field for each parameter, select the name of the data source you want to use. If your data uses standard mnemonics, the parameter will be automatically mapped to the correct mnemonic. See "List of well parameters".
  4. In the Mapping Action field, select a mapping action for invalid and null values. See "Choosing a mapping action".
  5. When you have finished defining the well parameters, click the Save Changes button () to save your changes.

The well parameters will be updated with the values from the selected data sources. If your data uses standard mnemonics, the parameters will be automatically mapped to the correct mnemonics. See "List of well parameters". To view the current parameter values, click on the Indicators View utility () and select the Parameters option. See "Parameters indicator".

Choosing a mapping action

The option selected in the Mapping Action field tells Victus™ Simulation how to handle invalid values for well parameters. Null values and values outside of the minimum and maximum range defined on the Setup > Limits and Alarms > Parameters tab will be considered invalid, even if the alarm for that parameter is not enabled. See " Limits and Alarms".When Victus™ Simulation receives an invalid value, the selected mapping action will determine how Victus™ Simulation handles the invalid value. You can also use a mapping action to override an incoming value, even if it a valid value.

There are four options available in the Mapping Action field:

  • As Is – Ignores the validation results and accepts the invalid value as-is.
  • Default – Overrides the invalid value with the value entered in the Value field. Only invalid values will be overwritten with the default value.
  • Previous – Overrides the invalid value with the last valid value received.
  • Override – Overrides any value with the value entered in the Value field. All values for the parameter will be overwritten with the override value, even if they are valid values.
Tip

When you select the Default option or the Override option, you can use a simple expression for the override value. Simple expressions allow you to multiply an incoming value by a constant value. To enter a simple expression in the Value field, use the following format: [ConstantValue]*$value, e.g. 0.9*$value.
