About sensor configuration

Victus™ Simulation has two modes for using the measurements received from SBP and SPP sensors: single sensor measurement and multi-sensor measurement. Single sensor measurement is used when there is only one source selected for standpipe pressure or surface backpressure. Multi-sensor measurement is used when there are three or more sources selected. Using multiple sensors to measure pressure allows Victus™ Simulation to provide the most accurate pressure measurement possible.

SBP and SPP sensors are treated as separate sets of sensors. The sensor measurement mode is determined individually for both of these sensor sets. One sensor set can use single sensor mode, while the other sensor set uses multi-sensor mode. Or, both sets of sensors can use the same measurement mode. Victus™ Simulation automatically switches to the correct measurement mode, depending on the number of sensors selected for SBP and SPP on the Setup > Surface Equipment > Equipment Configuration tab.

You can also connect a flow meter or another flow measurement device to the spare sensors. Victus™ Simulation converts the flow readings into a pump rate, based on the calibrated settings for the sensor. The calculated pump rate is used to determine the total flow rate. See " Pump configuration".

Updating sensor information

On the Piping and Instrumentation tab, you'll need to specify the type of sensor connected to each source on the Equipment Configuration tab. Victus™ Simulation will only monitor the status of active sensors. If the sensor type is set to None, the instrument is considered to be inactive and all features related to the instrument will be disabled.

  1. Go to the Equipment Configuration tab on the Setup > Surface Equipment page.
  2. In the SBP Source field, select the sources for the SBP sensors.
  3. In the SPP Source field, select the sources for the SPP sensors.
  4. Go to the Piping and Instrumentation tab on the Setup > Surface Equipment page.
  5. Click the Table View button () to view the list of equipment on the rig.
  6. In the equipment list, select the row for Instrumentation.
  7. In the panel that appears, select the type of sensor connected to each source. If there is no sensor connected to one of the sources, select the None option.
  8. Click the Save Changes button () to save your changes.

The rig configuration will be saved to the Victus™ Simulation database, and any inactive sensors will be disabled throughout Victus™ Simulation. After specifying the sensor types, you'll need to calibrate each analog sensor on the Setup > Calibration > Default Sensors tab. See "Default sensors".

Using multiple SBP sensors

To use multiple sensors to measure surface backpressure:

  1. Go to the Piping and Instrumentation tab on the Setup > Surface Equipment page.
  2. Click the Table View button () to view the list of equipment on the rig.
  3. In the equipment list, select the row for Instrumentation.
  4. In the panel that appears, select the type of pressure sensor connected to each SBP source. If there is no pressure sensor connected to one of the SBP sources, select the None option.
  5. Click the Save Changes button ().
  6. Go to the Equipment Configuration tab.
  7. In the SBP Source field, select three or more sensors for surface backpressure to enable multi-sensor management.
  8. Click the Save Changes button ().

Victus™ Simulation will initialize the surface backpressure sensors and acquire the values for the initial sensor readings. See "Sensor initialization". Victus™ Simulation will only monitor the status of active sensors. If the sensor type is set to None, the instrument is considered to be inactive and all features related to the instrument will be disabled throughout Victus™ Simulation. Select three or more sensors for surface backpressure to enable multi-sensor management.

Sensor initialization

When you activate multiple pressure sensors for SBP or SPP, Victus™ Simulation will run a series of basic diagnostics to establish communication with each sensor and calculate an initial average value from the sensor readings. Each set of sensors is initialized separately, but the same process is used for both sets.

  1. Victus™ Simulation will acquire raw values from each sensor, then convert the raw values into engineering units.
  2. The readings from each sensor will be validated, based on the number of sensors selected. An algorithm will be used to determine the accuracy of the readings from each sensor.
  3. If Victus™ Simulation successfully validates all sensors in the set, the sensor readings will be used to calculate the initial average value. The initial average value is calculated using the sum of the sensor readings, divided by the number of sensors.

Sensors are not validated until the fluid pressure exerted on the sensor exceeds 20 psi. If the pressure falls below 20 psi, the sensor set will revert back to the beginning of the initialization phase.

Sensor failures

After a sensor set has been initialized, Victus™ Simulation continuously performs diagnostic checks on each pressure sensor to ensure that it is communicating with the MPD manifold and reporting valid readings. These checks are performed every 40 ms. If a sensor fails any of these checks, the sensor will be suspended for the current iteration and checked again during the next round of diagnostic checks.

The first diagnostic check is for core fault failures. This is a basic test of the communication between the MPD manifold and the pressure sensor. Digital sensors are considered to have a core fault failure if the sensor reading doesn't change for two seconds. Analog sensors are considered to have a core fault failure if the sensor reading is below 3.7 mA or above 20 mA.

The next diagnostic check is for tolerance range failures. During this test, Victus™ Simulation compares each individual sensor reading to the initial average value. If the reading is within a pre-defined tolerance range, the sensor reading will be used to calculate a new average value. If the reading is not within the tolerance range, the sensor fails the diagnostic check and is suspended for the current iteration. The reading will not be used to calculate a new average value, but the sensor will be checked again during the next iteration. The tolerance range is 5 - 15% of the current pressure. At 0 psi, the tolerance range is 15%. At 5000 psi or higher, the tolerance range is 5%.

The last diagnostic check is for deviation failures. Victus™ Simulation uses an algorithm to compare different possible combinations of pressure sensor subsets to determine the validity of individual sensor readings. If the algorithm determines that a pressure reading is inaccurate, the sensor will be suspended for the current iteration.
