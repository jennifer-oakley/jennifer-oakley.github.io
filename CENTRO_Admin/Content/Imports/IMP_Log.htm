About log data imports

On the Well Data Import page, you can import date-time, elapsed time, measured depth, or TVD Log data from an LAS or CSV file into a WITSML Log. You must import Log data into an existing WITSML Log. There are three basic steps required to import a Log from an LAS file:

  • Select the source. When you import the file, the data will be converted into WITSML curves using the mapping settings. See "Selecting the source file".
  • Select the destination Log. The destination settings tell CENTRO® Admin which well, wellbore, and WITSML Log to import the data into. See "Selecting the destination".
  • Select mapping options. The mapping settings tell CENTRO® Admin how to convert the data in the import file into WITSML curves. See "Selecting mapping options".

When you upload the import file, CENTRO® Admin will automatically detect the mnemonics used in the file.

You can also create a template to save the settings for mapping a Log to WITSML data. After you create a template, you can use it the next you import a Log. The mapping settings in the template will be applied to the new import file. See "Creating mapping templates" and "Applying a template".

Note

If a client company is assigned to your account, you will only be able to import data into wells that are associated with the same client company. See "User accounts".

Selecting the source file

To import a Log:

  1. Go to the Well Data Import page.
  2. Click the Choose button, then select the file you want to import.
  3. Click the Process button to upload the file.
  4. In the Import To field, select the Log option.
  5. In the Index Type field, select the method used to index the data in the file.
  6. Click the Next button at the bottom of the screen.
  7. Select the destination for the imported data, then click Next. See "Selecting the destination".
  8. Select the mapping options for the import file, then click Next. See "Selecting mapping options".
  9. Verify the range for the data and click the Import button.

The Log data will be imported from the selected file and converted into log curves for the selected WITSML Log. The mnemonics in the file will be converted into WITSML curves, based on the mapping options you selected.

Selecting the destination

To select the destination for the imported data:

  1. In the Well Name and Wellbore fields, select the name of the WITSML well and wellbore for the Log.
  2. Click the Next button at the bottom of the screen.
  3. Select the mapping options for the import file, then click Next. See "Selecting mapping options".

After you finish configuring the import, the selected file will be imported into the destination Log.

Selecting mapping options

To select the mapping options for a log import:

  1. Review and update the list of mnemonics detected in the import file. The WITSML mnemonic specified for the index curve must match the name of the index curve in the WITSML log.
  2. For each mnemonic listed, select the equivalent WITSML unit class and unit.
  3. Go to the Index section below the mapping table and select the curve used to index the rest of the data.
  4. If your log data is time-based, configure the time settings for your data. See "Field definitions".

When you import the file, the settings for the log mapping will be used to convert the data into WITSML curves. To save the mapping settings for future imports, you can create a mapping template. See "Creating mapping templates".

Creating mapping templates

To create a template for a Log import:

  1. During the import process, select the mapping settings for the imported file. See "Selecting mapping options".
  2. Click the Create Template button () at the top of the Log Mappings table.
  3. In the Mapping Template window, enter a name for the template and click Save.

All of the mapping settings will be saved to the Log template. The next time you import a Log file, you can apply the template to use the same settings again. See "Applying a template".

Applying a template

To apply a Log template:

  1. On the Mapping screen, click the Preview Template button () at the top of the Log Mappings table.
  2. Select the template you want to use, then click the Load Template button ().
  3. Click the Apply button.
  4. Review the mapping settings, then click Next to continue the import process.

The settings in the Log template will be applied to mapping for the import file.

Field definitions

The table below defines the columns in the mnemonics table.

Field definitions for the mnemonics table
Column Name Definition
Import Indicates that the mnemonic will be imported into the WITSML trajectory. If the Import option is not selected, the mnemonic will be skipped during the import process.
LAS Mnemonic Name of the mnemonic in the LAS import file. Only available for LAS imports.
CSV Mnemonic Name of the mnemonic in the CSV import fle. Only available for CSV imports.
LAS Unit Unit used for the mnemonic in the LAS import file. Only available for LAS imports.
CSV Unit Unit used for the mnemonic in the CSV import file. Only available for CSV imports.
WITSML Mnemonic

Name of the equivalent WITSML mnemonic
WITSML Unit Class Type of measurement unit associated with the WITSML mnemonic
WITSML Unit Unit associated with the WITSML mnemonic

The table below defines the fields in the Index section for time-based logs.

Field definitions for date-time indexes
Field Name Definition
Date Format Type

Method for entering the date and time format used in the import file.

  • Generic – Enter the date and time format by selecting the appropriate options from several menus for the date, year, month, day and time.
  • Custom – Enter the date and time format by entering a text string.
Date Time Format

Order that the date, time, and time zone appear in the index curve for the import file. Only available when the Generic option is selected in the Date Format Type field.

  • Date Time – Date appears first, followed by the time. No time zone specified.
  • Date Time TimeZone – Date appears first, followed by time, then by the time zone.
  • Time Date – Time appears first, followed by date. No time zone specified.
  • Time Date TimeZone – Time appears first, followed by date, then by the time zone
Date Format

Format used for the date and time in the index curve.

  • If the Generic option is selected for the date time format, select the order that the month, day, and year appear within the date in the index curve.
  • If the Custom is option is selected, enter a text string to specify the format for date and time, e.g. YYYY-MM-DD hh:mm:ss,
Year Format Format used to specify the year in the date portion of the timestamp. Only available when the Generic option is selected in the Date Format Type field.
Month Format Format used to specify the month in the date portion of the timestamp. Only available when the Generic option is selected in the Date Format Type field.
Day Format Format used to specify the day in the date portion of the timestamp. Only available when the Generic option is selected in the Date Format Type field.
Date-Time Delimiter Character used to separate the date and the time, e.g. a space or a period. Only available when the Generic option is selected in the Date Format Type field.
Month-Day-Year Delimiter Character used to separate the month, day, and year within the date, e.g. a slash or a dash. Only available when the Generic option is selected in the Date Format Type field.
Time Format Format used to specify the time portion of the timestamp. Only available when the Generic option is selected in the Date Format Type field.
Fractional Seconds Precision Number of decimal places used to specify fractions of a second. Only available when the Generic option is selected in the Date Format Type field.
Time Zone Offset Offset of the timestamp from GMT
