About alarms

The Alarms > Alarm Management page shows you all of the alarms that have been created for each well on the data center. To view the alarms for a well, go to the Alarms Management page and click on the name of the well.

You can create two types of alarms on the Alarm Management page:

  • Threshold alarms alert you when CENTRO® receives a value that is outside of defined minimum or maximum ranges. Threshold alarms can be applied to curves on WITSML logs and trajectories. See "Adding threshold alarms".
  • No Contact alarms alert you when CENTRO® does not receive any data within the expected time frame. No contact alarms can be applied to logs, trajectories, and mud logs. See "Adding no-contact alarms".

After you create an alarm, you'll need to update the subscription settings for the alarm. The subscription settings determine who gets notified when the alarm is triggered, as well as how they are notified. You can subscribe individual users to an alarm, or you can subscribe a role. When you subscribe all role, all of the users assigned to that role will be notified when the alarm is triggered.

Note

The name of an alarm must be unique across all wells on the data center.

Adding no-contact alarms

No Contact alarms alert you when CENTRO® does not receive any data within the expected time frame. No contact alarms can be applied to logs, trajectories, or mud logs. They can also be applied to specific curves in a log, trajectory, or a mud log.

To add a no contact alarm:

  1. Go to the Alarms > Alarms Management page.
  2. Click on the name of the well that you want to add an alarm to.
  3. Click the Add Alarm button () and select the No Contact option.
  4. On the left side of the screen, enter a name for the alarm in the Alarm Name field.
  5. Select the type of WITSML object.
  6. Select the name of the WITSML object that you want to monitor.
  7. If you want to monitor specific curves, select the Apply On Curve option and select the curves you want to monitor.
  8. In the table on the right side of the screen, enter the interval at which an alarm should be raised for each alarm level.
  9. Click the Create button () in the upper right corner to save the alarm settings.
  10. Configure the subscription settings for the alarm. See "Subscribing to an alarm".

If you selected the Apply On Curve option, an alarm will be raised if no data is received for the selected WITSML curves. If you did not select the Apply On Curve option, an alarm will be raised if no data is received for the selected WITSML object. The length of time required for an alarm to be raised is determined by the values entered in the Interval column. When the alarm is triggered, notifications will be sent based on the settings on the Subscription tab. See "Subscribing to an alarm".

Adding threshold alarms

Threshold alarms allow you to create a conditional statement to determine if an alarm should be raised. See "Adding threshold alarms". There are four basic components that you can use to build a threshold alarm:

  • Logic components define the logic that determines whether or not the alarm will be triggered. Logic components can be defined using a combination of math components, mnemonic components, and other logic components. See "Adding logic components".
  • Math components define the mathematical values used in a logic component. See "Adding math components".
  • Alarm Mnemonic components define the WITSML mnemonics used in a logic component. See "Adding alarm mnemonics".
  • Alarm Action components define the alarm severity when the alarm is triggered, as well as the message that appears in the alarm notification. See "Configuring alarm actions".

The foundation of a threshold alarm is the If component. CENTRO® will evaluate the condition defined for the If component. If the condition is true, the alarm will be triggered using the settings in the action component.

A basic threshold alarm contains one conditional statement in the If component. You can also configure the If component to include additional conditional statements. See "Configuring the If component". For each conditional statement, you'll need to add logic components to define the logic for the condition.

To add a threshold alarm:

  1. Go to the Alarms > Alarms Management page.
  2. Click on the name of the well that you want to add an alarm to.
  3. Click the Add Alarm button () and select the Threshold option.
  4. On the left side of the screen, enter a name for the alarm in the Alarm Name field.
  5. To add conditional statements to the alarm, configure the If component. See "Managing alarms".
  6. Add logic components to each conditional statement. See "Managing alarms".
  7. Add math components and WITSML mnemonics to the logic components. See "Adding math components" and "Adding alarm mnemonics".
  8. Configure the alarm actions for each conditional statement. See "Managing alarms".
  9. Click the Create button in the upper right corner to save the alarm settings.
  10. Configure the subscription settings for the alarm. See "Subscribing to an alarm".

CENTRO® will evaluate the conditions defined for each conditional statement in the If component. If a condition is true, the alarm will be triggered using the settings in the action component. When the alarm is triggered, notifications will be sent based on the settings on the Subscription tab.
