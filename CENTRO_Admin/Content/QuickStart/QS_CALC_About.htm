About calculations

The Calculations menu is used to map mnemonics to curves and configure calculations. There are several pages in the Calculations menu:

  • The Mappings page allows you to map the curve data for physical measurements to the corresponding WITSML mnemonics. For more information about this feature, refer to the main user guide topic. See "Mappings".
  • The Despike page allows you to create and edit despike calculations that automatically remove data spikes from a log. See "Despike calculations".
  • The Rig States page allows you to configure a rig state calculation that automatically determines the current rig activity for a well. This page is only available for data hubs. See "Rig state calculations".
  • The Fingerprinting page allows you to create and edit flowback calculations for monitoring the flow in the well, based on incoming data for flow in, flow out, and standpipe pressure. See "Fingerprinting calculations".
  • The Pressure Pulse page allows you to create calculations to identify pressure pulses, based on incoming data for standpipe pressure. See "Pressure pulse calculations".

Despike calculations

On the Calculations > Despike page, you can configure calculations to automatically remove data spikes from a time-based or depth-based log. Despiking automatically removes false zeroes and invalid data points that exceed the defined limits for a curve.

When a despike calculation is running, CENTRO® will check the incoming values of the mnemonics specified in the despike calculation. If a false zero or an invalid value is received, it will be replaced with a new value, based on the despiking strategy selected for that mnemonic. See "Despike ranges".

To create a despike calculation:

  1. Go to the Calculations > Despike page.
  2. Click the Create Calculation button () at the top of the page.
  3. On the New Despike Calculation page, go to the Well field and select the name of the well.
  4. In the Wellbore field, select the wellbore name.
  5. In the Log field, select the name of the log you want to despike.
  6. In the Mapping for Physical Measurements and Log Mnemonics section, map the mnemonics you want to despike to the appropriate WITSML measurement units. See "Adding mappings to a despike calculation".
  7. Create an index range for the time period or depth range that you want the despiking calculation to be applied to. See "Adding index ranges".
  8. For each index range, add the curves you want to evaluate for invalid values. See "Adding curves to an index range".
  9. Click the Save button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML log will be created for the despike curves. The new log will have the same name as the original log, with _DS added to the end of the name, e.g. [OriginalLogName]_DS. The new calculation will be listed on the Calculations > Despike page. The Calculations > Despike page will show the current status of the calculation and the time that the calculated was created.

Rig state calculations

On the Calculations > Rig States page, you can configure a calculation to automatically determine the current rig activity for a well.

When you start the calculation, a new WITSML log will be created with the curves listed below. A value of 1 means that the condition is true, and a value of 0 means that the condition is false.

  • AtSurface – Indicates that the bit is close to the surface.
  • BitMovingDown – Indicates that the bit depth is increasing, and the block is moving down.
  • BlockIsIdle – Indicates that the block position is remaining the same.
  • FarBottom – Indicates that the bit is not near the bottom of the well.
  • InSlips – Indicates that the block weight exceeds the hookload.
  • OnBottom – Indicates that the bit depth is the same as the well depth.
  • PumpsOn – Indicates that fluid is pumping.
  • RigState – Indicates that the rig activity was successfully calculated.
  • RotationOn – Indicates that the top drive is rotating.

The constants used to determine if these conditions are true can be customized in the Additional Settings table. See "Formula constants". These curves are used to determine the current rig activity, which is converted to a rig activity code. The WITSML log also includes a RigActivityCode curve, which contains this code. See "Activity codes".

To configure a rig state calculation:

  1. Go to the Calculations > Rig States page.
  2. If you are creating a calculation on the data center, click the Create Calculation button () at the top of the page.
  3. In the Well Name field, select the well that you want to configure calculations for.
  4. In the Wellbore field, select the name of the wellbore.
  5. In the Log field, select the log that contains the drilling parameters for the well.
  6. If the BHA contains a mud motor, select the Has Drilling Motor option.
  7. In the Block Weight field, enter the weight of the block.
  8. Go to the Additional Settings table and update the default constants as needed. See "Formula constants".
  9. Go to the Mapping for Calculation Parameters and Log Mnemonics table and select the WITSML mnemonic and unit for each parameter.
  10. Click the Save button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML log will be created for the calculation conditions and result. The new log will have the same name as the drilling parameters log, with _Calc added to the end of the name, e.g. [DrillingParametersLogName]_Calc. The Calculations > Rig States page will show the current status of the calculation, the time that the drilling parameters log started receiving data, and the time of the last calculation.

Note

If any of the values for the calculation parameters are null or missing, the rig activity will not be calculated.

Fingerprinting calculations

On the Calculations > Fingerprinting page, you can configure fingerprinting calculations to monitor the flow out of the well. When the pumps turn on, the fingerprinting calculation will calculate the flowback profile, which shows the amount of fluid flowing out of the well over time. A time-based log with data for flow in, flow out, and standpipe pressure is required for fingerprinting calculations.

The formula used to determine if the pumps are on is shown below the log parameters. If the current values for both flow rate and standpipe pressure exceed the minimum values defined in the Additional Settings section, the pumps are assumed to be on. When the values for either flow rate or standpipe pressure drop below the minimum values, the pumps are assumed to be off. See "Model parameters".

To create a fingerprinting calculation:

  1. Go to the Calculations > Fingerprinting page.
  2. Click the Create Calculation button () at the top of the page.
  3. On the New Fingerprinting Calculation page, go to the Well field and select the name of the well.
  4. In the Wellbore field, select the wellbore name.
  5. In the Log field, select the name of the log that contains the flow data.
  6. In the Unit for Elapsed Time field, select time-based unit you want to use for the output.
  7. In the Mapping for Calculation Parameters and Log Mnemonics section, map the mnemonics for flow out, flow in, and standpipe pressure.
  8. Click the Save button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML log will be created for the fingerprinting curves. The new log will have the same name as the original log, with _FP added to the end of the name, e.g. [OriginalLogName]_FP. The new calculation will be listed on the Calculations > Fingerprinting page. The Calculations > Fingerprinting page will show the current status of the calculation and the time that the calculated was created.

Pressure pulse calculations

On the Calculations > Pressure Pulse page, you can configure pressure pulse calculations. Pressure pulse calculations allow you to identify a pressure pulse, based on incoming data for standpipe pressure. The pressure pulse cycle is used to activate a tool function when it is not possible to flow an RFID tag, e.g. to open a sleeve in a closed system.

The criteria for CENTRO® Admin to identify a pressure pulse are:

  • The range of change for standpipe pressure must be more than 350 psi/min.
  • The differential pressure must be more than 256 psi. Differential pressure is the difference between the base line pressure and the hold pressure.
  • The time between a rise and a fall in standpipe pressure must be more than three minutes.

When you create a pressure pulse calculation, a mnemonic for the rate of change in pressure will be added to the log. The RateOfPressureChange mnemonic shows the amount of change in the standpipe pressure since the last data point was received.

To create a pressure pulse calculation:

  1. Go to the Calculations > Pressure Pulse page.
  2. Click the Create Calculation button () at the top of the page.
  3. On the New Pressure Pulse Calculation page, go to the Well field and select the name of the well.
  4. In the Wellbore field, select the wellbore name.
  5. In the Log field, select the name of the log that contains the pressure data.
  6. In the Output Mnemonic Unit field, select time-based unit you want to use for the output mnemonic.
  7. In the Mapping for Calculation Parameters and Log Mnemonics section, map the mnemonics for standpipe pressure.
  8. Click the Save button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML mnemonic RateOfPressureChange will be added to the log. The RateOfPressureChange mnemonic shows the amount of change in the standpipe pressure since the last data point was received. The new calculation will be listed on the Calculations > Pressure Pulse page. The Calculations > Pressure Pulse page will show the current status of the calculation and the time that the calculated was created.
