About rig state calculations

On the Calculations > Rig States page, you can configure a calculation to automatically determine the current rig activity for a well. See "Configuring rig state calculations".

When you start the calculation, a new WITSML log will be created with the curves listed below. A value of 1 means that the condition is true, and a value of 0 means that the condition is false.

  • AtSurface – Indicates that the bit is close to the surface.
  • BitMovingDown – Indicates that the bit depth is increasing, and the block is moving down.
  • BlockIsIdle – Indicates that the block position is remaining the same.
  • FarBottom – Indicates that the bit is not near the bottom of the well.
  • InSlips – Indicates that the block weight exceeds the hookload.
  • OnBottom – Indicates that the bit depth is the same as the well depth.
  • PumpsOn – Indicates that fluid is pumping.
  • RigState – Indicates that the rig activity was successfully calculated.
  • RotationOn – Indicates that the top drive is rotating.

The constants used to determine if these conditions are true can be customized in the Additional Settings table. See "Formula constants". These curves are used to determine the current rig activity, which is converted to a rig activity code. The WITSML log also includes a RigActivityCode curve, which contains this code. See "Activity codes".

Note

Currently, you can only run rig state calculations for one well on each data hub. However, you can run rig state calculations for multiple wells on the data center.

Note

The Calculations > Rig States page only lists calculations for the wells that you have been granted access to.

Configuring rig state calculations

To configure a rig state calculation:

  1. Go to the Calculations > Rig States page.
  2. If you are creating a calculation on the data center, click the Create Calculation button () at the top of the page.
  3. In the Well Name field, select the well that you want to configure calculations for.
  4. In the Wellbore field, select the name of the wellbore.
  5. In the Log field, select the log that contains the drilling parameters for the well.
  6. If the BHA contains a mud motor, select the Has Drilling Motor option.
  7. In the Block Weight field, enter the weight of the block.
  8. Go to the Additional Settings table and update the default constants as needed. See "Formula constants".
  9. Go to the Mapping for Calculation Parameters and Log Mnemonics table and select the WITSML mnemonic and unit for each parameter.
  10. Click the Save button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML log will be created for the calculation conditions and result. The new log will have the same name as the drilling parameters log, with _Calc added to the end of the name, e.g. [DrillingParametersLogName]_Calc. The Calculations > Rig States page will show the current status of the calculation, the time that the drilling parameters log started receiving data, and the time of the last calculation.

Note

If any of the values for the calculation parameters are null or missing, the rig activity will not be calculated.

Exporting rig state calculations

After you have saved a rig state calculation, you can export it to a file and import it into another well. To export the current rig state mapping to a file:

  1. Go to the Calculations > Rig States page.
  2. In the Well Name field, select the well that contains the calculations you want to export.
  3. In the Wellbore field, select the name of the wellbore.
  4. In the Log field, select the log that contains the calculations you want to option.
  5. Click the Export to Rig State Calculation File button ().

The calculation file will be automatically downloaded to the Downloads directory on your computer.

Note

Calculation files can't be exported while a calculation is running.

Importing rig state calculations

To import rig state calculations from a file:

  1. Go to the Calculations > Rig States page.
  2. If you are creating a calculation on the data center, click the Create Calculation button () at the top of the page.
  3. In the Well Name field, select the well that you want to configure calculations for.
  4. In the Wellbore field, select the name of the wellbore.
  5. In the Log field, select the log that contains the drilling parameters for the well.
  6. Click the Import From Rigstate Calculation File button ().
  7. In the Import Rigstate Calculation File window, click the Choose button.
  8. Navigate to file you want to import, then click Open.
  9. In the Import Rigstate Calculation File window, click the Overwrite button.
  10. Click the Save button () to save your changes, then click the Start button () to begin calculations.

The settings for the rig state calculation will be imported from the selected file. If any of the log mnemonics in the imported file do not match the log mnemonics for the current log, you will need to manually configure those mnemonics. See "Configuring rig state calculations".

Formula constants

The table below lists the constants used to calculate the rig activity. The values for these constants can be configured in the Additional Settings table on the Calculations > Rig States page.

Definitions for formula constants
Constant Definition
Block Speed Down Constant used to determine if the block is moving down. If the block speed is less than the Block Speed Down value, and the bit speed is greater than the Running Speed - Down value, the BitMovingDown condition will be true.
Distance From Surface Constant used to determine if the bit is near the surface. If the bit depth is less than the Distance From Surface value, the AtSurface condition will be true.
Far Bottom Difference Constant used to determine how the bit is from the bottom of the well. If the difference between bit depth and well depth is greater than the Far Bottom Difference value, the FarBottom condition will be true.
Idle Block Speed Constant used to determine if the block is moving. If the block speed is less than the Idle Block Speed value, the BlockIsIdle condition will be true.
In Slips Coefficient Coefficient used to determine if the block is in slips. If the block weight exceeds the hookload by a factor larger than the In Slips Coefficient value, the InSlips condition will be true.
On Bottom Difference Constant used to determine if the bit is on-bottom. If the difference between the bit depth and well depth is less than the On Bottom Difference value, the OnBottom condition will be true.
Pumps On - Flow Rate Constant used to determine if the pumps are on. If the flow rate is greater the Pumps On - Flow Rate value, and standpipe pressure is greater than the Pumps On - SPP value, the PumpsOn condition will be true.
Pumps On - SPP Constant used to determine if the pumps are on. If the standpipe pressure is greater than the Pumps On - SPP value, and the flow rate is greater the Pumps On - Flow Rate value, the PumpsOn condition will be true.
Rotary Cutoff Constant used to determine if the top drive is rotating. If the top drive speed is greater than the Rotary Cutoff value, the RotationOn condition will be true.
Running Speed - Down Condition used to determine if the bit depth is increasing. If the bit speed is greater than the Running Speed - Down value, and the block speed is less than the Block Speed Down value, the BitMovingDown condition will be true.

Using the Rig States page

The search box above the table on the Rig States page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.

The buttons on the Rig States page are used to add, start, stop, and delete rig state calculations.

Buttons on the Rig States page
Button Function
Adds a new rig state calculation. See "Configuring rig state calculations".
Refreshes the status of the calculations on the Rig States module
Starts the corresponding rig state calculation
Stops the corresponding rig state calculation
Allows you to view and edit the corresponding rig state calculation
Creates a copy of the corresponding rig state calculation
Deletes the corresponding rig state calculation. If the calculation is currently running, you must stop the calculation, before you can delete it.

When you add or edit a rig state calculation, the buttons on the Edit Rig State Calculation page allow you to start and stop the calculation, as well an export and import the calculation settings.

Buttons on the Edit Rig State Calculation page
Button Function
Saves all changes
Save the current calculation settings and starts calculations
Exports the current mappings to a file. See "Exporting rig state calculations".
Imports a rig state calculation file into the current mappings. See "Importing rig state calculations".
Stops the rig state calculation
Deletes the existing output log and restarts the calculation into a new log, from the start time.
