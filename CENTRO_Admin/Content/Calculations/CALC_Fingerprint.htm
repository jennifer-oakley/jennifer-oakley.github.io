About fingerprinting calculations

On the Calculations > Fingerprinting page, you can configure fingerprinting calculations to monitor the flow out of the well. When the pumps turn on, the fingerprinting calculation will calculate the flowback profile, which shows the amount of fluid flowing out of the well over time. A time-based log with data for flow in, flow out, and standpipe pressure is required for fingerprinting calculations.

The formula used to determine if the pumps are on is shown below the log parameters. If the current values for both flow rate and standpipe pressure exceed the minimum values defined in the Additional Settings section, the pumps are assumed to be on. When the values for either flow rate or standpipe pressure drop below the minimum values, the pumps are assumed to be off. See "Model parameters".

Note

If you stop and restart calculations, calculations will resume at the last processed index point.

Note

The Calculations > Fingerprinting page only lists calculations for the wells that you have been granted access to.

Creating fingerprinting calculations

To create a fingerprinting calculation:

  1. Go to the Calculations > Fingerprinting page.
  2. Click the Create Calculation button () at the top of the page.
  3. On the New Fingerprinting Calculation page, go to the Well field and select the name of the well.
  4. In the Wellbore field, select the wellbore name.
  5. In the Log field, select the name of the log that contains the flow data.
  6. In the Unit for Elapsed Time field, select time-based unit you want to use for the output.
  7. In the Mapping for Calculation Parameters and Log Mnemonics section, map the mnemonics for flow out, flow in, and standpipe pressure.
  8. Click the Save button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML log will be created for the fingerprinting curves. The new log will have the same name as the original log, with _FP added to the end of the name, e.g. [OriginalLogName]_FP. The new calculation will be listed on the Calculations > Fingerprinting page. The Calculations > Fingerprinting page will show the current status of the calculation and the time that the calculated was created.

Model parameters

The table below defines the model parameters in the Additional Settings table on the Fingerprinting page.

Definitions for the model settings
Constant Definition
Pump On - Flow Rate Minimum flow rate used to determine if the pumps are on. If the flow rate is below this value, the pumps are assumed to be off.
Pumps On - SPP Minimum standpipe pressure used to determine if the pumps are on. If the standpipe pressure is below this value, the pumps are assumed to be off.

Using the Fingerprinting page

The search box above the table on the Fingerprinting page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.

The buttons on the Fingerprinting page are used to add, start, stop, and delete fingerprinting calculations.

Buttons on the Fingerprinting page
Button Function
Adds a new fingerprinting calculation. See "Creating fingerprinting calculations".
Refreshes the status of the calculations on the Fingerprinting module
Starts the corresponding fingerprinting calculation
Stops the corresponding fingerprinting calculation
Allows you to view and edit the corresponding fingerprinting calculation
Creates a copy of the corresponding fingerprinting calculation
Deletes the corresponding fingerprinting calculation. If the calculation is currently running, you must stop the calculation, before you can delete it.

When you add or edit a fingerprinting calculation, the buttons on the Edit Fingerprinting Calculation page allow you to start and stop the calculation.

Buttons on the Edit Fingerprinting Calculation page
Button Function
Saves all changes
Save the current calculation settings and starts calculations
Stops the fingerprinting calculation
Deletes the existing output log and restarts the calculation into a new log, from the start time.
