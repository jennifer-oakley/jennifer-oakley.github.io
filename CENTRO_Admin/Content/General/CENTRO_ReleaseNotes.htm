CENTRO® Admin v6.0 Release Notes

In this article

New features

Bugs fixed

Welcome to CENTRO® Admin v6.0. This version of CENTRO® Admin contains several new features and enhancements, as well as many bug fixes. For a full list of the updates in v6.0, refer to the sections below.

New features

The features in this section are new for CENTRO® Admin v6.0.

  • CENTRO® Admin now supports user authentication through Azure AD B2C. Azure AD B2C is a cloud-based identity management system that supports multi-factor authentication and single sign-on for several Weatherford applications, including CENTRO®, CENTRO® Engineer, and Surveyors Workbench. On the User Management > Users page, you can add an Azure AD B2C account to grant access to the data center. See "Azure B2C user accounts".
  • Wells created in CENTRO® Admin can now be integrated with CENTRO® Monitor. When you integrate a well with CENTRO® Monitor, the well will be mapped to a new project in CENTRO® Monitor. CENTRO® Admin will also create well-specific roles, which are used to grant access to the well. When a user is assigned to a well-specific role, a user account with a similar access level will also be created in CENTRO® Monitor. To use this feature, go to the Wells > Well Management page and click the Create Project/Groups button () next to the well name. See "CENTRO® Monitor well integration".
  • Improved the time required to load the user well access report and the well status report on the Reports > User/Well page. Previously, reports with a large number of wells took an excessive amount of time to create the reports.
  • Improved the time required to fetch WITSML well details for all wells on the server.

Bugs fixed

The critical issues in this section have been fixed in CENTRO® Admin v6.0.

  • When you select multiple logs for an ROP optimization calculation on the Predict Models page, the list of mnemonics for the selected log should now include the mnemonics for flow rate. Previously, the mnemonics for flow rate were only shown when using a single log for ROP optimization.
  • Fixed an issue that caused wells with CustomData attributes to fail to load on the Home page in CENTRO® Monitor.
