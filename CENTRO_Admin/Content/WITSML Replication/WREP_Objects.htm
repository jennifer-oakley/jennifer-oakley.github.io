About replicating WITSML objects

If you do not include all WITSML objects in a replication route, you will need to specify which objects you want to include on the Select Objects To Replicate screen. You can include date-time logs, elapsed time logs, depth logs, mud logs, and trajectories in a replication route.

The Select Objects To Replicate screen is only available when you disable the Replicate All Growing Objects option on the Select Well/Wellbore screen. See "Well settings".

If you select the Replicate All Growing Objects option, the setup wizard will skip the Select Objects To Replicate screen.

Adding WITSML objects

To add WITSML objects to the replication route:

  1. Go to the WITSML Replication page.
  2. Click the Create WITSML Replicator button () to create a new replication route, or click the More Info button () to edit an existing route.
  3. Go to the Select Objects to Replicate page, then click the Add button () and select the type of object you want to replicate.
  4. In the Object Selection window, select the name of the object you want to replicate.
  5. Updated the object replication settings as needed, then click the Save button. See "Field definitions".
  6. When you are finished adding the WITSML objects that you want to replicate, click the Next button.

No data will be copied to the destination server until you start the replication. When you start the replication, the selected data will be copied from the source well to the destination well. If you are copying data to a new well, the destination well will be automatically created on the destination server. If you are copying data to an existing well, the data from the source well will be added to the destination well.

Field definitions

The table below defines the fields for the settings for log objects on the WITSML Replication > Select Objects To Replicate screen.

Field definitions for log settings
Section Name Field Name Definition
-- Log Name of the log you want to copy from the source well
Data Source Start Index Starting time or depth to begin replicating the log. If not specified, the start index from the source log will be used.
End Index Ending time or depth to stop replicating the log. If not specified, the end index from the source log will be used.
New Offset Time zone offset of the time stamp from GMT. If not specified, the time zone offset from the source log will be used. Only available for date-time logs.
Block Size Maximum number of rows that are copied to the destination log during each data transaction
Delay Between Data Transactions Frequency with which the destination server will check for new data on the source server
Data Destination Use Source's Indicates that the data will be copied to a new log with the same name as the source log
Existing Indicates that the data will be copied to an existing log in the destination well
Custom Indicates that the data will be copied to a new log with a different name from the source log
Log Name Name of the log on the destination server. When the Use Source's option is selected, this field is read only.
Log UID UID of the log on the destination server. Only available when the Custom option is selected.
