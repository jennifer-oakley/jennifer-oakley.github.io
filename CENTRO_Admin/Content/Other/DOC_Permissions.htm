About document store administration

Access to the document store can be granted to a folder, a well, or the entire document store. When you edit access levels, the new permissions will be inherited by any items inside that folder or well. There are four levels of access:

  • None – The folder or well will not be visible for users with this access level.
  • Read – Users can view the folder contents and download files. However, they will not be able to create folders or manage files.
  • Edit – Users can view the folder contents, download files, and upload new files. However, they will not be able to create folders or manage existing files.
  • Full – Users have full access to the folder or well. In addition to downloading and uploading files, they can also manage existing files and change the permissions for the folder or well.

The permissions set at the folder or well level are inherited by all of the folders and documents inside of it. However, if additional permissions are set on a lower-level folder, those permissions will override the top-level permissions. See "Changing well and folder permissions".

The Document Store page is only available for the server that you are logged into. If you are logged into the data center, you cannot access the document store for any of the data hubs connected to it. To access the document store for a data hub, you will need to log into the data hub directly.

Note

Users with a client company assigned to their account will only be able to view documents that are associated with the same client company. See "User accounts".

Changing well and folder permissions

To change the permissions for a well or folder:

  1. Go to the Document Store page.
  2. Navigate to the folder or well that you want to grant access to.
  3. Click the Actions button () next to the name of the folder or well, then select the ACLs option.
  4. In the window that appears, use the sliders to set the access level for each role or user.
  5. Click the Save button.

The permissions will be applied to the contents of the well or folder. Unless additional permissions are set on a lower-level folder, all documents and folders will inherit the new permissions.

Managing subscriptions

In the Manage Subscriptions for Well window, you can manage the users who are subscribed to the document store for the well. Users who are subscribed to the document store will receive an email notification when a new document is uploaded to the well.

To open the Manage Subscriptions for Well window, click the Actions button () next to the name of well you want to subscribe to, then select the Subscriptions option. The list of users who are currently subscribed to notifications will be shown in the Subscribed Users list.

  • To subscribe users to the document store, select the users you want to add in the Available Users list. Then, click the Add button () to move the selected users to the Subscribed Users list.
  • To subscribe all available users to the document store, click the Add All button () to move all users to the Subscribed Users list.
  • To unsubscribe users from the document store, select the users you want to remove in the Subscribed Users list. Then, click the Remove button () to move the selected users to the Available Users list.
  • To unsubscribe all subscribed users from the document store, click the Remove All button () to move all users to the Available Users list.

Viewing document downloads

The No. of Hits column shows the number of times a document has been downloaded, as well as the download history for the document.

To view the download history of a document:

  1. Go to the Document Store page.
  2. Navigate to the document you want to view the download history for.
  3. Go to the No. of Hits column for the document and click on the value inside the cell.

The download history for the document will open in a new window.
