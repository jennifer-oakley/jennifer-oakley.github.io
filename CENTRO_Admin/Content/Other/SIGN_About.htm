About sign-up requests

The Signup Requests page lists all of the pending sign-up requests for the data center. From this page, pending requests can be approved or rejected.

When a sign-up request for a user with a new client company is approved, several client-specific roles will be created.

  • The [ClientName]_Administrator role grants access to CENTRO® Admin and administrator access to CENTRO® Monitor. In CENTRO® Admin, client administrators have access to manage data hubs, wells, documents, WITSML data, and user accounts.
  • The [ClientName]_ReadWrite role grants read/write access to WITSML data in CENTRO® Admin, as well as read/write access to CENTRO® Monitor.
  • The [ClientName]_ReadOnly role grants read only access to WITSML data in CENTRO® Admin, as well as read only access to CENTRO® Monitor.

All client-specific roles limit access based on the client company. Users who have a client-specific role assigned to their account will only be able to view data for their client company, such as data hubs, wells, documents, WITSML data, users, and roles.

Reviewing sign-up requests

To review a sign-up request:

  1. Go to the Signup Requests page.
  2. Locate the request you want to review, then click the Edit button ().
  3. Go to the Client Information section and select the client company for the new user.
    • If the client company already exists on the server, select the Existing Client option and select the name of the client company.
    • If the client company does not exist, select the Create New Client option and enter the name of the company.
  4. In the User Count (Max) field, enter the maximum number of licenses allowed for the client company.
  5. To approve the request, click the Approve button (). To reject the request, click the Reject button ().

If you reject the request, you will be prompted to enter a reason for the rejection. After you confirm the rejection, the request will be removed from the list of active sign-up requests.

If you approve the request, you will be prompted to confirm the approval. A new account will be automatically created for the new user, with the client-specific administrator role assigned to it. An automatically generated email will be sent to the new user with the CENTRO® URL, user name, and temporary password. If the client company is a new company, a new client will be added on the Lookups > Clients page.

After you approve or reject the request, rejection, the request will be removed from the list of active sign-up requests.
