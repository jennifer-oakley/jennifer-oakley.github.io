About managing alarms

The Alarms > Alarm Management page shows you all of the alarms that have been created for each well on the data center. To view the alarms for a well, go to the Alarms Management page and click on the name of the well.

You can create two types of alarms on the Alarm Management page:

  • Threshold alarms alert you when CENTRO® receives a value that is outside of defined minimum or maximum ranges. Threshold alarms can be applied to curves on WITSML logs and trajectories. See "Threshold alarms".
  • No Contact alarms alert you when CENTRO® does not receive any data within the expected time frame. No contact alarms can be applied to logs, trajectories, and mud logs. See "No contact alarms".

After you create an alarm, you'll need to update the subscription settings for the alarm. The subscription settings determine who gets notified when the alarm is triggered, as well as how they are notified. You can subscribe individual users to an alarm, or you can subscribe a role. When you subscribe all role, all of the users assigned to that role will be notified when the alarm is triggered.

Subscribing to an alarm

To configure the subscription settings for alarm:

  1. Go to the Alarms > Alarms Management page.
  2. Click on the name of the well for the alarm you want to update the subscription settings for.
  3. Click on the name of the alarm that you want to update.
  4. In the alarm settings, click on the Subscription tab.
  5. Click the Add button ().
  6. In the Choose Users and Roles window, select the users and roles you want to subscribe to the alarm in the Available Users & Roles list, then click the Add button ().
  7. Click the Add button () at the bottom of the window.
  8. In the Users & Groups table, select the notification settings for each user and role.
    • If you want users to receive an email when an alarm is triggered, go to the Email column and select the Allow option.
    • If you want users to receive a notification in the CENTRO® application, go to the UI column and select the Allow option.
    • If you want users to receive a text message when an alarm is triggered, go to the SMS column and select the Allow option.
  9. In the Severity & Above column, select the minimum severity level at which notifications should be sent to each user and role.
  10. Click the Save button () in the upper right corner to save the alarm settings.

When the alarm is triggered, users will receive email, text, and UI notifications according to the options selected in the Permissions column. If you subscribed a role to the alarm, notifications will be sent to each user assigned to that role. The email notification will be a single email sent to all email subscribers, using the email address listed in each user's account. Text notifications will be sent to the phone number selected in the Notification field in the user's account. See "User accounts".

Warning

If a phone number isn't selected in the Notification field of the user's account, text notifications will not be sent. See "User accounts".

Tip

To select multiple users or roles at the same time, press the Ctrl key and select the users and roles you want to add.

Field definitions

The table below defines the fields when you view the alarms for a well on the Alarms Management page.

Column names on the Alarms Management page
Column Name Definition
Alarm Name Name of the alarm
Alarm Type Icon indicating the type of alarm. See "Using the Alarms Management page".
Object Name of the WITSML object that the alarm applies to. The icon on the right indicates the type of WITSML object. See "Using the Alarms Management page".
Variables Name of the WITSML curve that the alarm applies to. If no curve name is shown, the alarm applies to the entire WITSML object.
Effective From Time that the alarm was created, or the current depth at the time that the alarm was created
Last Evaluated Depth or time that the alarm was last evaluated
Last Evaluated Time Stamp Time that the alarm was last evaluated
Status

Icon indicating the current status of the alarm.

  • A gray icon indicates that the alarm has not been triggered at any point.
  • A yellow icon indicates that the alarm was triggered previously, but has now been resolved. The number next to the icon indicates the number of inactive alerts.
  • A red icon indicates that the alarm has been triggered and has not been resolved. The number next to the icon indicates the number of active alerts.

Using the Alarms Management page

The search box above the table on the Alarms Management page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.
