About alarm templates

On the Alarms > Alarm Templates page, you can view and create templates for threshold alarms. After you create an alarm template, other users will be able to use the template to create a new alarm. Alarm templates can be applied to alarms for any well on the server. See "Applying an alarm template".

Alarm templates are created in the same way as threshold alarms, except that they use placeholders for WITSML curves. Like threshold alarms, there are four basic components for building an alarm template:

  • Logic components define the logic that determines whether or not the alarm will be triggered. Logic components can be defined using a combination of math components, mnemonic components, and other logic components. See "Adding logic components".
  • Math components define the mathematical values used in a logic component. See "Adding math components".
  • Placeholder components are used to define where WITSML mnemonics will be used when the template is applied to an alarm.
  • Alarm Action components define the alarm severity when the alarm is triggered, as well as the message that appears in the alarm notification. See "Configuring alarm actions".

A basic alarm template contains one conditional statement in the If component. You can also configure the If component to include additional conditional statements. See "Configuring the If component". For each conditional statement, you'll need to add logic components to define the logic for the condition.

Creating a template

To add an alarm template:

  1. Go to the Alarms > Alarm Templates page.
  2. Click the Add Template button ().
  3. In the Template Detail window, enter a name for the template in the Template Name field.
  4. To add conditional statements to the template, configure the If component. See "Configuring the If component".
  5. Add logic components to each conditional statement. See "Adding logic components".
  6. Add math components and to the logic components. See "Adding math components".
  7. Click on the Placeholder heading and drag the Curve Placeholder component to the grid area.
  8. Connect the curve placeholder to a logic component.
  9. Configure the alarm actions for each conditional statement. See "Configuring alarm actions".
  10. Click the Save button to save the template.

The new template can be applied to a threshold alarm for any well on the server. To apply to the template to a new alarm, create a threshold alarm and click the Load Alarm Template button (). See "Applying an alarm template".

Configuring the If component

To configure the If component:

  1. Click on the Settings button () in the top left corner of the If component.
  2. In the window that appears, click on the component you want to add in the left side of the window.
    • The Else If component adds a conditional statement to the If component. If the condition defined for the initial If component is false, the alarm rule will check the condition defined for the Else If component.
    • The Else component adds an alarm action to the If component. If the condition defined for the initial If component is false, the alarm rule will execute the action defined for the Else component.
  3. Connect the component to the If component in the right side of the window.
  4. Click the Settings button () again to close the window.

Additional conditional statements and alarm actions will appear in the If component. For each conditional statement, you'll need to add logic components to define the logic for the condition. See "Adding logic components". For each alarm action, you'll need to add action components to define what happens when the alarm triggered. See "Configuring alarm actions".

Adding logic components

To add logic components:

  1. On the left side of the grid, click on the Logic heading and drag a logic component to the grid area. See "Logic components".
  2. Connect the logic component to a conditional statement.
  3. If the logic component contains an operator, click on the dropdown arrow to select the operator you want to use.

For most types of logic components, you'll need to add math and curve placeholders to define the values for the logic statement. See "Adding math components". You can also nest additional logic components inside another logic component. See "Logic components".

Adding math components

To add math components:

  1. On the left side of the grid, click on the Math heading and drag any required math components to the grid area.
  2. Connect the math components to a logic component.
  3. If the math component contains an operator, click on the dropdown arrow to select the operator you want to use.

When the template is applied to an alarm, CENTRO® will use the math components and mnemonic components to determine if the conditional statement is true. If the conditional statement is true, the alarm will be triggered.

Configuring alarm actions

To configure the alarm actions:

  1. On the left side of the grid, click on the Alarm Actions heading and drag the action component to the grid area.
  2. Connect the action component to the rule.
  3. In the alarm component, click on the dropdown arrow to select the severity of the alarm.
  4. In the text box for the alarm condition, enter the message that should be displayed when the alarm is triggered.

CENTRO® will evaluate the conditions defined for each conditional statement in the If component. If a condition is true, the alarm will be triggered using the settings in the action component.

Logic components

The table below defines the different types of logic components.

Types of logic components
Component Type Definition
Adds an If component to the action portion of the primary If component.

Compares one value to another. Requires math components, mnemonic components, logic components, or a combination. Additional operators are available when you click on the dropdown arrow.

  • = – Returns true if the values are equal.
  • ≠ – Returns true if the values are not equal.
  • < – Returns true if the first value is less than the second value.
  • ≤ – Returns true if the first value is less than or equal to the second value.
  • > – Returns true if the first value is greater than the second value.
  • ≥ – Returns true if the first value is greater than or equal to the second value.

Compares one logic statement to another. Requires logic components. Additional operators are available when you click on the dropdown arrow.

  • And – Returns true if both logic statements are true.
  • Or – Returns true if one of the logic statements is true.

Returns the opposite of the attached logic statement. Requires logic components.

  • Returns true if the logic statement is false.
  • Returns false if the logic statement is true.
Returns either true or false. Can be attached to other logic components. Additional operators are available when you click on the dropdown arrow.
Returns null. Can be attached to other logic components.
Checks the logical statement attached to the Test parameter. If the condition is true, returns the value attached to the If True parameter. If the condition is false, returns the value attached to the If False parameter.

Math components

The table below defines the different types of math components.

Types of math components
Component Type Definition

Rounds a number up or down. The numeric component may be replaced with a math component or a WITSML mnemonic. Additional functions are available when you click on the drop down arrow.

  • Round – Rounds the number to the nearest whole number.
  • Round up – Rounds the number up to the nearest whole number.
  • Round down – Rounds the number down to the nearest whole number.
Sets a numeric value.

Returns the specified function for a number. The numeric component may be replaced with a math component or a WITSML mnemonic. Additional functions are available when you click on the drop down arrow.

  • Square root – Returns the square root of the number.
  • Absolute – Returns the absolute value of the number.
  • - – Returns the negation of the number.
  • ln – Returns the natural logarithm of the number.
  • log10 – Returns the base 10 logarithm of the number.
  • e^ – Returns e to the power of the number.
  • 10^ – Returns 10 to the power of the number.

Returns a value over a range of data points. Points is number of data points to consider in the range. Shift is the number of seconds ago that range will begin. Requires a numeric component or a WITSML mnemonic. Additional functions are available when you click on the drop down arrow.

  • Average – Returns the average value for the range.
  • Max – Returns the maximum value for the range.
  • Min – Returns the minimum value for the range.
  • Sum – Returns the sum of the range.

Returns the value of a trigonometric function for a degree. The numeric component may be replaced with a math component or a WITSML mnemonic. Additional functions are available when you click on the drop down arrow.

  • Sin – Returns the sine of a degree.
  • Cos – Returns the cosine of a degree.
  • Tan – Returns the tangent of a degree.
  • Asin – Returns the arcsine of a degree.
  • Acos – Returns the arccosine of a degree.
  • Atan – Returns the arctangent of a degree.

Returns the value of an irrational number. Additional numbers are available when you click on the drop down arrow.

  • π – Returns the value of pi.
  • e – Return's the value of Euler's number.
  • φ – Returns the value of the golden ratio.
  • sqrt(2) – Returns the value of the square root of 2.
  • sqrt(1/2) – Returns the value of the square root of 0.5.
  • ∞ – Returns the value of infinity.

Checks if a number is even, odd, prime, whole, positive, negative, or divisible by a certain number. The numeric component may be replaced with a math component or a WITSML mnemonic. Additional functions are available when you click on the drop down arrow.

  • Even – Returns true if the number is even.
  • Odd – Returns true if the number is odd.
  • Prime – Returns true if the number is a prime number.
  • Whole – Returns true if the number is a whole number.
  • Positive – Returns true if the number is a positive number.
  • Negative – Returns true if the number is a negative number.
  • Divisible by – Returns true if the number is divisible by the second numeric component.

Returns the sum, difference, product, quotient, or exponential value of two numbers. The numeric components may be replaced with a math component or a WITSML mnemonic. Additional functions are available when you click on the drop down arrow.

  • + – Returns the sum of the two numbers.
  • - – Returns the difference of the two numbers.
  • × – Returns the product of the two numbers.
  • ÷ – Returns the quotient of the two numbers.
  • ^ – Returns the value of the first number raised to the power of the second number.
Returns the remainder from dividing two numbers. The numeric components may be replaced with a math component or a WITSML mnemonic.
Constrains a number to be between the specified limits, inclusive. The numeric components may be replaced with a math component or a WITSML mnemonic.
Returns a random integer between the specified limits, inclusive. The numeric components may be replaced with a math component or a WITSML mnemonic.
Returns a random fraction between 0.0 (inclusive) and 0.1 (exclusive).
