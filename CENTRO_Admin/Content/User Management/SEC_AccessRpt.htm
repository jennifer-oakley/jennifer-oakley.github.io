About the access report

On the User Management > Access Report page, you can view the permissions granted to users and roles. To view the current permissions, select a user or role in the Role/User field at the top of the page.

The access report includes the following sections:

  • User List For Selected Role – Lists the users assigned to the selected role. Only available when a role is selected.
  • Role List For Selected User – Lists the roles assigned to the selected user. Only available when a user is selected.
  • Application Features – Lists the application privileges granted to the role or user. Application privileges determine which pages a user can access within CENTRO® Admin. See "List of application privileges".
  • WITSML Object Access – Lists the WITSML privileges granted to the role or user. WITSML privileges determine which wells, wellbores, and wellbore child objects a user can access on the WITSML server, as well as which actions the user can perform.
  • Document Store Access – Lists the document store privileges granted to the role or user. Document store privileges determine which wells a user can access on the document store, as well as which actions the user can perform.
  • Alarm Subscriptions – Lists the alarms that the role or user is subscribed to, as well as the notification settings for each alarm.
  • Document Store Subscriptions – Lists the document stores that the user is subscribed to. Only available when a user is selected.

When you select a user, the Derived Through column lists the roles used to grant each privilege. If a privilege is granted to the user account, instead of through a role, this will be noted by using the term Direct.
