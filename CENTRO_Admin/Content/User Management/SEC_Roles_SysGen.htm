About system-generated roles

There are two types of security roles that are automatically created by CENTRO®.

  • Well-specific roles are created when a well is integrated with CENTRO® Monitor. Well-specific roles limit access based on the well that the role is associated with. See "Well-specific roles".
  • Client-specific roles are created when a sign-up request for a user with a new client company is approved. All client-specific roles limit access based on the client company. See "Client-specific roles".

Well-specific roles

When a well is integrated with CENTRO® Monitor, CENTRO® will create the roles required to access the well in CENTRO® Monitor. To integrate a well with CENTRO® Monitor, go to the Wells > Well Management page and click the Create Project/Groups button () next to the well name. See "CENTRO® Monitor well integration".

When you begin the integration process, three roles will be created in CENTRO® Admin:

  • [RigName]_[WellName]_ReadOnly – Grants read-only access to CENTRO® Monitor, as well as read and list access to the WITSML data store for the well.
  • [RigName]_[WellName]_NormalUser – Grants normal user access to CENTRO® Monitor, as well as read and list to the WITSML data store for the well.
  • [RigName]_[WellName]_Admin – Grants admin access to CENTRO® Monitor, as well as create, update, read, and list access to the WITSML data store for the well.

The ACLs and WITSML ACLs required for each access level will be automatically assigned to the appropriate role. See "Security Access Control Lists" and "WITSML Access Control Lists".

Client-specific roles

When a sign-up request for a user with a new client company is approved, several client-specific roles will be created.

  • The [ClientName]_Administrator role grants access to CENTRO® Admin and administrator access to CENTRO® Monitor. In CENTRO® Admin, client administrators have access to manage data hubs, wells, documents, WITSML data, and user accounts.
  • The [ClientName]_ReadWrite role grants read/write access to WITSML data in CENTRO® Admin, as well as read/write access to CENTRO® Monitor.
  • The [ClientName]_ReadOnly role grants read only access to WITSML data in CENTRO® Admin, as well as read only access to CENTRO® Monitor.

All client-specific roles limit access based on the client company. Users who have a client-specific role assigned to their account will only be able to view data for their client company, such as data hubs, wells, documents, WITSML data, users, and roles.

Note

If you have a client administrator account, you will only be able to view and manage roles that are associated with the same client company.
