About Azure B2C accounts

Azure AD B2C is a cloud-based identity management system that supports multi-factor authentication and single sign-on for several Weatherford applications, including CENTRO®, CENTRO® Engineer, and Surveyors Workbench.

On the User Management > Users page, you can add an Azure AD B2C account to grant access to the data center. There are two types on accounts for data centers with Azure B2C enabled:

  • Local users – Local user accounts are administered by the CENTRO® administrator for the data center. Users will be able to log in with their assigned credentials for CENTRO®. If the user does not exist in Azure B2C, you will need to create a local user account. See "Azure B2C user accounts".
  • External users – External user accounts are created in Azure B2C by a domain administrator. External users will be able to log in to CENTRO® using the Weatherford button on the log-in screen, without entering their log-in credentials. If the user does exist in Azure B2C, you will need to create an external user account. See "Azure B2C user accounts".
Note

This feature is only available for data centers that have Azure B2C user authentication enabled. Azure B2C is not enabled by default. To enable Azure B2C user authentication, refer to the Server-Side Configuration chapter of the CENTRO® Admin Install Guide.

Creating local accounts

To create a new local user account:

  1. Go to the User Management > Users page.
  2. Click the Add User button () at the top of the page.
  3. Enter the user's email address, then click the Search button. Information from the selected AD account will automatically appear below the search results.
  4. Select the Local User option.
  5. Enter the new user's name and contact information.
  6. In the Notification field, select the number that text notifications for alarms should be sent to.See "Alarm management".
  7. Enter a temporary password in the Password and Confirm Password fields.
  8. Select the Approved option to activate the account.
  9. Assign the desired roles to the new account. See "Assigning roles".
  10. Click the Save button.

The new user account will be created, with the permissions granted by the roles that you assigned to the account. If you selected the Must Change Password option, they will be prompted to change their password when they log in for the first time. If you selected the Send Welcome Email option, a welcome email will be automatically sent to the new user with the CENTRO® URL, user name, and password.

Creating external user accounts

If the user already exists in the Azure B2C directory, CENTRO® will automatically look up the user's account information in the Azure B2C directory.

  1. Go to the User Management > Users page.
  2. Click the Add User button () at the top of the page.
  3. Enter the user's email address, then click the Search button. Information from the selected AD account will automatically appear below the search results.
  4. Enter any missing information, such as their contact information.
  5. Assign the desired roles to the new account. See "Assigning roles".
  6. Click the Save button.

The B2C account will be added to CENTRO®, with the permissions granted by the roles that you assigned to the account. They will be able to log in to CENTRO® using the Weatherford button on the log-in screen, without entering their log-in credentials.

Assigning roles

To assign a role to a user account:

  1. Go to the User Management > Users page.
  2. Click the Edit button () for the user account that you want to grant access to.
  3. In the window that appears, go to the Roles table at the bottom of the window.
  4. Select the roles you want to assign in the Available Roles list, then click the Add button ().
  5. Click the Save button at the bottom of the window.

The Selected Roles list will be updated with the assigned roles. The user account will be granted all of the privileges assigned to the selected roles. If the user is currently logged in, they will need to log out and log back in.

Tip

To select multiple roles at the same time, press the Ctrl key and select the roles you want to add.

Field definitions

The table below defines all of the fields in the User Details window, which appears when you add or edit Azure B2C users on the User Management > Users page.

Field definitions for Azure B2C users on the User Management > Users page
Field Definition
Username Log-in ID for the account. This field is read only and can't be edited for external users.
First Name First name of the user. This field is read only and can't be edited for external users.
Last Name Last name of the user. This field is read only and can't be edited for external users.
Mobile Cell phone number for the user. This field is read only and can't be edited for external users.
Office Office phone number for the user. This field is read only and can't be edited for external users.
Email Email address of the user. This field is read only and can't be edited for external users. If the user is subscribed to email notifications for any alarms, email notifications will be sent to this email address.See "Subscribing to an alarm".
Notification Phone number to use for text notifications. If the user is subscribed to text notifications for any alarms, text notifications will be sent to this phone number. If an option isn't selected, text notifications will not be sent.See "Subscribing to an alarm".
Designation Job title or role on the project
Client Client company the user is affiliated with. If a client company is assigned to the user account, the user will only be able to view wells and documents associated with the same client company. See "Well management" and "Document store".
Fax Fax number for the user
Approved Indicates the user is approved for access to the current data center or data hub. If this option is not selected, the user will not be able to log in.
Must Change Password Indicates the user will be required to change their password the next time they log in. Only available for local user accounts.
Send Welcome Email Indicates that a welcome email will be automatically sent to the new user. The welcome email will include the CENTRO® URL, user name, and password.
