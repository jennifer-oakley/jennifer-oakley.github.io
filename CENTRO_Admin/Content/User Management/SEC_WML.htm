About WITSML ACLs

On the User Management > WITSML ACLs page, you can view and update the privileges for the wells, wellbores, and wellbore child objects on the WITSML server. WITSML privileges determine which WITSML objects a user can access on the WITSML server, as well as which actions the user can perform. You can assign WITSML privileges to individual wells and child objects, as well as all wells on the WITSML server. See "Permission levels".

The access level for each WITSML privilege may be directly assigned to a user account, or it may be inherited from a security role. When the privileges assigned to a security role are updated, the privileges for all users with that security role will also be updated.

There are five levels of access that you can grant to all wells, an individual well, or a child object. See "Permission levels".

  • Create – Allows the user to add WITSML objects.
  • Read – Allows the user to build WITSML queries and view the properties of WITSML objects.
  • Update – Allows the user to update existing WITSML objects.
  • Delete – Allows the user to delete WITSML objects.
  • List – Allows the user view the well in the WITSML Explorer. All roles and users that need access to an individual well or any of the child objects for the well must have this permission level assigned to them.

These privileges must be assigned to each well that the user needs access to. When you grant access to a well, the permissions will be automatically inherited by all wellbores and child objects within the well. However, the inherited permission can be overridden by assigning permissions directly to the wellbore or wellbore child object.

All roles and users must have the List privilege assigned. If the List privilege is not assigned, the user will not be able to access the well.

Note

If a client company is assigned to your account, you will only be able to view the WITSML ACLs for wells that are associated with the same client company. See "User accounts".

Note

The permissions granted to a role on the WITSML ACLs page will be inherited by all users that role is assigned to. However, the inherited permissions are not displayed.

Granting access to wells

To grant access to a well:

  1. Go to the User Management > WITSML ACLs page.
  2. In the list of wells on the left, select the well that you want to grant access to.
  3. In the Permissions table, select the List option for each role or user that you want to have access to the well.
  4. Select additional privileges as needed, then click the Save button ().

If you granted access to a security role, the new privileges will be granted to all users with the selected security role. However, these changes will only be applied to the server where the changes were made. If you granted access to a user account, the account will be granted all of the selected privileges. If the user is currently logged in, they will need to log out and log back in.

Applying filters

The Apply Filter option at the top of the Permissions table allows you to filter the list of roles and users, based on the WITSML privileges assigned to them.

To apply filters to the Permissions table:

  1. Go to the User Management > WITSML ACLs page.
  2. In the list of wells on the left, select the well that you want to view privileges for.
  3. In the Permissions table, select the Apply Filter option at the top of the table.
  4. In the same row, select the privileges you want to view roles and users for.

The list will be automatically filtered to only show roles and users with the selected privileges to the well. If you selected multiple permissions, only roles and users with all of the selected permissions will be shown.

Permission levels

The table below defines the permission levels for all wells, individual wells, and the child objects in a well

Permission levels for WITSML ACLs
Permission Level All Wells Individual Well Child Object
Create
  • Grants access to create new wells using CENTRO® or any WITSML client. When the user creates a well, they will automatically have create, read, update, and list access to the child objects for the well.
  • Does not grant any access to wells created by other users.
  • Does not grant delete access to wells created by the user. Delete access must be granted separately for each individual well.
  • Grants access to create child objects using CENTRO® or any WITSML client.
 n/a
Read
  • Grants access to view all wells and the well information on the WITSML server using CENTRO® or any WITSML client.
  • Does not grant access to view child objects for any well.
  • Grants access to view the well and the well information on the WITSML server using CENTRO® or any WITSML client.
  • Does not grant access to view child objects for the well.
  • Grants access to view the child object using CENTRO® or any WITSML client.
Update n/a
  • Grants access to update the properties of the well using CENTRO® or any WITSML client.
  • Does not grant access to update child objects for the well.
  • Grants access to update the properties of the child object using CENTRO® or any WITSML client.
Delete n/a
  • Grants access to delete the well using CENTRO® or any WITSML client.
  • Does not grant access to delete individual child objects for the well.
  • Grants access to delete the child object using CENTRO® or any WITSML client.
List n/a
  • Grants access to view the well in the WITSML Explorer.
  • Only grants access to view the well name and well ID.
 n/a

Using the WITSML ACLs page

The search box above the table on the WITSML ACLs page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.

The buttons on the User Management > WITSML ACLs page are used to save and undo changes.
