About user accounts

User accounts are used to access the servers in the Weatherford Real-Time System. The User Management > Users page lists all system accounts and user accounts for CENTRO® Admin. There are several ways to grant access to CENTRO®:

  • Local user accounts are created on the User Management > Users page. Local user accounts are administered by the CENTRO® administrator for the data center or data hub. See "Local user accounts".
  • Active Directory (AD) accounts are created on a local domain by a domain administrator. On the User Management > Users page, you can add an AD account to grant access to the data center. See "AD user accounts".
  • Azure AD accounts are cloud-based AD accounts, created by a domain administrator. On the User Management > Users page, you can add an Azure AD account to grant access to the data center. See "Azure AD user accounts".
  • Azure AD B2C is a cloud-based identity management system that supports multi-factor authentication and single sign-on. On the User Management > Users page, you can add an Azure AD B2C account to grant access to the data center. See "Azure B2C user accounts".
  • Sign-up requests are submitted from the CENTRO® Admin log-in page. After a sign-up request is approved by a system administrator, a user account will be automatically created for the new user with the client-specific administrator role assigned to it. See "Sign-up requests".

For users who will be managing well data, the recommended best practice is to set up the security settings on the data hub. Well data is only replicated from the data hub to the data center, so any changes that a user makes to a well on the data center would not get replicated to the data hub.

Note

If you have a client administrator account, you will only be able to view and manage user accounts that are associated with the same client company.

Note

User accounts are not synchronized between the data center and the data hubs. When you create a new account, it is only created on the selected server.

System accounts

There are two built-in system accounts:

  • admin – This account is the security and well administration account for each server. This account has access to manage users, roles, privileges, vendors, clients, and wells.
  • ServiceAccount – This account is used by CENTRO® Admin to write incoming WITS records to the database.

The built-in system accounts cannot be modified or deleted. These accounts are not intended to be used by individuals. The recommended best practice is to create a user account for the system administrator, which can be used to manage the users and wells for CENTRO® Admin.

Note

If the Users privilege is not granted to you on the User Management > ACLs page, the Users page will not be visible. Contact your administrator to request access to this page.

Using the Users page

The search box above the table on the Users page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.

The buttons on the User Management > Users page are used to add, edit, and delete users, as well as reset passwords.

When you add or edit or a user account, the buttons in the User Details window allow you to add and remove roles from the user account.
