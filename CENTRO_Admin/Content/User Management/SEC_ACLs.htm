About access control lists

On the User Management > ACLs page, you can view and update the security privileges assigned to user accounts and security roles. Application privileges determine which pages a user can access within CENTRO® Admin.

The access level for each privilege may be directly assigned to a user account, or it may be inherited from a security role. When the privileges assigned to a security role are updated, the privileges for all users with that security role will also be updated. The icons next to each privilege indicates the level of access for that privilege. For most privileges, there are three levels of access:

  • None – No access granted. The specified page will not be visible to users with this access level. This access level is indicated by the No Access icon ().
  • Read Only – Grants read-only access to the privilege. The specified page will be visible to users with this access level, but they will not be able to make any changes. This level of access is indicated by the Some Access icon ().
  • Full Access – Grants full access to the privilege. Users with this access level will be add, edit, and delete items on the specified page. This level of access is indicated by the Full Access icon ()

When you hover your mouse over the icon, you will see additional information about the access level for the privilege. The privileges assigned to a default system role or account cannot be modified. However, you can update the privileges assigned to user accounts and custom security roles. See "Assigning privileges to roles and accounts".

Assigning privileges to roles and accounts

To assign privileges to a security role or user account:

  1. Go to the User Management > ACLs page.
  2. In the Role/ User field at the top of the page, select the role or account that you want to assign privileges to.
  3. In the table that appears, use the sliders to set the access level for each privilege. See "List of privileges".
  4. Click the Save button () at the top of the screen.

If you selected a security role, the new privileges will be granted to all users with the selected security role. However, these changes will only be applied to the server where the changes were made. If you selected a user account, the account will be granted all of the selected privileges. If the user is currently logged in, they will need to log out and log back in.
