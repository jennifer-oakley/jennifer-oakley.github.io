About security roles

Security roles are used to grant access to users, based on the user's responsibilities within CENTRO® Admin. Privileges are assigned to each security role. The User Management > Roles page lists all of the system-created and custom security roles for the selected server. An administrator role is created automatically for CENTRO® Admin. However, the privileges assigned to this role cannot be modified. If you require more specific security roles, you can create custom roles. See "Adding security roles".

For roles related to managing well data, the recommended best practice is to set up the security settings on the data hub. Well data is only replicated from the data hub to the data center, so any changes that a user makes to a well on the data center would not get replicated to the data hub.

Note

Roles are not synchronized between the data center and the data hubs. When you create a new role, it is only created on the selected server.

Adding security roles

To add a security role:

  1. Go to the User Management > Roles page.
  2. Click the Add button () at the top of the page.
  3. In the window that appears, enter a name for the new role.
  4. In the Client field, select the name of the client company that the role is for.
  5. In the Users table, add the users that you want to assign to the role. See "Security Roles".
  6. Click the Save button.

The new role will appear in the lists of roles on the ACLs page. If a client company is associated with the role, the users assigned to the role will only be able to view the roles associated with that client company. By default, no privileges will be assigned to the new role. You can assign privileges to the role on the ACLs page. See "Security Access Control Lists".

Using the Roles page

The search box above the table on the Roles page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.

The buttons on the Roles page are used to add and delete roles.

Buttons on the User Management > Roles page
Button Function
Adds a new security role to the server
Indicates that the role is active
Indicates that the role is inactive
Allows you to edit the selected role. System roles cannot be edited.
Deletes the selected role from the server. System roles cannot be deleted.

When you add or edit or a role, the buttons in the Role Details window allow you to add and remove users and groups from the role.

Buttons in the Role Details window
Button Function
Indicates that the account is a system account
Indicates that the account is an Active Directory account
Adds the selected user account to the role
Adds all available user accounts to the role
Removes the selected user account from the role
Removes all user accounts from the role
