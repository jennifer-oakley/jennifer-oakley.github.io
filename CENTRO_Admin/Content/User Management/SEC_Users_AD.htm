About AD accounts

AD accounts are created in Active Directory by a domain administrator. On the User Management > Users page, you can add an AD account to grant access to the data center.

AD users will be able to log in to CENTRO® using the Corporate Login button on the log-in screen, without entering their log-in credentials. If they have access to CENTRO® Monitor, they will also be automatically logged in when they use the direct URL for CENTRO® Monitor. When you add an AD user, you'll also need to assign the appropriate roles to the account. No permissions are granted to new accounts by default, so the new user will not be able to access anything on the server until the access has been granted.

Note

This feature is only available for data centers that have AD user authentication enabled. Active Directory is not enabled by default. To enable AD user authentication, refer to the Server-Side Configuration chapter of the CENTRO® Admin Install Guide.

Adding AD accounts

To add an Active Directory account:

  1. Go to the User Management > Users page.
  2. Click the Add User button () at the top of the page.
  3. In the window that appears, select the AD User option.
  4. Use the search fields at the top to search for the account, then click the Search button.
  5. In the search results that appear, click on the user account that you want to add. Information from the selected AD account will automatically appear below the search results.
  6. Assign the desired roles to the new account. See "Assigning roles to user accounts".
  7. Click the Save button.

The AD account will be added to CENTRO®, with the permissions granted by the roles that you assigned to the account. They will be able to log in to CENTRO® using the Corporate Login button on the log-in screen, without entering their log-in credentials. If they have access to CENTRO® Monitor, they will also be automatically logged in when they use the direct URL for CENTRO® Monitor. See "Security Access Control Lists".

Assigning roles to user accounts

If you are a client administrator, you are required to assign one of the client-specific roles when you create a new user account. Only one other account can have the client administrator role assigned to it. You can assign the other client roles to multiple user accounts, up to the number of licenses assigned to your account.

To assign a role to a user account:

  1. Go to the User Management > Users page.
  2. Click the Edit button () for the user account that you want to grant access to.
  3. In the window that appears, go to the Roles table at the bottom of the window.
  4. Select the roles you want to assign in the Available Roles list, then click the Add button ().
  5. Click the Save button at the bottom of the window.

The Selected Roles list will be updated with the assigned roles. The user account will be granted all of the privileges assigned to the selected roles. If the user is currently logged in, they will need to log out and log back in.

Tip

To select multiple roles at the same time, press the Ctrl key and select the roles you want to add.

Field definitions

The table below defines all of the fields in the User Details window, which appears when you add or edit AD users on the User Management > Users page.

Field definitions for AD users on the User Management > Users page
Field Definition
Username Log-in ID for the account. This field is read only and can't be edited.
First Name First name of the user. This field is read only and can't be edited.
Last Name Last name of the user. This field is read only and can't be edited.
Mobile Cell phone number for the user. This field is read only and can't be edited.
Office Office phone number for the user. This field is read only and can't be edited.
Email Email address of the user. This field is read only and can't be edited. If the user is subscribed to email notifications for any alarms, email notifications will be sent to this email address.See "Subscribing to an alarm".
Designation Job title or role on the project
Client Client company the user is affiliated with. If a client company is assigned to the user account, the user will only be able to view wells and documents associated with the same client company. See "Well management" and "Document store".
Send Welcome Email Indicates that a welcome email will be automatically sent to the new user. The welcome email will include the CENTRO® URL, user name, and password.
