About predictions

On the Predict Models page, you can configure an algorithm to optimize the rate of penetration. optimized ROP is based on several drilling parameters, such as bit depth, WOB, flow rate, and RPM. The algorithm uses the last five minutes of data to predict the ROP value at the last point.

Note

The Predict Models page only lists calculations for the wells that you have been granted access to.

Configuring ROP optimization

To configure an ROP optimization calculation:

  1. Go to the Predict Models page.
  2. Click the Add New button () at the top of the page, then select the ROP Optimization option.
  3. In the Well Name field, select the well that you want to configure calculations for.
  4. In the Wellbore field, select the name of the wellbore.
  5. In the Log field, select the logs that contains the drilling parameters for the well.
  6. If you selected multiple logs, click the Co-relate Logs button () specify the primary log. See "Mapping multiple logs".
  7. Go to the Input Parameters section to configure the expected values for RPM, flow rate, and WOB. See "Configuring drilling parameters".
  8. Go to the Mapping for Model Parameters with Log Mnemonics table and select the log, WITSML mnemonic, and unit for each parameter.
  9. Click the Save Prediction button () to save your changes, or click the Start button () to begin calculations.

When you start calculations, a new WITSML log will be created for the optimization results. The new log will have the same name as the drilling parameters log, with _Opt_ROP added to the end of the name, e.g. [DrillingParametersLogName]_Opt_ROP. The Predict Models page will show the current status of the calculation, the time that the drilling parameters log started receiving data, and the time of the last calculation.

Tip

To manually change the start time, click the calendar button () in the Start Time field. The selected value must be greater than or equal to the start time of the log.

Mapping multiple logs

If you want to use data from multiple logs for ROP optimization, you'll need to specify the primary log for the calculation. The primary log must be a time-based log, and additional logs may be either time-based or depth-based.

  1. Click the Co-relate Logs button () to open the Map Primary Logs with Additional Logs window.
  2. In the Primary Log field, select the name of the primary log you want to use for ROP optimization.
  3. For each additional log, verify the index curve selected in the Additional Log Curve field.
  4. Click the OK button to continue configuring the ROP optimization calculation.

When the calculation begins, the index curve for the primary log will be used to align the index curves for the secondary logs.

Configuring drilling parameters

In the Input Parameters section, you can specify the expected values for RPM, flow rate, and ROP. These parameters are required and must be configured. If desired, you can also enter the expected values for bit depth and gamma ray.

To specify the expected value for each parameter:

  1. Go to the Parameter Type field and select the type of expected value.
    • Fixed – The parameter has a known value that will remain constant throughout the calculation.
    • Optimize – The parameter has a range of acceptable values. The Optimize option must be selected for at least one parameter.
  2. In the Value field, enter the expected value for the parameter. If you selected the Optimize option, enter the minimum and maximum values for the parameter.
  3. In the WITSML Unit field, select the measurement unit for the parameter.

The values entered for each parameter will be used in the ROP optimization calculations. These values will override the values received from the WITSML log. To add expected values for bit depth and gamma ray, click the Add Parameter button ().

Note

The input parameters cannot be edited while the calculation is running. To edit the parameters, click the Stop button () to stop the calculation.

Using the Predict Models page

The search box above the table on the Predict Models page allows you to filter the rows displayed. To view additional rows of data, click on the page numbers or arrows at the bottom of the table. To change the number of rows shown on each page, use the dropdown menu next to the page numbers.

The buttons on the Predict Models page are used to add, start, stop, and delete optimization calculations.

Buttons on the Predict Models page
Button Function
Adds a new optimization calculation. See "Predictions".
Starts the corresponding optimization calculation

Allows you to view and edit the corresponding optimization calculation
Deletes the corresponding optimization calculation. If the calculation is currently running, you must stop the calculation, before you can delete it.

When you add or edit a optimization calculation, the buttons on the Edit Predict Models page allow you to start and stop the calculation.
