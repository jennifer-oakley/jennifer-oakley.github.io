Surveyor's Workbench v2.7 Release Notes

In this article

What's new

Bugs fixed

Welcome to Surveyor's Workbench v2.7. This version of Surveyor's Workbench contains several new features and enhancements, as well as some big fixes. For a full list of the updates in v2.7, refer to the sections below.

What's new

The features in this section are new for Surveyor's Workbench v2.7. For additional information about any of these features, please refer to the Surveyor's Workbench user guide. To open the user guide, go to the Help menu at the top of the screen and select the User Manual option.

  • Surveyor's Workbench will now run on Windows 11. Previously, the application would install, but did not open on Windows 11 systems.
  • Added a new user guide to replace the previous RTOC field instruction. The new user guide includes a quick start guide, interactive demos, and detailed information about many features in Surveyor's Workbench. To open the new user guide, go to the Help menu and select the User Manual option.
  • Created a web-based application called SWB Orchestrator, which allows the RTOC and the rig to easily share surveys and corrections. Rig engineers upload the raw survey data and FMS files to SWB Orchestrator, which can be downloaded by the RTOC. The RTOC uses the raw survey data and FMS files uploaded by rig engineers to perform MSA analysis and sag correction, then uploads the corrections to SWB Orchestrator.
  • Added the ability to connect SWB Orchestrator projects to Surveyor's Workbench. To open a run created in SWB Orchestrator, go to the File menu in Surveyor's Workbench, then select the New Orchestrator Project option. You will be prompted to enter the project ID and the URL for the RTOC. You can only create an SWB Orchestrator project if you are an RTOC user. See "Demo: Creating Orchestrator projects".
  • When you open an SWB Orchestrator project in Surveyor's Workbench, you can automatically load the latest survey uploaded from the rig. Go to the Inputs screen and click the Load File button to load the data from the rig, which is stored on the SWB file share. You must be connected to the Weatherford network to access the SWB file share.
  • Added the ability to create multi-lateral SWB Orchestrator projects in Surveyor's Workbench. To create a multi-lateral project, create a new SWB Orchestrator project and load the survey data. In SWB Orchestrator, create a new job for the sidetrack well. Then, go to the File menu in Surveyor's Workbench, then select the New Multi Lateral Project option. When prompted, enter the new project ID.
  • For SWB Orchestrator projects, you can review the corrections on the Reports > Corrections tab. The Corrections tab shows the previous, available, and new corrections. The Previous Corrections table shows the corrections that have already been sent to the rig, and the Available Corrections table shows the new and updated corrections that can be sent to the rig. The New Corrections table shows the corrections that will be sent the next time corrections are sent. To send the new corrections to the rig, click the Send to Rig button at the bottom of the screen. See "Demo: Sending corrections".
  • Added the ability to copy the raw survey data and magnetic parameters from an existing project to a new project. Open the existing project in Surveyor's Workbench. Then, go to the File menu in Surveyor's Workbench, then select the Save Project As option. See "Demo: Adding and closing runs".
  • When AutoPilot mode is enabled, the MSA threshold alarm will only trigger when the threshold has been exceeded for both raw data and FMS data. Previously, the threshold alarm did not consider FMS data, resulting in erroneous alarms. To set the thresholds for raw data, go to the Settings menu, then go to the MSA: Raw section in the Settings Editor window.
  • On the MSA > Optimisation tab, only one set of short collar readings will now be shown in the Combined Data, Corrected Data, and Corrected Survey views. When you import survey data from Spectrum, Surveyor's Workbench will automatically determine whether AES1 or AES2 readings should be used.
  • Added validations to check for surveys that are in the no-go zone. For raw survey data, a warning message will appear in the banner at the top of the screen. For planned survey data, the surveys in the no-go zone will be highlighted in red. Any survey that exceeds the inclination or azimuth limits is considered to be in the no-go zone. To change the inclination and azimuth limits, go to the Settings menu, then go to the No Go Zone section in the Settings Editor window.
    • The default inclination limit for all survey types is 70 degrees.
    • For non-IFR surveys, the default azimuth limit is ± 30 degrees from east or west.
    • For IFR surveys, the default azimuth limit is ± 25 degrees from east or west.
  • Added the ability to run a sag analysis with azimuth corrections calculated using the short collar readings. The short collar readings are used to adjust the Z-axis magnetometer reading by calculating the expected clean reading for each axis and define the amount of azimuth error. To enable sag analysis with AES corrections, go to the Inputs screen and select the Sag + AES option for the analysis type.
  • Added a field for entering the run number to the Job Details section on the Inputs screen.
  • Added the ability to select which surveys to include in sag analysis. For sag only projects, all surveys will be selected by default. For MSA with sag projects, all surveys included in MSA optimization will be automatically selected.

Bugs fixed

The critical issues in this section have been fixed in Surveyor's Workbench v2.7.

  • When you create a copy of a project using the File > Save Project As option, some settings should no longer be reset to their default values. Previously, affected settings included MSA/FMS thresholds, scale factors, conversion factors, units, and SAG + MSA + IFR1 Assembly parameters. WISE 1105755.
