About runs

The table on the Dashboard page shows the status of the active runs, as well as other information about each run. The status of an active run is updated automatically when rig users are assigned to the job, survey data is received, or correction files are sent.

  • Under Setup – The job has been created, but there are no active runs for the job, or no rig users have been assigned to the job.
  • Waiting For Surveys – The rig users have been assigned to the job, but no survey data has been received. Only a rig engineer can upload data from the rig.
  • SWB Processing – The RTOC has received the surveys and is analyzing the data.
  • Survey Received – The raw survey data was uploaded and attached to the run by a rig user. When a rig user uploads survey data, the run status will be automatically updated.
  • Correction Sent – The correction file was uploaded and attached to the run by an RTOC user. When an RTOC user uploads corrections for the run, the run status will be automatically updated. See "Uploading corrections".

All files uploaded for a run are stored in the SWB file share. A folder will be created for the job, using the job number for the folder name. Each run will have its own folder within the folder for the job. When a new run is created, all of the survey and FMS files for the previous run will be copied the new run folder.

Connecting runs to Surveyor's Workbench

After the raw survey data is received, you'll need to connect the run to Surveyor's Workbench. After you load the raw survey data into Surveyor's Workbench, the run status will be automatically updated in SWB Orchestrator.

To connect a SWB Orchestrator project to Surveyor's Workbench:

  1. In SWB Orchestrator, go to the Dashboard page and copy the project ID for the run.
  2. In Surveyor's Workbench, go to File > New Orchestrator Project.
  3. In the New Orchestrator Project window that opens, enter a name for the project in the SWB Project Name field.
  4. In the Project ID field, paste the project number that you copied from SWB Orchestrator.
  5. In the Orchestrator URL field, select the URL for the RTOC.
  6. Click the OK button.
  7. In the window that opens, sign in with your RTOC account.

The Orchestrator project will open in Surveyor's Workbench. If you have the SWB file share mapped, click the Load File button to load the most recently uploaded survey. Otherwise, you can manually load the survey file. See "Downloading files".

Note

You must be connected to the Weatherford network in to connect to SWB Orchestrator through Surveyor's Workbench.

Changing the run mode

A run is a drilling section of the well and the equivalent of an Surveyor's Workbench project. In addition to the run status, there are also two run modes:

  • In Calibration mode, the RTOC is gathering data and waiting for a sufficient number surveys to run MSA analysis and sag corrections. When you load a survey file from the run into Surveyor's Workbench, the run status will reset to Waiting For Surveys.
  • In Real-time mode, the RTOC has sufficient survey data to provide survey corrections. When you load a survey file from the run into Surveyor's Workbench, the run status will change to SWB Processing.

The default mode for all new runs is Calibration mode. To change the run mode to Real-time mode, the run mode must be changed in both Surveyor's Workbench and SWB Orchestrator.

To change the run mode:

  1. In Surveyor's Workbench, click the Calibration button at the top of the screen. The button text will change to Real Time.
  2. In SWB Orchestrator, go to the Dashboard page and click the Edit Job button () in the row for the run.
  3. Locate the run you want to edit in the Add Runs Details section. The active run will be highlighted in green.
  4. Go to the Run Mode field and select the Real-time option.
  5. Click the Save button to save your changes.

The run mode will be changed to Real-time mode. The next time you load a survey into Surveyor's Workbench, the run status will change to SWB Processing.

Adding runs to a job

To add a run to an existing job:

  1. Go to the Dashboard page and click the Edit Job button () in the row for the active run.
  2. Go to the Add Runs Details section and enter the run number and the size of the open hole section.
  3. Click the Add Run button.
  4. Click the Save button to save your changes and create the run.
  5. Open the Surveyor's Workbench project for the previous run.
  6. Go to the File menu and select the Save Project As option.
  7. In the Save As window, select the Save as new orchestrator with new id but same reference data option, then click OK.
  8. In the Cloned Orchestrator Project window that opens, enter a name for the project in the SWB Project Name field.
  9. In the Project ID field, enter the project ID for the new run.
  10. In the Orchestrator URL field, select the URL for the RTOC.
  11. Click the OK button.
  12. In the Save Project window that opens, select the location where you want to save the project and the report, then click OK.

In SWB Orchestrator, the new run will be added to the list of runs in the Edit Job & Run window and the Dashboard page. The run status will be set to Waiting For Surveys until the raw survey data is uploaded, and the run mode will be set to Calibration mode. All of the survey and FMS files for the previous run will be copied to the run folder on the SWB file share.

In Surveyor's Workbench, the survey data will be automatically loaded in the project for the new run. The magnetic parameters from the project for the previous run will be copied to the new project. To add the FMS data, go to the MSA > FMS tab.

Closing a run

To close a run:

  1. Go to the Dashboard page and click the Edit Job button () in the row for the active run.
  2. Locate the run you want to edit in the Add Runs Details section. The active run will be highlighted in green.
  3. Go to the Run Status field and select the Closed option.
  4. Click the Save button to save your changes and close the run.

The run will be moved from the Dashboard page to the Archives page. The run will also be removed from the rig dashboard for the users assigned to the job. The run status will be changed to Run Closed. If there are no more active runs for the job, the job will still be listed on the Dashboard page, but without any active runs. See "Adding runs to a job" and See "Archiving a job".
