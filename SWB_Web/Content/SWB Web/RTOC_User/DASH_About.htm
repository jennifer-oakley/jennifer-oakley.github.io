About the dashboard

The Dashboard page lists all of the active runs for the RTOC center. From the Dashboard page, you can create new jobs, add runs to a job, assign rig users to a run, and upload data to a run. A job contains all of the runs for a well. A run is a drilling section of the well and the equivalent of an Surveyor's Workbench project. See "Jobs" and "Runs".

Switching the RTOC

To switch to another RTOC:

  1. On the Dashboard page, click the Change Center button at the top of the screen.
  2. Click Yes on the confirmation message that appears.
  3. Locate the RTOC you want to switch to, then click the Go To Center button () in the row for the RTOC.

The Dashboard page will reload with the active runs for the selected RTOC center.

Field definitions

The table on the Dashboard page shows the status of the active runs, as well as other information about each run.

Definitions for the Dashboard page
Column Name Definition
Project # Unique project ID, which is used to connect the run to Surveyor's Workbench. See "Connecting runs to Surveyor's Workbench".
Job # Any name to identify the project, such as the Weatherford file number
Customer Name of the client company
Rig Name Name of the rig at the well site
Well Name Unique identifier for the well. The well name should match the well name in Spectrum, if available.
Run # Unique identifier for a drilling section of the well
Status

Status of the run

  • Under Setup – The job has been created, but there are no active runs for the job, or no rig users have been assigned to the job.
  • Waiting For Surveys – The rig users have been assigned to the job, but no survey data has been received. Only a rig engineer can upload data from the rig.
  • SWB Processing – The RTOC has received the surveys and is analyzing the data.
  • Survey Received – The raw survey data was uploaded and attached to the run by a rig user. When a rig user uploads survey data, the run status will be automatically updated.
  • Correction Sent – The correction file was uploaded and attached to the run by an RTOC user. When an RTOC user uploads corrections for the run, the run status will be automatically updated. See "Uploading corrections".
Last Survey Received Date and time that the last survey was received by the RTOC from the rig
Last Survey Sent Date and time that the last set of corrections was shared with the rig
Last Survey Depth Depth of the deepest correction sent

Using the Dashboard page

The toolbar in the Action column allows you to manage the job and the runs associated with the job.

Toolbar buttons on the Dashboard page
Button Function
Opens the chat conversation for the run. See "Chat".
Opens the Rig User Details window, which lists all of the rig users assigned to the job. You can also assign users to the job and create new users from this window. See "Adding users to a job".
Opens the Edit Job & Run window, which allows you to edit the job and run details. See "Jobs" and "Runs".

Opens a menu with additional actions

  • Download RAW – Downloads a text file with the raw sensor data. See "Downloading files".
  • Download FMS – Downloads any FMS files that have been uploaded to the job. See "Downloading files".
  • Upload Corrections – Opens the Upload Corrections window, which allows you to upload survey corrections in a CSV file. See "Uploading corrections".
  • View Log – Opens the Run Log window, which shows a list of the status changes for the job.
