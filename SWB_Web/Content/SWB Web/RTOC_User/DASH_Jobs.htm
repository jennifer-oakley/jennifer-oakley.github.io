About jobs

There are several ways to manage jobs:

  • Create a job. When a new well is being drilled, you'll need to create a new job in SWB Orchestrator. A job contains all of the runs for a well. See "Creating jobs".
  • Assign rig users to the job. The rig users assigned to the job will be able to view the active run on the rig dashboard. See "Adding users to a job".
  • Archive the job. When the job has been completed and all active runs are closed, you can archive the job. See "Archiving a job".

Creating jobs

To create a new job:

  1. Go to the Dashboard page and click the Create Job button at the top of the screen.
  2. In the Create Job & Run window that opens, enter the job details under the Add Job section.
  3. In the Add Run Details section, enter the run number and the size of the open hole section.
  4. Click the Add Run button.
  5. Click the Save & Next button.
  6. On the Assign Rig User(s) screen, go to the Users field and select the users you want to assign to the job.
  7. Click the Save button to save your changes and create the job.

The new job will be created, and the first run will be listed on the Dashboards page. The run status will be set to Waiting For Surveys until the raw survey data is uploaded. After the raw survey data is received, you'll need to connect the run to Surveyor's Workbench. See "Connecting runs to Surveyor's Workbench".

The users that you assigned to the job will receive a notification email. When the users log in to SWB Orchestrator, the job will be listed on their dashboard. When new runs are added to the job, the users assigned to the job will be automatically assigned to the new run. See "Adding runs to a job". A new chat channel will also be created for the job, and the users assigned to the job will be added to the chat channel. See "Chat".

Adding users to a job

To add a user to an existing job:

  1. Go to the Dashboard page and click the Rig Users button () in the row for the job.
  2. In the Rig User Details window that opens, go to the Select User(s) field and select the users you want to assign to the job.
  3. Click the Assign button to add the users to the job.

The users that you assigned to the job will receive a notification email. When the users log in to SWB Orchestrator, the job will be listed on their dashboard. When new runs are added to the job, the users assigned to the job will be automatically assigned to the new run. See "Adding runs to a job".

Archiving a job

Before you can archive a job, all active runs must be closed. See "Closing a run".

To archive a job:

  1. Go to the Dashboard page and click the Edit Job button () in the row for the job.
  2. In the Edit Job & Run window, go to the Job Status field and select the Archive option.
  3. Click the Save button to save your changes, or click the Save and Create New button to save your changes and create a new job.

The job will be moved from the Dashboard page to the Archives page. The job will also be removed from the rig dashboard for the users assigned to the job. If you selected the Save and Create New button, the job will be archived and the Create Job & Run window will open. The customer, rig name, and field name from the archived job will be copied to the new job.
