About projects

There are four types of Surveyor's Workbench projects:

  • Local Surveyor's Workbench projects and all of the project data are stored on your local machine. For local projects, you will need to import the raw survey data into Surveyor's Workbench. Corrections can be exported to a file on the Reports screen. See "Local Projects".
  • Multi-lateral projects are used for runs where you need to use the exact same configuration as an existing project but with different survey data, such as sidetracks. When you create a multi-lateral project, the job details, magnetic parameters, FMS data, and BHA configuration are copied from the original project, but the raw survey data is not. See "Multi-Lateral Projects".
  • Auto-sag projects allow you to automatically send sag corrections for new surveys received from Spectrum. The job information, magnetic parameters, and raw survey data will be parsed from the Spectrum input file. As new surveys are added to the Spectrum job, Spectrum will send the surveys to Surveyor's Workbench. When Surveyor's Workbench receives the new surveys, they will be automatically corrected and sent back to Spectrum. See "Auto-Sag Projects".
  • Orchestrator projects are created on the SWB Orchestrator website, and corrections are performed in Surveyor's Workbench. For Orchestrator projects, the raw survey data will automatically loaded into the Surveyor's Workbench project. Corrections can be uploaded to the Orchestrator project on the Reports screen. See "Orchestrator Projects".

Saving projects

To save a local Surveyor's Workbench project or an SWB Orchestrator project:

  1. Go to File > Save Project in the main menu.
  2. In the Save Project window, go to the Project Location field and select the file location where you want to save the project.
  3. In the Report Location field, select the file location where you want reports to be saved.

The project will be saved to the selected file location. When you generate reports on the Reports screen, they will be saved to the selected report location.

Copying local projects

When you create a copy of a project, all of the project data will be copied to the new project. This includes survey data, magnetic parameters, and FMS data. However, custom project settings, configured under Settings in the main menu, are not copied to the new project.

To create a copy of a project:

  1. Go to File > Save Project As in the main menu.
  2. In the Save Project window, go to the Project Location field and select the file location where you want to save the project.
  3. In the Report Location field, select the file location where you want reports to be saved.

The new project will be saved to the selected file location. All of the project data, including surveys, magnetic parameters, and FMS data, will be copied to the new project. When you generate reports on the Reports screen, they will be saved to the selected report location.

Copying Orchestrator projects

When you create a copy of a project, all of the project data will be copied to the new project. This includes survey data, magnetic parameters, and FMS data. However, custom project settings, configured under Settings in the main menu, are not copied to the new project.

There are three options for copying Orchestrator projects:

  • Save as new Orchestrator with new ID but same reference data – Creates a copy of the Orchestrator project, using a different project ID than the current project. This is useful for creating a project for the next run in the well. See "Adding runs to a job".
  • Save as new Orchestrator with same ID – Creates a copy of the Orchestrator project using the same Orchestrator project ID, connecting to the same run in Orchestrator.
  • Save as normal project – Converts the Orchestrator project into a local project. All project data is preserved, but the Orchestrator connection is removed. This is useful if you need to work on the project offline. This cannot be undone — a local project cannot be converted back to an Orchestrator project.

To create a copy of an Orchestrator project:

  1. Go to File > Save Project As in the main menu.
  2. In the Save As window, select the desired option for copying the Orchestrator project.
  3. If you selected the Save as new Orchestrator with new ID but same reference data option, enter the project name and the project ID for the new run. See "Adding runs to a job".
  4. In the Save Project window, go to the Project Location field and select the file location where you want to save the project.
  5. In the Report Location field, select the file location where you want reports to be saved.

The new project will be saved to the selected file location. Surveyor's Workbench will connect the run to SWB Orchestrator, using the project ID for the run. All of the project data, including surveys, magnetic parameters, and FMS data, will be copied to the new project. When you generate reports on the Reports screen, they will be saved to the selected report location.
