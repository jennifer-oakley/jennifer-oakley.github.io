Sag Settings

The table below defines the settings in the Sag section of the Settings Editor window.

Field definitions for sag settings
Field Name Definition
Sag Lengths Unit system for BHA component lengths, including component length, cumulative length, bit-to-sensor distance, and bit-to-bend distance, as well as the start and end depth in the Inputs > Job Setup section. The Imperial unit system uses feet, and the Metric unit system uses meters.
Mud Weight Unit system for mud weight. The Imperial unit system uses ppg (lb/gal), and the Metric unit system uses kg/m3.
Sag Diameters Unit system for BHA component OD and ID. The Imperial unit system uses inches, and the Metric unit system uses millimeters.
New Sag Algorithm When enabled, SWB uses the updated sag algorithm, which is the recommended default. Both algorithms iteratively apply wall reaction forces to model BHA deflection within the borehole, but differ in how they determine where and how large a reaction force to apply. The updated algorithm prioritizes elements where the reaction force will have the greatest impact, resulting in faster convergence.
Maximum Sag Iterations The maximum number of iterations allowed for the sag solver at each survey station. If the solution does not converge within this limit, a warning is reported for that station and no correction is calculated for it, but sag corrections are still applied to all other stations.
Enforce RMS When enabled, sag analysis cannot be run if the MSA solution exceeds the RMS tolerance threshold. This prevents sag corrections from being calculated using a poor-quality MSA solution. If disabled, sag analysis can proceed regardless of MSA RMS results.
Δ Sag Tolerance The maximum allowable change in sag-corrected inclination between consecutive surveys. If the sag correction differs from the previous survey's corrected inclination by more than this threshold, a warning will appear to confirm before continuing. If AutoPilot mode is enabled, clicking No on the warning message will disable AutoPilot mode.
