About outgoing WITSML data

On the Setup > DAQ > WITSML Out tab, you can send calculated values from Victus™ Operation to one of the logs on the WITSML server. Victus™ Operation does not automatically create or configure any logs for outgoing WITSML data. However, after the appropriate logs have been created on the WITSML server, you can configure curve data to be sent from Victus™ Operation to the WITSML server. See "Configuring outgoing data".

Note

WITSML data may become briefly out of sync when the WAN1 port is disconnected or reconnected.

Warning

The validations on the WITSML Out tab currently allow you to map multiple curves to the same WITSML mnemonic. However, if multiple curves are mapped to the same WITSML mnemonic, the WITSML server will return an error.

Configuring outgoing data

To send additional curve data from Victus™ Operation:

  1. Go to the WITSML Out tab on the Setup > DAQ page.
  2. In the Frequency field, enter the frequency at which data should be sent to the WITSML channel.
  3. Click the Add button () at the bottom of the screen.
  4. In the new row that appears, select the check box in the first column.
  5. In the Category field, select the category for the curve data you want to send.
  6. In the Parameter field, select the name of curve you want to send from Victus™ Operation.
  7. In the WITSML Channel and WITSML Log fields, select the name of the WITSML channel and log that you want to send data to.
  8. In the Log Curve field, select the name of the curve in the WITSML log.
  9. In the Unit field, select the unit used for the curve in the WITSML log. The unit selected must match the unit assigned to the curve on the WITSML server.
  10. Click the Save Changes button () at the bottom of the screen.

Victus™ Operation will begin sending the data for the selected curve to the WITSML server.

Note

You cannot create WITSML logs and curves on the Setup > DAQ > WITSML Out tab. To create a new WITSML log, contact the administrator for the WITSML server.

Importing WITSML Out settings

When you export WITSML Out settings, they will be saved to a CSV file that you can import into another well.

To import WITSML Out settings:

  1. Go to the WITSML Out tab on the Setup > DAQ page.
  2. Click the Import button ().
  3. In the window that appears, navigate to the file you want to import and click Open.
  4. Click the Save Changes button ().

When you import WITSML Out settings, Victus™ Operation will attempt to map the parameters listed in the file to logs and curves from the primary WITSML channel. Victus™ Operation will begin sending the data for the selected curves to the WITSML server.

Publishing historical data

After you configure outgoing data, you can publish existing calculation data to the WITSML channel.

To publish historical data to WITSML:

  1. Go to the WITSML Out tab on the Setup > DAQ page.
  2. Click the Export button (), then select the Publish to WITSML option in the menu that appears.
  3. In the Publish to WITSML window, enter the start and end time for the historical data that you want to publish.
  4. Click the Export button () at the bottom of the window.

The progress for publishing the data will appear in the banner at the top of the screen. Another message will appear in the banner when all data has been published.

Field definitions

The table below defines the WITSML channel settings on the Setup > DAQ > WITSML Out tab.

Properties of outgoing WITSML data
Field Name Definition
Frequency Frequency at which data should be sent to the WITSML channel
Category Category that contains the Victus™ Operation curve data
Parameter Name of the Victus™ Operation curve
WITSML Channel WITSML channel receiving the data from Victus™ Operation
WITSML Log Log on the WITSML channel receiving the data
Log Curve Name of the curve receiving the data on the WITSML server
Unit Type Unit type for the selected Victus™ Operation curve. This is a read-only field, determined by the curve selected in the Parameter field.
Unit Unit assigned to the curve on the WITSML server
