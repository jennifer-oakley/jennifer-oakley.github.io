WITS data

WITS wellsite information transfer specification is a two-way communication system used for sending and receiving drilling data. WITS data can be sent through a serial port on the UPI, as well as across the network through a TCP/IP connection. Before sending or receiving WITS data over a network connection, you'll need to configure the UPI's internal router to correctly route WITS data. See "Router configuration".

The Setup > DAQ > WITS tab is used to configure Victus™ Operation to send and receive WITS data. There are four basic steps for sending and receiving WITS data:

  • Connect the UPI to the mud logger. The UPI has serial ports and WAN ports, which allow you to connect the UPI to the mud logger. See "Connecting the UPI".
  • Add a WITS channel. If you are receiving WITS data across a network, you'll need to add the WITS channels to Victus™ Operation. To connect to a WITS server, you'll need to know the IP address and port number for the server. See "WITS channels".
  • Add incoming records. Victus™ Operation will only receive data for the records listed on the WITS > In tab. See "Incoming WITS records".
  • Add outgoing records. On the WITS > Out tab, you can specify WITS data to send from Victus™ Operation. See "Outgoing WITS records".
