About the run algorithm

The run algorithm is defined on the Run tab. The run algorithm is the master workflow, and it is connected to the events, calculations, conditions, and flows defined on the Setup > OneClick™ page. By default, the run algorithm for Victus™ Operation is blank. The algorithm can be used to define special circumstances, like a kick or a loss. The algorithm is triggered in real-time, at one-second intervals.

There are three types of nodes you can add to the run algorithm:

  • Flow nodes are workflow nodes that direct the run algorithm to a flow defined on the Flows tab. When the flow is completed, the run algorithm will return to the next step in the algorithm. See "Adding flow nodes to the run algorithm".
  • Trigger nodes are conditional nodes that use the condition checks defined on the Conditions tab. Each trigger node has a No branch and a Yes branch. If the condition check is false, the run algorithm will follow the No branch. If it is true, it will follow the Yes branch. See "Adding trigger nodes to the run algorithm".
  • Action nodes are used to execute certain types of actions, such as triggering an event or adding a comment. When the run algorithm encounters the node, it will execute the actions listed in the action node. See "Adding action nodes to the run algorithm".

Adding flow nodes to the run algorithm

Before you can add a flow node to the run algorithm, you'll need to create the appropriate flow on the Flows tab. See "Creating flows".

To add a flow node to the run algorithm:

  1. Go to the Run tab on the Setup > OneClick™ page.
  2. Select the node that is closest to where you want to add the flow node.
    • To add a node immediately below the selected node, click the Flow Node button () in the gray box below the existing node.
    • To insert a node above the selected node, click the Insert Flow Node button () in the toolbar at the bottom of the screen.
  3. In the menu that appears, select the flow you want to use for the node.
  4. Click the Save Changes button () to save the run algorithm.

The run algorithm will be updated in the Victus™ Operation database. When the run algorithm encounters the flow node, it will be directed to the flow assigned to the flow node. When the flow is completed, the run algorithm will return to the next step in the algorithm.

Adding trigger nodes to the run algorithm

Before you can add a trigger node to the run algorithm, you'll need to create the appropriate condition check on the Conditions tab.

To add a trigger node to the run algorithm:

  1. Go to the Run tab on the Setup > OneClick™ page.
  2. Select the node that is closest to where you want to add the trigger node.
    • To add a node immediately below the selected node, click the Trigger Node button () in the gray box below the existing node.
    • To insert a node above the selected node, click the Insert Left Trigger Node button () or the Insert Right Trigger Node button () in the toolbar at the bottom of the screen.
  3. In the menu that appears, select the condition you want to use for the node.
  4. Click the Save Changes button () to save the run algorithm.

The run algorithm will be updated in the Victus™ Operation database. When the run algorithm encounters the trigger node, the node will be used to determine which branch the algorithm follows.

Adding action nodes to the run algorithm

Action nodes can be used to execute the following types of actions:

  • Comment – Adds a comment to the log on the Feedback > Comments page.
  • Event – Triggers the selected event, defined on the Setup > OneClick™ > Events tab.
  • Message – Triggers a message that appears in the status banner at the top of the screen.

To add an action node to the run algorithm:

  1. Go to the Run tab on the Setup > OneClick™ page.
  2. Select the node that is closest to where you want to add the action node.
    • To add a node immediately below the selected node, click the Action Node button () in the gray box below the existing node.
    • To insert a node above the selected node, click the Insert Action Node button () in the toolbar at the bottom of the screen.
  3. In the menu that appears, select the Actions option.
  4. In the table that appears, select the actions that should be associated with the new action node.
  5. Click the Save Changes button () to save the run algorithm.

The run algorithm will be updated in the Victus™ Operation database. When the run algorithm encounters the node, it will execute the actions listed in the new action node.
