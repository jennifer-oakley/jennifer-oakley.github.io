WITS data

The WITS Data window shows the incoming data received from WITS channels. The values shown in this window are based on the incoming WITS records defined on the Setup > DAQ > WITS > In tab. See "WITS data".

To view incoming WITS data:

  1. Click the Indicators button () in the Utilities toolbar.
  2. Select the DAQ option.
  3. Select the WITS option from the list that appears.

The WITS Data window will open. All of the incoming data for all WITS channels will appear in the WITS Data window.

Tip

If there is no data shown in the WITS Data window, or the data seems to be out-of-date, Victus™ Operation may have lost communication with the data acquisition service. To fix this issue, try restarting the DAQ service. See "WITS alarms".

Analog sensors

The Analog Data window shows the incoming data received from analog sensors. The values shown in this window are based on the spare analog sensors defined on the Setup > Calibration > Spare Analogs tab. See "Spare analogs".

To view incoming analog sensor data:

  1. Click the Indicators button () in the Utilities toolbar.
  2. Select the DAQ option.
  3. Select the Analogs option from the list that appears.

The Analog Data window will open, showing all incoming data from analog sensors.

WITSML data

The incoming data from each WITSML log is shown in a separate window for each log. The values shown in this window are based on the mnemonics selected for the channel on the Setup > DAQ > WITSML tab. See "WITSML channels".

To view incoming data from a WITSML log:

  1. Click the Indicators button () in the Utilities toolbar.
  2. Select the DAQ option.
  3. Select the name of the WITSML channel from the list that appears.
  4. Select the name of the log.

A window for the WITSML log will open, showing all of the incoming data for that log.

Tip

If there is no data shown in the WITSML window, or the data seems to be out-of-date, Victus™ Operation may have lost communication with the data acquisition service. To fix this issue, try restarting the DAQ service. See "WITSML alarms".
