About WITS data

WITS wellsite information transfer specification is a two-way communication system used for sending and receiving drilling data. WITS data can be sent through a serial port on the UPI, as well as across the network through a TCP/IP connection. Before sending or receiving WITS data over a network connection, you'll need to configure the UPI's internal router to correctly route WITS data. See "Configuring the UPI for WITS".

The Setup > DAQ > WITS tab is used to configure Victus™ Operation to send and receive WITS data. There are three basic steps for sending and receiving WITS data:

  • Connect the UPI to the mud logger. The UPI has serial ports and WAN ports, which allow you to connect the UPI to the mud logger. See "Connecting the UPI".
  • Add a WITS channel. If you are receiving WITS data across a network, you'll need to add the WITS channels to Victus™ Operation. To connect to a WITS server, you'll need to know the IP address and port number for the server. See "Adding WITS channels".
  • Add incoming records. Victus™ Operation will only receive data for the records listed on the WITS > In tab. See "Adding incoming records".

Connecting the UPI

Null modem serial cable

Straight serial cable

To send or receive WITS data, you'll need to connect the mud logger to the appropriate port on the UPI. There are four ports on the UPI that can be used to connect the UPI to the mud logger:

  • The WITS / Serial 1 and Serial 2 ports are bi-directional serial ports that can be used for incoming and outgoing WITS data. The Serial 1 port uses a straight serial cable, and the Serial 2 port uses a null modem serial cable.
  • The WITS Out serial port can only be used for outgoing WITS data, using a straight serial cable.
  • The WAN-2 port is a bi-directional network port that can send and receive third-party WITS data, using a standard network cable.

The WITS channels for the serial ports on the UPI are created automatically, and can't be edited. If you are using a network connection for WITS, however, you'll need to create a WITS channel for each connection. See "Adding WITS channels".

Adding WITS channels

The Setup > DAQ > WITS > Channels tab lists the incoming and outgoing WITS channels. This list tells Victus™ Operation the IP address and port number to use for sending and receiving WITS data.

Warning

If you delete a WITS channel, any associated raw data curves used in a dashboard on the Home screen will also be deleted.

Note

It is only necessary to add WITS channels if you are using a network connection to send WITS data. If the UPI is only using serial ports to connect to the mud logger, you do not need to add any WITS channels. The WITS channels for the serial ports on the UPI are created automatically.

To add a WITS channel:

  1. Connect the mud logger to the WAN-2 port on the UPI.
  2. Go to the Channels tab on the Setup > DAQ > WITS tab.
  3. Click the Add button () at the bottom of the screen.
  4. In the new row that appears, enter a name for the channel in the Channel field.
  5. In the IP Address field, enter the IP address of the computer that will be sending the data.
  6. In the Port field, enter the port number on which the data will be sent or received.
  7. Click the Save Changes button () to save your changes.

The new channel will appear in the list of available WITS channels on the DAQ > WITS > In and DAQ > WITS > Out tabs. The data acquisition service will also be restarted so that Victus™ Operation can connect to the new channel.

Adding incoming records

The Setup > DAQ > WITS > In tab lists the incoming WITS mnemonics that will be received by Victus™ Operation. This list tells Victus™ Operation what WITS data to look for and where to look for it. Before Victus™ Operation can receive any WITS data, you'll need to add entries for all incoming WITS mnemonics. Any incoming WITS data that is not listed on the DAQ > WITS > In tab will be ignored by Victus™ Operation.

To add an entry for an incoming WITS mnemonic:

  1. Go to the In tab on the Setup > DAQ > WITS tab.
  2. Click the Add button () at the bottom of the screen.
  3. In the new row that appears, go to the Category field and select the WITS record that the data will be sent on.
  4. In the Tag ID field, select the name of the WITS mnemonic.
  5. In the Channel field, select the name of the WITS channel on which the mnemonic will be received.
    • If the data will be received through the WITS / Serial 1 port on the UPI, select the Serial 1 channel.
    • If the data will be received through the Serial 2 port on the UPI, select the Serial 2 channel.
  6. In the Title field, enter a name for the entry. The name can be the same as the WITS mnemonic, or it can be something different.
  7. In the Type field, select the type of measurement unit for the mnemonic, e.g. length or penetration rate. See "List of unit types".
  8. In the Unit field, select the measurement unit for the incoming data.
  9. When you have finished adding WITS mnemonics, click the Save Changes button () to save your changes.

The new entries will be added to the Victus™ Operation database. The names entered in the Title field will appear in the mapping settings and the WITS Data window. To view the incoming WITS data, click the Indicators View button () in the Utilities toolbar and select the DAQ option. See "Data acquisition indicators".

After you set up the WITS data, the next step is to configure any incoming WITSML data. See "Setting up WITSML data".
