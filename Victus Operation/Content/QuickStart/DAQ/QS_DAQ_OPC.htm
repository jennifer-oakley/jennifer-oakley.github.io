About OPC UA data

The OPC UA protocol is a data exchange standard used for industrial communication, which allows machine-to-machine and PC-to-machine communication. On the Setup > DAQ > OPC UA tab, you can configure Victus™ Operation to send and receive data from OPC UA devices.

There are three basic steps for sending and receiving OPC UA data:

  • Add an OPC UA channel. To connect to an OPC UA device, you'll need to know the URL to access the OPC UA device. See "Adding OPC UA channels".
  • Add inbound data tags. You'll need to add a data tag for each inbound parameter received from the OPC UA device. See "Adding inbound data tags".
  • Add outbound data tags. If you want to send data to the OPC UA device, you'll also need to add data tags for each outbound parameter sent to the OPC UA device. See "Adding outbound data tags".
Warning

If you delete a OPC UA channel, any associated raw data curves used in a dashboard on the Home screen will also be deleted.

Adding OPC UA channels

To add an OPC UA channel:

  1. Go to the Setup > DAQ > OPC UA tab.
  2. Click the Add button (). A new tab will appear in the right toolbar.
  3. In the Channel Name field, enter a name for the OPC UA channel.
  4. In the URL field, enter the full address of the OPC UA device.
  5. Enter the frequency with which Victus™ Operation should query the device for data.
  6. Add data tags for each inbound parameter received from the OPC UA device. See "Adding inbound data tags".
  7. Add data tags for each outbound parameter sent to the OPC UA device. See "Adding outbound data tags".
  8. Click the Save Changes button ().

Victus™ Operation will begin receiving data from the OPC UA device. To view the incoming OPC UA data, click the Indicators View button () in the Utilities toolbar and select the DAQ option. See "OPC UA data". To check the status of incoming and outgoing data for the channel, go to the Alarms > Status > OPC UA tab. See "OPC UA alarms".

Adding inbound data tags

To add an inbound data tag to an OPC UA channel:

  1. Go to the Setup > DAQ > OPC UA tab.
  2. Select the OPC UA channel that you want to add a data tag to.
  3. Go to the Inbound tab for the channel.
  4. Click the Add Data Tag button (). A new row will appear in the lower portion of the screen.
  5. In the Name field, enter the name of the parameter on the OPC UA device.
  6. In the Node Id field, enter the ID of the node on the OPC UA device that is sending the data.
  7. In the Unit Type field, select the type of measurement unit for the parameter, e.g. pressure or temperature. See "List of unit types".
  8. In the Unit field, select the measurement unit for the data.
  9. Click the Save Changes button ().

The data tags will be added to the Victus™ Operation database. The name entered in the Name field will appear in the mappings settings and the DAQ indicator for the OPC UA channel. To view the incoming data from the log, click the Indicators View button () in the Utilities toolbar and select the DAQ option. See "OPC UA data".

After you configure the incoming OPC UA data, the next step is to map the incoming data to the well parameters on the Setup > Mappings page. See "Defining well parameters".
