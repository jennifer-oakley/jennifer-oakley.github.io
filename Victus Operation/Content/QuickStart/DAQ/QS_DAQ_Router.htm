About router configuration

WITS data is sent and received through the WAN-2 port on the UPI. Before you can send or receive WITS data over a network connection, you'll need to configure the UPI's internal router so that it routes WITS data correctly. You'll need to obtain the following network details from the mud logger:

  • IP address of the mud logger
  • Subnet mask
  • Default gateway

If the UPI will be receiving data from a WITS provider, you'll also need the port number. After you've obtained the required network information, you can configure the UPI's internal router. There are three basic steps required to configure the WAN-2 port:

  • Configure the connection settings. The router can use either a static or dynamic IP address, depending on the way the WITS network is set up. If you don't know which type of connection you should use, contact the WITS provider. See "Configuring the connection settings".
  • Bind incoming WITS data to the WAN-2 port. Binding the incoming data to the WAN-2 port ensures that all data received from the mud logger will be routed through the WAN-2 port on the UPI. See "Receiving WITS data".
  • Bind outgoing WITS data to the WAN-2 port. Binding the outgoing data to the WAN-2 port ensures that all data sent to the mud logger will be routed through the WAN-2 port on the UPI. See "Sending WITS data".

Configuring the connection settings

To configure the connection settings for the WAN-2 port:

Connection settings for a static IP address

Connection settings for a dynamic IP address

  1. Connect a network cable from the operation station to the LAN-1 port on the UPI.
  2. In Internet Explorer, go to https://192.168.79.254.
  3. When prompted, click the Continue to this website button.
  4. Log in to the router, using the following log-in credentials:
    • Username: Microflux
    • Password: Micr0flux
  5. Using the menu on the left side of the page, go to the Setup > Network page.
  6. Under the WAN Settings Table, select the WAN2 option and click the Edit button.
  7. On the WAN Connection Settings page, enter the connection settings for the WAN-2 port.
    • To use a static IP address, go to the WAN Connection Type field and select Static IP. Enter the IP address, subnet mask, and default gateway address provided by the mud logger.
    • To use a dynamic IP address, go to the WAN Connection Type field and select Obtain an IP automatically.
  8. Click the Save Changes button to save your changes.

The WAN-2 port on the UPI will be able to connect to the WITS provider on the rig, using the specified connection settings.

Receiving WITS data

To configure the WAN-2 port to receive WITS data:

Dual WAN settings for the UPI

Protocol bindings for incoming WITS data

  1. Using the menu on the left, go to the System Management > Dual WAN page.
  2. Go to the Load Balance section and select the Load Balance (Auto Mode) option.
  3. Under the Interface Setting Table, select the WAN2 option and click the Edit button.
  4. Go to the Protocol Binding Table, then click the Service Management button below the table.
  5. Scroll to the bottom of the Service Management window that appears, then click the Add button to create a new service.
  6. In the new row that appears, enter the properties of the new service.
    • Service Name: Any name useful for identifying the service.
    • Protocol: TCP
    • Port Range: The port range should include the port number provided by the mud logger.
  7. Click the Save Changes button to save your changes.
  8. Navigate back to the Protocol Binding Table for WAN2. Click the Add button to add a new protocol.
  9. In the new row that appears, select the name of service that you created. Enter the following settings:
    • Source IP: 192.168.79.101 to 192.168.79.150
    • Destination IP: The range of destination IP address should include the IP address for the mud logger. For example, if the mud logger's IP address is 172.19.45.82, the range of destination IP addresses could be 172.19.45.80 to 172.19.45.85.
    • Interface: WAN2
    • Status: Enabled
  10. Click the Save Changes button to save your changes.

The new protocol will be saved to the UPI's internal router. All data received from the mud logger will be routed through the WAN-2 port on the UPI.

Sending WITS data

To configure the WAN-2 port to send WITS data:

Dual WAN settings for the UPI

Protocol bindings for outcoming WITS data

  1. Using the menu on the left, go to the System Management > Dual WAN page.
  2. Go to the Load Balance section and select the Load Balance (Auto Mode) option.
  3. Under the Interface Setting Table, select the WAN2 option and click the Edit button.
  4. Go to the Protocol Binding Table and click the Add button to create a new protocol.
  5. In the new row that appears, enter the properties of the new service.
    • Service: WITS-Out [TCP/4000~4003]
    • Source IP: 192.168.79.101 to 192.168.79.150
    • Destination IP: The range of source IP addresses should match the IP range from the third party.
    • Interface: WAN2
  6. Select the checkmark in the Status column to enable the protocol.
  7. Click the Save Changes button to save your changes.

The new protocol will be saved to the UPI's internal router. All data sent to the mud logger will be routed through the WAN-2 port on the UPI.

After you configure the UPI for WITS, you'll need to set up the incoming WITS data. See "Setting up WITS data".
