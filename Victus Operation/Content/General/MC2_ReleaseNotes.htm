Victus™ Operation v2.3.9.1 Release Notes

In this article

Bugs fixed

Archived release notes

Welcome to Victus™ Operation v2.3.9.1. For a full list of the updates in v2.3.9.1, refer to the sections below.

In addition, installation and upgrade guides for Victus™ Operation v2.3.9.1 may be downloaded from the Software Solutions Site

What's new

The features in this section are new for Victus™ Operation v2.3.9.1.

  • Updated the validations for the low limit. The maximum value of the low limit is determined by the option selected in the System goes to SBP mode at field on the Setup > Control Algorithms > Limits tab. The low limit cannot exceed the percentage of the high limit at which the system switches to SBP mode. If it exceeds that value, the low limit will set to the percentage of the high limit. See "Limits algorithm".
  • Updated the banner message that appears when communication with the ICU failed. The message will now include the ping status of the ICU and the UPI, as well as ports 6100 and 6200 on the ICU. See "Banner messages".
  • When one of the panels connected to the MPD network has a different time or time zone than the Operation panel, a message will now appear on the Alarms > Active page. A message will also appear if the ICU has a different time than the Operation panel, or if daylight savings time is turned off on the Operation panel. See "Alarm messages".
  • The Feedback > Logs page now includes the logs entries for data acquisition. See "Diagnostic logs".
  • Added an option for 3-inch Cortec CXP3 chokes to the list of choke models in the MPD Manifold properties on the Setup > Surface Equipment > Piping and Instrumentation tab. See "Equipment specifications".
  • Updated the firmware for Victus™ RCD to support the Advantec HCS.
  • Added the ability to map surface back pressure and flow out to an external data source, such as WITS or WITSML. To map the parameters, go to Setup > Mappings page and select the data source for surface back pressure and flow out. See "Mappings".
  • When the Flow Out parameter is mapped to an external data source on the Setup > Mappings page, the incoming data will be used for gain/loss calculations in the Virtual Trip Tank 1 and Virtual Trip Tank 2 windows. Previously, the hydraulics model only determined the value of flow out using sensor readings or simulated values from DrillSim, Endeavour, or RelyOnNutec. See "Counters".
  • Cements and muds are now both created on the Setup > Fluid Properties > Fluids tab, and cements can now be selected as the pump fluid. Previously, the Fluid Properties page had separate tabs for muds and cements. See "Fluid properties".
  • When you connect a UPI to Victus™ Operation, Victus™ Operation will now be able to read incoming sensor data received through analog sensors connected to the UPI. To calibrate the sensor data, go to the Setup > Calibration > Spare Analogs tab. See "Spare analogs".
  • Added the ability to map choke positions and density out to an external data source, such as WITS or modbus. To map the parameters, go to Setup > Mappings page and select the data sources for the choke positions and density out.See "Mappings".
  • Added the ability to export the logs and databases for Victus™ Operation to a zip file. Exporting logs and databases allows you to easily share important troubleshooting information with the support team when you encounter issues with Victus™ Operation. You can include agent logs, DAQ logs, panel logs, primary and archived databases, and the well configuration in the export. To export logs and databases, go to the File > Export page and select the Logs and Databases option. See "Exporting logs and databases".
  • Updated Victus™ Operation to support the latest version of the Advantec HCS Modbus controller for Weatherford RCDs. If any alarms are detected from the HMI, they will be listed on the Alarms > Sensors > RCD tab. See "RCD communication".

Bugs fixed

The bugs in this section have been fixed in Victus™ Operation v2.3.9.1.

  • When you use the Capture Video utility, the video should now capture the entire screen.
  • When the ICU is restarted or power-cycled, the Auto Control option will now be automatically enabled on the Setup > Control Algorithms > Kick tab. Previously, auto-control was automatically enabled on the ICU after restart, but not in the Victus™ Operation application.
  • Fixed an installation issue with the Victus™ prerequisites. Previously, the installation frequently failed, due to an issue with the SQL service account.
  • When pore pressure exceeds annular pressure, the difference between these values will used to calculate a safe low limit value. Previously, when this scenario occurred, the low limit would exceed the high limit.
  • When there isn't a value for pore pressure or annular pressure, the calculated low limit will no longer be negative.
  • Increased frictional pressure gradient threshold. WISE 1092992
  • When you are running HPFM totalizer calculations, the calculations will now stop if the communications alarm, clog alarm, or wear alarm are triggered for any HPFM. Previously, the calculation continued running, even when there was no connection to the HPFM.
  • When you switch to AP mode, the high limit will become dynamic. Previously, the high limit would remain the same when you switched to AP mode from C mode. WISE 1100369
  • Restored the OneClick™ Events, Conditions, Flows, and Run tabs on the Setup > OneClick™ page. WISE 1097816 See "OneClick™ settings".
  • Updated the high limit calculation to address situations where the high limit is zero. If the high limit cannot be calculated, or the calculated high limit is zero, the high limit will be set to the previous value. If there is no previous value, the high limit will be set to 5000 psi. WISE 1100108 See "About the high limit".
  • The standby station must now be fully synced in order to take the operation role when the operation station loses connection. If the standby station is not fully synced, a fully-synced backup panel will become the new operation station. If this situation occurs, a banner message will also appear on the standby station. WISE 1100108. See "Guidelines for network roles".
  • When the operation station loses its connection to the rig network due to power loss or another connectivity issue, an alarm will now be triggered on all stations. WISE 1100826.
  • Fixed a firmware issue with Cortec CXP3 electric chokes not responding when the high limit was reached. WISE 1097088.
  • When you change the pump fluid, the current fluid volumes will be immediately saved to the database. Previously, the fluid configuration was only saved every 15 seconds. When a role transfer occurred within that window, the pump fluid would be reset to the default fluid. WISE 1019861, 1016310

Notes to consider

The issues in this section are known issues for Victus™ Operation v2.3.9.1.

  • When you drag a curve in a time-based graph from one track to another track, the curve data may get corrupted and show abnormal spikes in the data.
  • Bit to Surface time under the Counters menu does not consider booster flow when the bit is above the booster.
  • If a choke is in calibration mode, it will not open when surface backpressure reaches 98% of the high limit. The choke will only open when it is out of calibration mode.
  • When communication with the high pressure flow meter fails, Victus™ Operation keeps the last known good values. Additionally, HPFM totalizers continue counting and do not stop due to the loss of communication.

  • If any active chokes are switched to Manual mode when Victus™ Operation switches to SBP mode, the SBP setpoint will update, according to the current pressure. If any chokes are switched back to Auto mode, and the surface backpressure does not exceed the threshold selected in the System goes to SBP mode at field on the Setup > Control Algorithms > Limits tab, the SBP setpoint will be set to the current pressure at the time the chokes were switched to Auto mode. See "Limits algorithm".
